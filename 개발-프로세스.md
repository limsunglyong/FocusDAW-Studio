## 1. 전체 로드맵 (Phase Flow)

현재 출발점은 `v1.1.11`(Mixer 창 버그 수정 + Electron 42 보안 업데이트 완료)이며,
다음 핵심 목표는 **트랙별 시간(BPM)·주파수(Pitch) 제어 고도화**입니다.

```mermaid
flowchart TD
    START([v1.1.11<br/>현재: Web Audio 엔진 안정화 완료])

    subgraph GOAL [핵심 목표 · 섹션 2]
        G1[BPM 측정<br/>BPM Detection]
        G2[Time Stretching<br/>Pitch 유지 · BPM 변경]
        G3[Pitch Shifting<br/>BPM 유지 · Pitch 변경]
    end

    P0[Phase 0<br/>BPM 1차 프로토타입<br/>현행 Web Audio 기반]
    P1[Phase 1<br/>BPM 측정 엔진 + UI]
    P2[Phase 2<br/>Time-Stretch / Pitch-Shift<br/>DSP 프로세서]
    P3[Phase 3<br/>트랙별 편집 UI<br/>+ Export 연동]

    GATE{JUCE 전환<br/>판단 게이트}

    JUCE[JUCE 기반<br/>네이티브 오디오 엔진<br/>재구축 · 대형 작업]
    KEEP[Web Audio / AudioWorklet<br/>기반 유지 · 고도화]

    START --> GOAL
    GOAL --> P0
    P0 --> P1 --> P2 --> P3 --> GATE
    GATE -->|품질·정밀도 부족| JUCE
    GATE -->|충분| KEEP

    style START fill:#2d5a3d,color:#fff
    style P0 fill:#3a5f7d,color:#fff
    style GATE fill:#7d5a2d,color:#fff
    style JUCE fill:#5a2d3d,color:#fff
```

## 3. JUCE 전환 판단 기준 (Decision Gate)

Phase 0~3 완료 후, 아래 조건을 평가하여 JUCE 네이티브 엔진 전환 여부를 결정합니다.

```mermaid
flowchart TD
    Q1{다중 트랙 실시간<br/>Time Stretch 품질이<br/>충분한가?}
    Q2{실시간 재생과<br/>Export 결과가<br/>안정적으로 일치하는가?}
    Q3{VST3 · LUFS/True Peak<br/>· 네이티브 DSP가<br/>필수 요구사항인가?}

    KEEP[Web Audio / AudioWorklet<br/>기반 유지]
    JUCE[JUCE 전환<br/>별도 대형 작업으로 계획]

    Q1 -->|예| Q2
    Q1 -->|아니오| JUCE
    Q2 -->|예| Q3
    Q2 -->|아니오| JUCE
    Q3 -->|예| JUCE
    Q3 -->|아니오| KEEP

    style JUCE fill:#5a2d3d,color:#fff
    style KEEP fill:#2d5a3d,color:#fff
```

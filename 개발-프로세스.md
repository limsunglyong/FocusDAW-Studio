## 3. Time Stretch 진행 기준

Time Stretch 고도화는 “새 엔진이 좋아 보인다”가 아니라 “현행 `ffmpeg atempo`가 실제 작업에서 부족하다”는 신호가 있을 때 시작합니다.

```mermaid
flowchart TD
    INPUT[실제 작업 샘플<br/>WAV/MP3/M4A/FLAC 등]
    ATEMPO[현재 기준선<br/>ffmpeg atempo]
    CHECK{청감/수치 기준에서<br/>문제가 있는가?}

    OK[현행 유지<br/>앱 사용성 개선으로 이동]
    CASES[문제 케이스 보관<br/>원본 + 비율 + 관찰 내용]
    CANDIDATES[후보 엔진 추가 비교<br/>SoundTouch / Rubber Band<br/>Phase Vocoder]
    SCORE[측정 항목<br/>길이 정확도 · 피치 유지<br/>트랜지언트 · LUFS/RMS<br/>처리 시간 · 패키징 부담]
    DECIDE{기준선보다<br/>명확히 우수한가?}
    ADOPT[Export 엔진 채택 검토]
    REJECT[보류<br/>기준선 유지]

    INPUT --> ATEMPO --> CHECK
    CHECK -->|아니오| OK
    CHECK -->|예| CASES --> CANDIDATES --> SCORE --> DECIDE
    DECIDE -->|예| ADOPT
    DECIDE -->|아니오| REJECT

    style ATEMPO fill:#2d5a3d,color:#fff
    style CHECK fill:#7d5a2d,color:#fff
    style DECIDE fill:#7d5a2d,color:#fff
    style ADOPT fill:#3a5f7d,color:#fff
```

## 4. 실시간 Keep Pitch 판단 기준

실시간 피치 보존은 Export보다 범위가 큽니다. v1.2.33에서는 AudioWorklet로 바로 들어가지 않고 캐시형 Time Stretch 프리뷰를 먼저 붙였으므로, B-123 결과에 따라 이 경로를 보정할지 AudioWorklet 단계로 넘어갈지 판단합니다.

```mermaid
flowchart TD
    R0{캐시형 프리뷰가<br/>B-123 기준을 만족하는가?}
    R1[현재 유지<br/>캐시형 프리뷰 + Export Keep pitch]
    R2[프리뷰 보정<br/>준비 표시·메모리 제한<br/>WSOLA 파라미터 조정]
    R3{동일 DSP 코어를<br/>실시간/Export에 공유 가능한가?}
    R4[AudioWorklet 프로토타입]
    R5[실시간/Export 일치성 검증]
    R6{CPU·음질·패키징<br/>기준 통과?}
    R7[본류 통합]
    R8[JUCE 전환 검토]

    R0 -->|예| R1
    R0 -->|아니오| R2 --> R3
    R3 -->|예| R4 --> R5 --> R6
    R3 -->|아니오| R8
    R6 -->|예| R7
    R6 -->|아니오| R8

    style R0 fill:#7d5a2d,color:#fff
    style R6 fill:#7d5a2d,color:#fff
    style R8 fill:#5a2d3d,color:#fff
```

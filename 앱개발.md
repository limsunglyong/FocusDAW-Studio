# FocusDAW — Stem Studio · 앱 개발 문서

## v1.17.18 미해결 버그 및 작업 정리 — 2026-07-09

### BUG — Audio In 입력 레벨 표시 경로 불일치

- 의도: Audio In 트랙의 `IN` 슬라이더 내부에 실제 audio input level을 초록/노랑/빨강 레벨 미터로 표시한다.
- 실제 증상:
  - 입력 레벨이 `IN` 슬라이더가 아니라 우측 출력 level meter에 표시된다.
  - 이후 재생에서는 해당 트랙의 오디오 출력 level meter가 작동하지 않는다.
  - 녹음 중에도 `IN` 슬라이더 내부에는 입력 레벨이 표시되지 않는다.
- 현재 판단: 하나의 `level` 값을 트랙 출력 미터와 IN 입력 미터가 공유하면서 표시 대상이 뒤바뀐 상태다. 다음 수정에서는 `inputLevel`과 `playbackLevel`을 별도 prop/state로 분리해야 한다.

### BUG — 첫 녹음 시작 지연

- 증상: 앱 실행 후 최초 녹음만 Record 클릭 후 실제 녹음 시작까지 지연되며, 두 번째 녹음부터는 상대적으로 즉시 시작된다.
- 적용했으나 효과가 없었던 시도:
  - Audio In 트랙 생성 시 입력 장치 사전 개방
  - ARM 활성화 시 입력 장치 재준비
  - Record 클릭 시 writer만 시작하도록 흐름 분리
- 현재 판단: UI 명령 시점이 아니라 JUCE 장치 재구성·첫 콜백 도착·writer thread 준비 중 어느 구간에서 지연되는지 계측이 필요하다. 다음 작업에서는 `setAudioInput 요청/완료`, `recordingStarted`, 첫 audio callback, 첫 writer write 시각을 각각 기록해 병목을 분리한다.

### 이번 작업 정리

- 앱 patch 버전을 `v1.17.18`로 갱신했다.
- 위 두 항목은 해결 완료로 처리하지 않고 재현 가능한 미해결 버그로 유지한다.
- 기존 사전 개방 코드는 효과가 확인되지 않았으므로 완료 근거로 사용하지 않는다.

### 최근 수정 및 반영 사항 (v1.17.17)
- **Audio In 개발 Phase 3 시작 — Merge Tracks / Bounce 1차 구현**: 선택된 파일/Bounce 트랙 2개 이상을 WebAudio 오프라인 렌더로 합쳐 새 `kind: "bounce"` 트랙을 생성하는 `mergeTracks(trackIds, options)` 및 `addBounceTrack()` 흐름을 추가했다. 기존 `joinClips()`와 분리된 트랙 단위 Bounce 동작이며, 항상 전체 프로젝트 범위를 기준으로 Stereo/Mono, 원본 처리(`Keep + Mute` 기본, Keep, Delete)를 모달에서 선택할 수 있다.
- **Merge Tracks 실행 흐름 단순화**: `Loop Range` 옵션을 제거하고 `Create Bounce`를 누르면 Merge Tracks 모달을 즉시 닫은 뒤 Rendering 로딩 화면으로 전환되도록 했다.
- **Merge Tracks 이름 편집 및 액션 영역 구분**: Merge Tracks 모달에 기본 Bounce 트랙 이름이 채워진 `Track Name` 입력칸을 추가했다. 사용자가 입력한 이름은 생성되는 Bounce 트랙 이름과 WAV 파일명에 반영되며, `Cancel`/`Create Bounce` 버튼 영역 위에는 수평 구분선을 추가했다.
- **Merge Tracks 비활성 커서 정리**: 파일 트랙 선택이 없거나 1개만 선택되어 Merge 실행 조건을 만족하지 않을 때 금지 커서가 뜨지 않도록 하고, 기존 안내 말풍선만 유지한다.
- **Bounce 파일 저장 경로 추가**: `.focus` 프로젝트를 열었는지 여부와 관계없이 결과 WAV를 첫 번째 선택 트랙의 원본 오디오 파일 폴더 아래 `Bounces` 폴더에 저장하고, 저장된 파일 경로를 Bounce 트랙 source에 연결한다. 원본 파일 경로가 없는 예외적인 트랙에서는 프로젝트 경로를 fallback으로 사용한다.
- **선택 트랙 렌더 정책 정리**: Bounce 렌더는 선택된 트랙만 대상으로 하며 Gain/Pan/automation/track FX/master FX를 기존 Export 그래프와 같은 WebAudio 경로로 반영한다. 다른 트랙의 Solo 상태 때문에 선택 트랙 Bounce가 무음이 되지 않도록 선택 범위 렌더에서는 외부 Solo를 무시하고, 선택 트랙 자체의 Mute는 반영한다.
- **B-217 보완 — Merge Tracks Undo/Redo 복원**: Undo 스냅샷이 기존 트랙의 파라미터만 복원하고 트랙 추가/삭제를 처리하지 못하던 문제를 수정했다. 스냅샷에 트랙 버퍼와 메타데이터를 포함하고, 복원 시 누락 트랙 추가/초과 트랙 제거/트랙 순서 복원을 수행한다. 네이티브 엔진 사용 중에도 Undo/Redo 후 트랙 목록을 다시 동기화한다.
- **B-217 보완 — 마지막 트랙 삭제 Undo 비활성화**: 트랙이 1개만 남은 상태에서 해당 트랙을 삭제하면 빈 프로젝트 정리 동작으로 보고 Undo/Redo 스택을 비운다. 이 경우 Undo 버튼과 Ctrl+Z로 삭제된 마지막 트랙이 복원되지 않는다.
- **B-217 사전 보완 — 앱 시작 검은 화면 수정**: `renderMergedTracks` callback이 선언되기 전에 `pushUndo`를 dependency array에서 참조해 첫 렌더링 시 renderer가 중단되던 문제를 수정했다. `pushUndo` 선언 뒤에 Bounce 실행 callback을 배치해 앱 시작 화면이 정상 표시되도록 정리했다.
- **B-217 사용자 확인 완료**: Merge Tracks / Bounce 1차 구현과 금일 보완 사항이 사용자 확인 완료되었고, 시험 항목은 [시험-아카이브.md](시험-아카이브.md)에 B-217로 이관했다.
- 대상: [audio-engine.js](audio-engine.js), [audio-bridge.js](audio-bridge.js), [app.jsx](app.jsx), [electron/main.js](electron/main.js), [electron/preload.js](electron/preload.js), [package.json](package.json), [version.js](version.js). 네이티브 재빌드 불필요, renderer 재빌드 필요.

### 이전 수정 및 반영 사항 (v1.17.16)
- **Audio In 개발 Phase 2 시작 — File Tracks 그룹 상태/선택 기반 안정화**: File Tracks 그룹 헤더에 전체 파일 트랙 수, 활성 트랙 수, Solo/Mute 상태 배지, 그룹 레벨 미터, 선택 트랙 수를 표시하도록 보강했다. 파일 트랙 헤더는 `Ctrl/Cmd + Click`으로 다중 선택, `Shift + Click`으로 범위 선택할 수 있으며, 선택된 트랙은 헤더와 레인 배경으로 강조된다.
- **Merge Tracks 진입점 설계**: 파일 트랙 2개 이상 선택 시 그룹 헤더의 `Merge Tracks...` 진입점이 활성화된다. 실제 Bounce 렌더링은 Phase 3 범위로 두고, 현재는 선택 트랙 목록과 Phase 3 안내 모달을 표시한다.
- **B-216 보완 반영**: `Shift + Click` 범위 선택을 헤더/레인에서 안정적으로 동작하도록 보강하고, 선택 강조를 테두리보다 헤더 배경 변화 중심으로 조정했다. `Merge Tracks...` 버튼 앞에는 두 파일이 하나로 합쳐지는 아이콘과 선택 트랙 개수를 함께 표시한다.
- **B-216 2차 보완**: 일반 레인 클릭도 파일 트랙 선택 앵커로 잡히도록 해 `Shift + Click` 범위 선택 실패 가능성을 줄였다. Merge 버튼 표시는 선택 개수 → 두 문서가 하나의 문서로 합쳐지는 아이콘 → 버튼 라벨 순서로 재배치했다.
- **B-216 3차 보완**: `Shift + Click` 범위 선택 확인 결과를 반영하고, Merge 버튼 아이콘을 단일 문서 아이콘으로 단순화했다. 표시 순서는 선택 개수 → 문서 아이콘 → `Merge Tracks...`로 조정했다.
- **B-216 선택 해제 보완**: 선택된 파일 트랙이 1개뿐일 때 같은 트랙을 다시 클릭하면 선택 상태가 해제되도록 했다.

  - 증상: 재생 도중 현재 재생 시점보다 **뒤(이전 시점)** 로 seek하고 **time zoom-in** 하면, 플레이헤드 세로선이 트랙 레인을 벗어나 **좌측 sticky 헤더 영역까지 세로로 그려짐**. (v1.15.2 이후 발견.) 1차 수정에서 일반 트랙은 잡혔으나 **Output FX 트랙에서는 여전히 플레이바가 헤더 위로 보이던 것**까지 함께 수정.
  - 원인: 각 레인의 플레이헤드 세로선이 `zIndex: 10`으로 sticky 헤더(`zIndex: 6`)보다 **위**에 있음. 레인은 `position: relative`지만 z-index가 auto라 **독자 stacking context를 만들지 않아**, 레인 안 요소(플레이헤드 10, Output FX의 loop 오버레이/핸들 4·5·6)가 형제인 헤더와 같은 컨텍스트에서 직접 경쟁 → 헤더보다 위로 그려짐. 플레이헤드 화면 x = `HEADER_W + phx − scrollLeft`이므로, **`phx < scrollLeft`(=플레이헤드가 뷰포트 왼쪽으로 스크롤 아웃)** 이면 화면상 위치가 sticky 헤더 구간(0..`HEADER_W`)에 들어오고, z-index가 높아 헤더를 덮음. seek-back 후 zoom-in이 바로 이 조건을 만듦. (플레이헤드 `zIndex:10`은 v1.14.15에 도입, 휠 줌 v1.15.0이 zoom-in을 쉽게 해 표면화.)
  - 수정: 개별 요소 z-index를 낮추는 대신 **각 레인 div에 `isolation: "isolate"`를 부여해 레인을 독자 stacking context로 만듦.** 이렇게 하면 헤더(양수 zIndex 6/9)가 레인(z auto) 전체보다 위에 그려지므로, 레인 안의 어떤 요소(플레이헤드·loop 오버레이·리사이즈 핸들)도 헤더를 넘어 그려질 수 없음. 레인 내부의 상대적 z-index 순서(플레이헤드가 loop 오버레이 위)는 그대로 유지 → 시각적 회귀 없음. 적용 대상: **일반 트랙 레인·Time Ruler 레인**([ui-tracks.jsx](ui-tracks.jsx)), **Output FX 레인**([ui-mixer.jsx](ui-mixer.jsx) `.outlane`).
  - 대상: [ui-tracks.jsx](ui-tracks.jsx)·[ui-mixer.jsx](ui-mixer.jsx). `build:renderers` 재빌드로 `build/ui-tracks.js`·`build/ui-mixer.js` 반영. 네이티브 재빌드 불필요.

### 이전 수정 및 반영 사항 (v1.15.1 → v1.15.2)
- **구간 반복(Repeat) ON 시 플레이바가 구간으로 진입하지 못하던 버그 수정** (버그 신고):
  - 증상: **재생 중** 플레이바보다 **뒤쪽(이후 시점)** 에 반복 구간을 (재)설정하면, 화면 플레이바는 구간으로 들어온 것처럼 보이지만 **실제 오디오는 구간으로 진입하지 않고** 선형 재생됨. (정지 상태, 구간이 플레이바보다 앞쪽인 경우는 정상.)
  - 원인: 반복 유지 로직이 **끝(end) 경계 하나로만** 설계됨 — `handleNativeMessage`가 `offset >= loopRange.end`일 때만 `seek(start)`로 되돌림. 그래서 플레이헤드가 **구간보다 앞(offset < start)** 인 상태(=구간을 뒤쪽에 설정)는 교정할 로직이 없음. 게다가 네이티브 `getPlayhead()`는 repeat ON일 때 반환값을 `[start,end]`로 **clamp** 하여 seek 없이도 화면 플레이바만 구간 안으로 들어온 듯 보이고 오디오는 밖에서 선형 재생됨(시각/오디오 괴리). 이 wrap 로직·반복 기능은 v1.14.15에 도입된 것으로 **마우스 휠 줌(v1.15.0)과 무관**함을 git 이력으로 확인.
  - **1차 (v1.15.1) — `setRepeatPlayEnabled` read-before-set**: 위 clamp 때문에 `setRepeatPlayEnabled`가 플래그를 켠 뒤 getPlayhead를 읽으면 seek 판정이 무력화됨. → **플래그를 켜기 전에** 참(clamp 전) 플레이헤드를 읽어 구간 밖이면 `seek(loopRange.start)`. native/web 분기를 `this.getPlayhead()`/`this.seek()`로 통합, **정지 상태에서도 스냅**되도록 `isPlaying` 게이트 제거. [audio-engine.js](audio-engine.js) 엔진 메서드도 동일 적용. → **Repeat 토글 순간**을 커버.
  - **2차 (v1.15.2) — 구간 드래그 종료 시 1회 스냅**: repeat가 **이미 켜진 상태에서 구간을 드래그/이동(`setLoopRange`)** 하는 경로는 토글이 아니므로 1차로는 안 잡힘. (앞서 `handleNativeMessage` wrap에 `offset < start`를 넣어 상시 교정하려던 시도는, 드래그 중 매 위치 프레임마다 seek가 발동해 SoundTouch 그레인이 반복 재시작되며 **'징~~~' 잡음·재생 지연**을 유발 → 원복.) → 대신 브리지에 `snapPlayheadToLoop()` 신설: **드래그 종료(mouseup) 시 딱 한 번**, **raw(비-clamp) 위치**로 구간 밖인지 판정해 밖이면 `seek(loopRange.start)`. 일반 사용자 seek와 동일한 1회 seek라 잡음 없음. [ui-mixer.jsx](ui-mixer.jsx) 루프 드래그 `up` 핸들러에서 `moved && repeatPlayEnabled`일 때 호출.
  - 대상: [audio-bridge.js](audio-bridge.js)(`setRepeatPlayEnabled`, `snapPlayheadToLoop`), [audio-engine.js](audio-engine.js)(엔진 메서드), [ui-mixer.jsx](ui-mixer.jsx)(드래그 종료 훅). ui-mixer는 `build/`로 컴파일되므로 `npm run build:renderers` 재빌드 반영. 네이티브 재빌드는 불필요.

## 4. 보류 항목

- **클립 편집 UI(1-D)**: 현재 미사용 방침. 엔진 코드는 유지하되 화면 노출은 보류.
- **변동 BPM / Tempo Map 분석**: 현재 전체 곡 비례 스케일링 구조에서는 필수 아님. 비트 그리드/워프 편집을 도입할 때 재검토.
- **AudioWorklet 기반 실시간 DSP 재시도**: 현재 네이티브 JUCE 엔진과 WebAudio fallback이 동작하므로 즉시 착수하지 않음.
## 2026-07-09 - Phase 4 단일 Audio In 녹음 1차 구현

- Settings에 `Device Setup` 섹션과 목차 항목을 추가했다. 네이티브 엔진이 열거한 입력 장치, Mono/Stereo 입력 채널, 버퍼 크기를 앱 설정으로 저장하고 재연결 시 복원한다.

- JUCE `AudioDeviceManager`를 입출력 동시 구성으로 확장하고 `setAudioInput`, `startRecording`, `stopRecording`, `cancelRecording` WebSocket 명령을 추가했다.

- 프로젝트가 저장된 경우 `<Project> Audio/Recordings`, 저장 전에는 소스 인접 `Recordings` 또는 앱 임시 폴더를 사용한다.
- JS 렌더러 빌드와 Node 정적 검사는 통과했다. 현재 머신의 Visual Studio C++ targets가 설치되어 있지 않아 네이티브 바이너리 재빌드와 실입력 장치 검증은 별도 환경에서 필요하다.

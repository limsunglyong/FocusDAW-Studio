### 최근 수정 및 반영 사항 (v1.15.1)
- **구간 반복(Repeat) ON 시 플레이바가 구간으로 진입하지 못하던 버그 수정** (버그 신고):
  - 증상: **재생 중** 플레이바보다 **뒤쪽(이후 시점)** 에 반복 구간을 만들고 Repeat를 켜면, 구간만 그려지고 플레이바가 구간 시작점으로 이동하지 않고 그대로 선형 재생됨. (정지 상태, 그리고 구간이 플레이바보다 앞쪽인 경우는 정상.)
  - 원인: **네이티브 엔진 전용 버그**. `getPlayhead()`는 `repeatPlayEnabled === true`일 때 반환값을 루프 구간 `[start, end]` 안으로 clamp/wrap 함([audio-bridge.js](audio-bridge.js)). 그런데 `setRepeatPlayEnabled`가 **`repeatPlayEnabled = true`를 먼저 켠 뒤** `getPlayhead()`를 읽어, 실제 위치가 루프보다 앞(raw < start)이어도 clamp되어 `start`로 보고됨 → `ph < start` 판정이 false가 되어 `seek(start)`가 실행되지 않음. 웹 엔진 `getPlayhead()`에는 loopRange clamp가 없어 웹 모드에선 미발생.
  - 수정: `setRepeatPlayEnabled`에서 **플래그를 켜기 전에** 참(clamp 전) 플레이헤드를 먼저 읽고, 그 값으로 구간 밖(`ph < start || ph > end`) 여부를 판정해 `seek(loopRange.start)` 실행. native/web 분기를 `this.getPlayhead()`/`this.seek()`(내부에서 엔진 구분)로 통합. 또한 **정지 상태에서도 스냅**되도록 `isPlaying` 게이트 제거(사용자 요청) — Repeat ON 시 플레이헤드가 구간 밖이면 항상 구간 시작점으로 이동(앞/뒤 양방향).
  - 대상: [audio-bridge.js](audio-bridge.js) `setRepeatPlayEnabled`, 일관성 위해 [audio-engine.js](audio-engine.js) 엔진 메서드도 동일 적용. 두 파일은 루트에서 직접 로드되어 `build/` 재빌드 불필요.
  - ⚠️ 후속 시도(v1.15.2, `handleNativeMessage` wrap에 `offset < start` 추가)는 재생 중 구간 변경 시 **seek 폭주로 '징~~~' 잡음·재생 지연**이 발생하여 **원복**함. 재생 중 구간을 뒤쪽으로 이동하는 경우의 오디오 추종은 별도 접근 필요(seek 폭주 없는 방식)로 미해결 과제로 남김.

### 이전 수정 및 반영 사항 (v1.15.0)
- **타임라인 zoom/스크롤 마우스 휠 단축키 추가** (수정요청 #1):
  - **Ctrl(+Cmd)+휠**: 타임라인을 **커서 위치를 중심으로** zoom in/out. 커서 아래 시각(time)이 확대/축소 후에도 화면상 같은 위치에 고정되도록 `scrollLeft`를 재조정.
  - **Shift+휠**: 타임라인을 좌우로 이동(가로 스크롤).
  - 수식어 없는 휠은 기존 세로 스크롤을 그대로 유지. Ctrl/Shift일 때만 `preventDefault`로 브라우저 페이지 줌/기본 동작을 차단.
  - 구현 위치: [app.jsx](app.jsx) `Studio` 컴포넌트의 arrange 스크롤 영역(`arrangeRef`). React 17 휠 이벤트가 root에서 passive로 위임되어 `onWheel` prop의 `preventDefault`가 무시되는 문제 때문에, 미니맵과 동일하게 `addEventListener("wheel", …, {passive:false})`로 직접 등록.
  - 커서 고정 정확도 확보: 확대 시 lane 폭은 리렌더 후에 바뀌므로, 목표 `scrollLeft`를 `pendingZoomScrollRef`에 저장했다가 `pxPerSec` 변경 시 `useLayoutEffect`에서 적용.
  - zoom 범위는 기존 슬라이더/미니맵과 동일하게 `[timelineMinPx, TIME_ZOOM_MAX]`로 클램프하고, 최소치 근처에서는 `fitTimelineRef`를 갱신해 리사이즈 시 fit 동작과 일관성 유지.

### 이전 수정 및 반영 사항 (v1.14.15)
- **구간 반복(Repeat Loop) 기능 추가 및 UI 개선**:
  - Output FX 트랙 영역 드래그로 반복 구간 설정, 시작/끝 라인 조절 핸들, 삭제 `[X]` 버튼 구현.
  - Repeat 활성화 시 재생 중 루프 안으로의 스냅 및 구간 내 반복 재생 지원.
  - 반복 구간 표시를 전체 트랙을 세로로 가로지르도록 연장하되 Output FX 트랙 하단 경계선에 정렬 및 짙은 amber 톤 시인성 보완.
- **Electron 네이티브 오디오 엔진 브릿징**:
  - `audio-bridge.js` 내 `setLoopRange(range)`, `setRepeatPlayEnabled(on)` 추가 및 C++ 엔진 연동.
  - 재생 시작 시 스냅, 정지 시 시작점 이동, 플레이헤드 wrapping 계산 반영.
  - 30ms 주기적 감시(`startLocalTickLoop`) 내 루프 감지 시 seek 명령 전달 및 seek 후 500ms 동안 수신되는 stale position 프레임 무시(`lastSeekSentAt` 타임스탬프)로 화면 떨림 해결.
- **Time Ruler & 타임라인 미니맵 연동**:

  - Time Ruler 상단 좌우 끝에 `clipPath` 기반의 sticky 삼각형 마크 추가.
  - playhead 컴포넌트들의 `zIndex`를 `10`으로 올려 레이아웃 정렬.
  - 타임라인 미니맵(`TimelineMinimap`)에 반복 구간 및 활성 상태 반영 음영 렌더링.

## 2. 릴리즈 전 남은 작업

   - Windows 설치본/포터블 빌드가 필요하면 `npm run dist:win` 실행.

2. **최종 스모크 테스트**
   - 앱 실행/종료, 프로젝트 신규 생성, 오디오 파일 import, 최근 프로젝트 복원.
   - 재생/정지/seek/repeat, Solo/Mute, 트랙 볼륨/팬, Mixer/Advanced Pan/Advanced Effects 창 열기.
   - Vari BPM, Vari Key, Fade in/out, Volume Automation이 재생과 Export에서 모두 반영되는지 확인.
   - MP3/WAV Export 완료 및 결과 파일 재생 확인.

3. **시험 문서 확인**
   - 남은 수동 시험 항목이 있으면 [시험.md](시험.md)에만 유지.
   - 완료된 시험은 [시험-아카이브.md](시험-아카이브.md)로 이동.

## 4. 보류 항목

- **클립 편집 UI(1-D)**: 현재 미사용 방침. 엔진 코드는 유지하되 화면 노출은 보류.
- **변동 BPM / Tempo Map 분석**: 현재 전체 곡 비례 스케일링 구조에서는 필수 아님. 비트 그리드/워프 편집을 도입할 때 재검토.
- **AudioWorklet 기반 실시간 DSP 재시도**: 현재 네이티브 JUCE 엔진과 WebAudio fallback이 동작하므로 즉시 착수하지 않음.

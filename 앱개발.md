### 최근 수정 및 반영 사항 (v1.15.3)
- **Repeat ON 중 반복구간 '밖' 클릭(seek) 동작 정리** (버그 신고):
  - 증상: repeat ON 재생 중 구간 밖을 클릭하면 (A) 구간 이전 클릭은 구간 밖에서 재생, (B) 구간 이후 클릭은 무조건 구간 시작으로 튕겨 **동작이 비일관**. (원인: `handleNativeMessage`의 끝-경계 wrap `offset >= end`가 사용자의 명시적 클릭까지 가로챔.)
  - 수정: 반복구간을 "감옥"으로 취급 — **repeat ON일 때 구간 밖을 겨냥한 타임라인 클릭 seek는 무시(no-op)** 하고, 구간 안 클릭만 정상 seek. 브리지에 `userSeek(t)` 신설([audio-bridge.js](audio-bridge.js)): `repeatPlayEnabled && loopRange && (t < start || t > end)`이면 반환. [app.jsx](app.jsx)의 타임라인 클릭 seek 3곳(Ruler·TrackRow·OutputTrack)을 `DAW.seek`→`DAW.userSeek`로 교체. (Return-to-start·화살표 nudge 등 명시적 트랜스포트 이동은 `seek()` 유지로 제한 없음.)
- **마우스 드래그로 오브젝트 선택/드래그 & 창 테두리 굵은 점선 방지** (버그 신고):
  - 증상: 앱 내부에서 텍스트/요소가 브라우저 기본 **HTML5 드래그**로 마우스에 붙고(고스트), arrange 영역 위를 지나며 **파일 임포트 드롭존 점선 아웃라인**([app.jsx](app.jsx) `dragOver`)이 화면 가장자리를 따라 굵게 그려짐.
  - 수정: ① [studio.html](studio.html)에 전역 `dragstart` `preventDefault`(capture) 추가 — 내부 HTML5 드래그 원천 차단(OS→앱 파일 드롭은 앱 내부 dragstart를 발생시키지 않아 임포트 정상). ② 전역 CSS `user-select:none`(input/textarea/select/contenteditable 예외)·`img/a/svg { -webkit-user-drag:none }`로 선택/고스트 방지. ③ 방어적으로 `onDragOver`/드롭존 아웃라인을 `dataTransfer.types`에 `Files`가 있을 때만 활성(`isFileDrag`).
  - 대상: [audio-bridge.js](audio-bridge.js)·[app.jsx](app.jsx)·[studio.html](studio.html). app.jsx는 `build:renderers` 재빌드 반영, studio.html·audio-bridge.js는 루트 직접 로드. 네이티브 재빌드 불필요.

### 이전 수정 및 반영 사항 (v1.15.1 → v1.15.2)
- **구간 반복(Repeat) ON 시 플레이바가 구간으로 진입하지 못하던 버그 수정** (버그 신고):
  - 증상: **재생 중** 플레이바보다 **뒤쪽(이후 시점)** 에 반복 구간을 (재)설정하면, 화면 플레이바는 구간으로 들어온 것처럼 보이지만 **실제 오디오는 구간으로 진입하지 않고** 선형 재생됨. (정지 상태, 구간이 플레이바보다 앞쪽인 경우는 정상.)
  - 원인: 반복 유지 로직이 **끝(end) 경계 하나로만** 설계됨 — `handleNativeMessage`가 `offset >= loopRange.end`일 때만 `seek(start)`로 되돌림. 그래서 플레이헤드가 **구간보다 앞(offset < start)** 인 상태(=구간을 뒤쪽에 설정)는 교정할 로직이 없음. 게다가 네이티브 `getPlayhead()`는 repeat ON일 때 반환값을 `[start,end]`로 **clamp** 하여 seek 없이도 화면 플레이바만 구간 안으로 들어온 듯 보이고 오디오는 밖에서 선형 재생됨(시각/오디오 괴리). 이 wrap 로직·반복 기능은 v1.14.15에 도입된 것으로 **마우스 휠 줌(v1.15.0)과 무관**함을 git 이력으로 확인.
  - **1차 (v1.15.1) — `setRepeatPlayEnabled` read-before-set**: 위 clamp 때문에 `setRepeatPlayEnabled`가 플래그를 켠 뒤 getPlayhead를 읽으면 seek 판정이 무력화됨. → **플래그를 켜기 전에** 참(clamp 전) 플레이헤드를 읽어 구간 밖이면 `seek(loopRange.start)`. native/web 분기를 `this.getPlayhead()`/`this.seek()`로 통합, **정지 상태에서도 스냅**되도록 `isPlaying` 게이트 제거. [audio-engine.js](audio-engine.js) 엔진 메서드도 동일 적용. → **Repeat 토글 순간**을 커버.
  - **2차 (v1.15.2) — 구간 드래그 종료 시 1회 스냅**: repeat가 **이미 켜진 상태에서 구간을 드래그/이동(`setLoopRange`)** 하는 경로는 토글이 아니므로 1차로는 안 잡힘. (앞서 `handleNativeMessage` wrap에 `offset < start`를 넣어 상시 교정하려던 시도는, 드래그 중 매 위치 프레임마다 seek가 발동해 SoundTouch 그레인이 반복 재시작되며 **'징~~~' 잡음·재생 지연**을 유발 → 원복.) → 대신 브리지에 `snapPlayheadToLoop()` 신설: **드래그 종료(mouseup) 시 딱 한 번**, **raw(비-clamp) 위치**로 구간 밖인지 판정해 밖이면 `seek(loopRange.start)`. 일반 사용자 seek와 동일한 1회 seek라 잡음 없음. [ui-mixer.jsx](ui-mixer.jsx) 루프 드래그 `up` 핸들러에서 `moved && repeatPlayEnabled`일 때 호출.
  - 대상: [audio-bridge.js](audio-bridge.js)(`setRepeatPlayEnabled`, `snapPlayheadToLoop`), [audio-engine.js](audio-engine.js)(엔진 메서드), [ui-mixer.jsx](ui-mixer.jsx)(드래그 종료 훅). ui-mixer는 `build/`로 컴파일되므로 `npm run build:renderers` 재빌드 반영. 네이티브 재빌드는 불필요.

### 이전 수정 및 반영 사항 (v1.15.0)
- **타임라인 zoom/스크롤 마우스 휠 단축키 추가** (수정요청 #1):
  - **Ctrl(+Cmd)+휠**: 타임라인을 **커서 위치를 중심으로** zoom in/out. 커서 아래 시각(time)이 확대/축소 후에도 화면상 같은 위치에 고정되도록 `scrollLeft`를 재조정.
  - **Shift+휠**: 타임라인을 좌우로 이동(가로 스크롤).
  - 수식어 없는 휠은 기존 세로 스크롤을 그대로 유지. Ctrl/Shift일 때만 `preventDefault`로 브라우저 페이지 줌/기본 동작을 차단.
  - 구현 위치: [app.jsx](app.jsx) `Studio` 컴포넌트의 arrange 스크롤 영역(`arrangeRef`). React 17 휠 이벤트가 root에서 passive로 위임되어 `onWheel` prop의 `preventDefault`가 무시되는 문제 때문에, 미니맵과 동일하게 `addEventListener("wheel", …, {passive:false})`로 직접 등록.
  - 커서 고정 정확도 확보: 확대 시 lane 폭은 리렌더 후에 바뀌므로, 목표 `scrollLeft`를 `pendingZoomScrollRef`에 저장했다가 `pxPerSec` 변경 시 `useLayoutEffect`에서 적용.
  - zoom 범위는 기존 슬라이더/미니맵과 동일하게 `[timelineMinPx, TIME_ZOOM_MAX]`로 클램프하고, 최소치 근처에서는 `fitTimelineRef`를 갱신해 리사이즈 시 fit 동작과 일관성 유지.

## 4. 보류 항목

- **클립 편집 UI(1-D)**: 현재 미사용 방침. 엔진 코드는 유지하되 화면 노출은 보류.
- **변동 BPM / Tempo Map 분석**: 현재 전체 곡 비례 스케일링 구조에서는 필수 아님. 비트 그리드/워프 편집을 도입할 때 재검토.
- **AudioWorklet 기반 실시간 DSP 재시도**: 현재 네이티브 JUCE 엔진과 WebAudio fallback이 동작하므로 즉시 착수하지 않음.

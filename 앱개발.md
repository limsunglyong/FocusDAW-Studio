# FocusDAW — Stem Studio · 앱 개발 문서

> 웹브라우저 기반(Electron) Stem 믹싱 DAW
> 하나의 곡에서 분리된 Stem을 불러와 구간별로 볼륨·이펙트를 적용하고, 하나의 MP3로 최종 출력하는 경량 DAW.

---

## 2. 구현 목표

### 2.1 핵심 컨셉
- **하나의 곡을 분리한 Stem 전체를 불러와**, 구간별로 볼륨·리버브·에코·필터 등 간단한 효과를 적용하고, 최종적으로 **하나의 MP3 파일을 생성**한다.
- 웹브라우저에서 동작하며, Electron 기반으로 **독립형 데스크톱 앱**으로도 사용 가능해야 한다.

### 2.2 기능 목표

**A. Stem 불러오기**
- 프로젝트 관리용 파일(`.focus`) 생성
- Stem(`*.mp3`, `*.wav`) 파일 일괄 등록 (프로젝트 폴더 선택 방식)
- 개별 Track 추가 및 Audio file 불러오기
- 각 Stem 파일 이름으로 Track 자동 생성

**B. Track**
- 트랙별 Solo / Mute
- Time scale(가로) 확대 보기
- Y축(파형 진폭) 확대 보기
- 이펙트(볼륨·리버브·에코)를 **구간별**로 적용하기 위한 이펙트 구간 편집 — 가위 툴로 구간을 자르고, 다시 잇는 툴 / 구간별 Automation
- Volume Automation — 파형 위에 오버레이(On/Off)

**C. Mixer 화면**
- Mixer 버튼 → 별도 창으로 표시 (브라우저 창 간 정보 공유)

**D. Output Effect Track**
- 전체 곡의 Equalizer 효과
- Fade In / Fade Out 효과
- 시간대별 적용 효과를 타임라인에 간단히 오버레이

## 3. 실제 구현 내용

### 3.1 앱 셸 / 프레임
- Electron 데스크톱 윈도우 크롬: 최상단 메뉴바(Project·Settings) + 중앙 Compact Transport(처음/정지/재생/루프 + 시간 표시) + 우측 프로젝트명.
- 화면 라우팅: `최근 프로젝트 복원 또는 빈 스튜디오(Empty State) → 스튜디오(어레인지)`.

### 3.2 Stem 불러오기 (로더 화면) ✅
- 프로젝트명 입력, **폴더 선택 / 파일 드래그앤드롭** 영역.
- 드롭 시 `.wav/.mp3` 등 오디오만 필터링하여 **일괄 등록**, 파일명 → 트랙명 자동 생성. 브라우저 `Import Folder`는 선택한 폴더의 루트에 있는 오디오만 가져오고 하위 디렉토리는 제외.
- 오디오 파일 디코딩/트랙 추가 중에는 현재 테마 색상에 맞는 로딩 오버레이 표시. 파일명, 진행률 바, 처리 개수 표시로 대용량 Stem import 체감 대기 시간을 완화.
- "Detected stems" 목록에서 등록 여부 토글(체크), 길이 표시.
- `.focus` 프로젝트 파일 생성 컨셉 명시, 데모 세션 4 Stem 선등록.
- `DAW.addFile()`이 `decodeAudioData`로 실제 디코딩 → 즉시 파형/재생 반영.

### 3.3 Track (어레인지 뷰) ✅
- **실제 파형 렌더링**: 디코드된 버퍼에서 peak(min/max) 추출 후 Canvas 드로잉(트랙 색상). 고줌/긴 오디오에서 전체 길이 Canvas를 만들지 않고 현재 스크롤 뷰포트 주변만 렌더링해 Chrome Canvas 크기 한계로 인한 흰 화면 깨짐을 방지.
- **Solo / Mute**: 엔진의 게인 라우팅에 반영(any-solo 시 비솔로 트랙 음소거).
- **인라인 볼륨 페이더 + 팬 노브 + VU 미터**(트랙 헤더, Medium 밀도).
- **Time scale 확대**: `pxPerSec` 28~420 단계 줌 — 줄로아웃/인 버튼 + **세련된 슬라이더**(앤버 글로우 텀 · 틱 눈금)로 연속 조절.
- **Y축 진폭 확대**: `ampZoom` 0.4~3.0 배율 — 동일한 슬라이더 + 버튼.
- **룰러(시간 표시)**: 상단 룰러를 **시간 스케일(m:ss)** 로 표시. 오디오 파일 로드 시 `DAW.duration` 기준으로 전체 길이까지 확장되며, 줌 배율에 따라 눈금 간격(0.25s~30m)을 자동 선택하고 보조 눈금 표시. 헤더 라벨 `TIME · m:ss`.
- **TIME 줌 전체 보기 보정**: `TIME` 슬라이더 최소값을 고정 px/sec가 아닌 `보이는 레인 폭 / 전체 오디오 길이` 기준으로 동적 계산. 오디오 파일/프로젝트를 불러오면 최소 줌에서 전체 waveform이 한 화면에 들어오도록 자동 맞춤.
- **트랜스포트 중앙 배치**: 컨트롤바를 3분할 — 좌측(시간/진폭 줌 + 행 높이), **중앙(트랜스포트 + 시간 표시, 화면 정중앙 고정)**, 우측(Track·Mixer·Export).
- 클릭-투-시크, 루프 트랜스포트, 단축키(Space = 재생/정지), 행 높이(S/M/L) 토글.
- **최근 프로젝트 자동 복원**: 앱 시작 시 `localStorage`의 최근 프로젝트 JSON을 자동 복원. 최근 프로젝트 정보가 없으면 `Project → New Project`와 동일한 빈 스튜디오/Empty State로 시작.

### 3.4 Volume Automation ✅
- 파형 위 SVG 오버레이로 **브레이크포인트 곡선** 표시. 트랙 헤더에서 **VOL AUTO** 라운드 버튼으로 On/Off(M·L), L 사이즈에선 그 아래 **Reset**(확인 모달 후 평탄 초기화)·**Curve**(curve fitting On/Off) 버튼 제공.
- **편집**: 포인트 드래그 / 라인 클릭으로 포인트 추가 / 우클릭으로 삭제.
- **curve fitting(단조 큐빅)**: `Curve` On 시 점들을 piecewise cubic Hermite(Fritsch–Carlson, 오버슈트 없음)로 부드럽게 보간. 비파괴(편집 점만 저장, 곡선은 실시간 파생) — 오버레이는 **편집 직선(점선)+적용 곡선(실선)** 을 오디오와 같은 샘플러로 동시 표시(화면=소리).
- **실제 청취 가능**: 재생 시 `setValueCurveAtTime`으로 게인 곡선을 루프마다 스케줄링하여 음량이 곡선대로 변함. 오프라인 렌더에도 동일 적용. seek/편집 직후에도 재생 위치와 1:1 정렬(현재 루프 곡선 슬라이스, ISSUE-1 해결).

### 3.5 Mixer 창 ✅
- Mixer 버튼 → **드래그 가능한 별도 OS 스타일 창**으로 표시(자체 타이틀바·트래픽 라이트).
- 트랙별 **채널 스트립**: 리버브·에코 노브, Solo/Mute, 팬, 세로 페이더 + VU 미터, dB 표기.
- **Master 스트립**: 9밴드 그래픽 EQ(드래그) + FFT 스펙트럼 + 마스터 페이더 + 스테레오 미터, 추천 EQ 프리셋 버튼(Reset·POP·Classic·HIP HOP).
- 어레인지 뷰와 **상태 실시간 동기화**(동일 엔진 상태 참조).

### 3.6 Output Effect Track ✅
- 어레인지 하단 전용 트랙: **마스터 EQ**(Low/Mid/High 노브 + 수치 오버레이).
- **Fade In / Fade Out** 핸들을 타임라인에서 드래그 편집, 램프 영역을 그래픽 오버레이.
- 실제 마스터 페이드 게인 스케줄링 반영.

### 3.7 Export to MP3 ✅
- 익스포트 다이얼로그: 파일명·**포맷(MP3/WAV)**·비트레이트·샘플레이트·노멀라이즈 옵션.
- **오프라인 렌더링**: `OfflineAudioContext`로 전체 믹스(트랙 FX + 오토메이션 + 마스터 EQ + 페이드 + 리미터)를 실제 렌더 → 진행률 바 → 파일 저장.
- 렌더 결과 피크 ≈ 1.06(마스터 리미터 적용)으로 클리핑 억제.
- **Electron 경로**: `ffmpeg-static`(npm 패키지, 플랫폼별 바이너리 내장) → 임시 WAV 파일 → ffmpeg 프로세스 → MP3 바이너리 반환. `asarUnpack`으로 바이너리 접근 보장.
- **브라우저 fallback**: `lamejs`(CDN) — Float32 PCM → Int16 변환 → 1152샘플 블록 단위 인코딩, 64블록마다 이벤트 루프 양보로 UI 반응성 유지.
- 단계별 `stepLabel` 표시: "Rendering mix…" → "Encoding MP3…" / "Encoding WAV…" 진행 단계 구분.

### 3.8 Settings / 테마 선택 ✅
- **Settings 메뉴** — 메뉴바에 "Settings" 항목 추가. 클릭 시 모달 다이얼로그 표시.
- **3종 컬러 테마** 지원:
  - `Warm Analog` — 에스프레소 배경 + 앰버 강조 (기본, 빈티지 콘솔)
  - `Classical Ivory` — 아이보리/페이퍼 배경 + 딥 앰버 (밝고 클래식)
  - `Modern Blue` — 딥 네이비 배경 + 블루 강조 (모던 프로)
- **테마 카드 UI** — 각 테마의 미니 스튜디오 프리뷰(메뉴바·트랙 3개·컬러 팔레트 바) + 이름·설명 표시. 선택된 테마는 해당 테마 강조색 테두리.
- **전환 방식**: `document.documentElement.dataset.theme` 속성 교체 → `studio.html`의 `:root[data-theme]` CSS 오버라이드 블록 활성화. 즉시 반영.
- **영속성**: `localStorage.setItem('focusdaw-theme', id)` 저장 → 재시작 후에도 테마 유지.

### 3.9 오디오 엔진 (Web Audio) ✅
- 데모 Stem **절차적 합성**(킥/스네어/하이햇/베이스/패드/리드, Am–F–C–G 진행)으로 파일 없이도 즉시 재생·파형 표시.
- 트랙 그래프: `source → fader → autoGain → panner → (dry + reverb send[Convolver] + echo[Delay+FB]) → masterBus`.
- 마스터 체인: `masterBus → EQ(Low/Mid/High) → fade → volume → Compressor(리미터) → analyser → destination`.
- 트랜스포트(재생/일시정지/정지/시크), 루프, 트랙·마스터 레벨 미터링.

## 4. 개발 방향

### 4.1 디자인 방향
- **Warm Analog(빈티지 콘솔)**: 깊은 에스프레소/차콜 배경, 크림 텍스트, **앰버** 강조색, **그린→앰버→레드** VU 미터.
- 개성 있는 지오메트릭 산세리프(Space Grotesk) + 모든 시간·수치 표기에 모노스페이스(Space Mono).
- AI 슬롭 회피: 과한 그라데이션·이모지·둥근 카드 남용 지양, 콘솔 메타포에 충실.

### 4.2 아키텍처 방향
- **오디오 로직과 UI 분리**: `audio-engine.js`(순수 JS, 단일 상태 소스) ↔ React UI(엔진 상태 구독·렌더).
- **무빌드 단일 페이지**: Babel 인라인 JSX, 파일 분할 후 `window`로 공유.
- **실제 동작 우선**: 목업이 아닌 Web Audio 실제 재생/이펙트/렌더 구현.
- **변형은 토글로**: 요소 변형안은 별도 비교 페이지로 제시 → 확정 시 본 앱에 통합.

### 4.3 Electron 전환 방향
- 현재 웹 기능을 Electron 셸에 그대로 탑재.
- 폴더 일괄 등록 → `dialog.showOpenDialog({ properties:['openDirectory'] })` + `fs`로 폴더 스캔.
- 믹서 별도 창 → 실제 `BrowserWindow` + `BroadcastChannel`/IPC로 상태 동기화(현재는 인앱 플로팅 창으로 대체).
- MP3 인코딩 → 네이티브 `ffmpeg`(또는 lamejs) 파이프라인으로 처리.

## 5. 개발 예상 이슈 및 해결 방안

| # | 이슈 | 영향 | 해결 방안 | 현재 상태 |

| 2 | **마스터 클리핑** — 트랙 합산 시 피크 > 1.0 | 출력 왜곡 | 마스터 `DynamicsCompressor` 리미터 삽입(라이브·오프라인 동일) | ✅ 적용(피크 ≈1.06) |

### BUG-6. 재생 중 Repeat 전환 시 UI play bar와 실제 오디오 상태 불일치
> 상태: **✅ 해결됨 (v0.16.21, 2026-06-11)**

- **증상 1**: Repeat OFF 상태로 재생 후 곡 끝 5~10초 전으로 이동하고 Repeat ON을 누르면 play bar는 움직이지만 실제 소리가 나지 않음.
- **증상 2**: Repeat ON 상태로 재생 후 곡 끝에서 처음으로 반복된 뒤 Repeat OFF를 누르면 현재 회차를 끝까지 재생하지 않고 즉시 stop됨.
- **원인**: `setLoop()`가 `loopEnabled` 값만 바꾸고, 이미 재생 중인 `AudioBufferSourceNode`의 loop 상태와 엔진의 종료 판단 기준이 서로 어긋남.
- **수정 계획**:
  1. 재생 중 Repeat 버튼을 눌러도 현재 재생 중인 `AudioBufferSourceNode`를 직접 재생성하지 않는다.
  2. `AudioBufferSourceNode.loop`에 반복을 맡기지 않고, 모든 소스는 1회 재생으로 시작한다.
  3. 곡 끝 도달 시점에 엔진이 `loopEnabled`를 확인하여 ON이면 `_offset = 0`으로 새 회차를 시작하고, OFF이면 `stop()`한다.
  4. Repeat OFF 전환은 즉시 정지가 아니라 현재 회차 종료 후 정지로 처리하여 버튼 클릭 순간의 오디오 끊김을 피한다.
- **수정 내용**:
  1. `play()`에서 `src.loop = false`로 고정하여 Web Audio native loop 상태와 UI 상태가 어긋나지 않도록 변경.
  2. `_loop()`가 매 프레임 raw playhead를 확인하고, `duration` 도달 시 최신 `loopEnabled` 값에 따라 새 회차 시작 또는 정지를 결정하도록 변경.
  3. 엔진 주도 반복 재시작 시 `skipFade` 옵션으로 마스터 fade-in/out을 반복 적용하지 않도록 처리.

### BUG-3. 줌 최대 → 줄일 때 minimap/TIME 슬라이더 sync 어긋남
> 상태: **✅ 해결됨 (v0.16.8-r2)**

### BUG-4. TimelineMinimap onWheel passive event 오류 (B-16)
> 상태: **✅ 해결됨 (v0.16.8)**

### ISSUE-1. Volume Automation 재생 sync 불일치 — 화면(곡선)과 실제 들리는 게인이 다름

> 상태: **✅ 해결됨 (v0.15.35)** — `_scheduleAutomation`의 현재 루프 곡선 압축을 `sliceCurveFrom`으로 부분 곡선 1:1 스케줄하도록 수정. (2026-06-08 분석 → 동일자 수정)

**증상**
- 현재 재생 중이 아닌 (무관한) 위치의 automation 포인트를 추가/상향하면, 갑자기 **현재 재생 중인 소리의 음량이 튄다**.
- 플레이바를 임의 위치로 클릭해 옮기면 그 지점의 automation 값이 아니라 **루프 앞부분(이전) 값처럼 들린다**. 재생 위치와 automation 적용 위치의 sync 문제로 추정.
- 종합하면 **파형 위 automation 오버레이 곡선(시각)** 과 **실제 적용되는 게인(청각)** 이 어긋난다.

**근본 원인 — 현재(부분) 루프의 automation 곡선이 "압축" 스케줄됨**
- 트랙 오디오 체인: `source → fader → autoGain → filter → panner` (`audio-engine.js` `_addTrack`). automation은 `autoGain.gain`에 `setValueCurveAtTime`으로 적용.
- `_scheduleAutomation()` (`audio-engine.js`, 현재 루프 처리부)에서 실제 호출이 다음과 같음:
  ```js
  const pos  = (this._offset + (now - this._startTime)) % dur; // 현재 재생 위치(루프 내)
  // l=0:
  g.setValueCurveAtTime(curve, now + 0.001, dur - pos);
  ```
- `curve`는 **루프 전체(위치 0 → dur)** 를 256포인트로 표현한 배열(`automationCurve`: `curve[0]`=위치 0 값). 그런데 `setValueCurveAtTime(curve, now, dur - pos)`는 그 **전체 곡선을 남은 시간 `(dur - pos)` 에 통째로 압축**해 넣는다.
- 결과:
  1. `now` 시점 게인이 `curve[0]`(루프 시작 위치 값)으로 **점프** — 실제 위치 `pos`의 값이 아님.
  2. 곡선이 오디오보다 **빠르게 흘러** 현재 루프 동안 automation과 오디오가 어긋남.
- 다음 루프부터(`l ≥ 1`)는 전체 dur에 정상 정렬되므로 **현재 부분 루프에서만** 어긋난다. (8초 루프라 seek/편집마다 매번 체감)

**증상-원인 매핑**
- 플레이바 이동: `seek()`가 `_offset`=클릭 위치로 두고 `play()` 재호출 → `pos>0` → 압축으로 루프 앞부분 값이 들림 ⇒ "이전 값" 체감.
- 무관한 지점 편집: `setTrackParam(autoOn|automation)` → `_scheduleAutomation()` 즉시 재호출 → `cancelScheduledValues(now)` 후 동일 압축 → 편집 지점과 무관하게 현재 게인이 `curve[0]`로 점프(그 값이 크면 음량 상승).

**수정 방향 (다음 단계, 승인 후)**
- 현재 루프는 **전체 곡선이 아니라 `pos` 이후를 잘라낸 부분 곡선**을 만들어 `(dur - pos)`에 1:1로 스케줄한다.
  - `startIdx = round(pos/dur * (n-1))`부터 `curve.slice(startIdx)` 사용(또는 `pos` 지점 값을 보간해 첫 샘플로 넣은 부분 배열 생성).
  - `g.setValueCurveAtTime(partial, now + 0.001, dur - pos)`.
- 실시간 경로(`_scheduleAutomation`)만 수정. 오프라인 렌더(`renderMix`)는 항상 처음부터 전체 dur로 스케줄하므로 영향 없음.
- 참고: 기존 [5. 개발 예상 이슈] 표의 #5("루프 경계 스케줄링 ✅")는 루프 경계 지터에 대한 것이며, 본 이슈(부분 루프 곡선 압축)는 별개의 미해결 건임.

### ISSUE-2. MP3 익스포트 시작 ~13ms 무음 — LAME 인코더 딜레이 (설계상 제약)

> 상태: **◐ 설계상 수용 (v0.16.17)** — WAV 익스포트는 딜레이 0ms. MP3 익스포트의 딜레이는 LAME 인코더 고유 특성으로 수용 결정.

**증상**
- Electron에서 MP3로 익스포트한 파일을 Chrome(`decodeAudioData`)으로 디코딩하면 오디오 시작 앞에 약 **13ms(576 샘플 @ 44100 Hz, 12ms @ 48000 Hz)의 무음**이 포함됨.
- WAV 익스포트 시에는 해당 현상 없음.

**원인**
- FFmpeg의 `libmp3lame`이 MP3 인코딩 시 알고리즘 특성상 고유하게 **576 샘플의 인코더 딜레이(encoder delay)**를 MP3 스트림 앞에 추가함.
- LAME은 이 딜레이 정보를 LAME 태그(Xing 헤더)에 기록하여 지원 플레이어가 보정할 수 있도록 함.
- 그러나 **Chromium의 `decodeAudioData`는 LAME 태그를 읽어 보정하지 않으므로**, Electron(Web Audio API 경유) 환경에서 576 샘플이 그대로 무음으로 표시됨.

**현황 및 수용 근거**
- MP3 코덱(LAME)의 알고리즘적 특성으로, 재인코딩 없이는 제거 불가.
- foobar2000, iTunes, mpg123 등 LAME 태그 지원 플레이어에서는 자동 보정됨.
- **WAV 익스포트를 사용하면 딜레이 0ms** — 딜레이가 문제가 되는 용도에서는 WAV 익스포트를 권장.

**렌더 트리밍 시 총 길이 보존 — WAV 길이가 1프레임 부족해 Windows에서 4초로 표시되던 문제 해결**

- **선행 무음 트리밍의 길이 단축 부작용 (audio-engine.js)**: v0.16.17에서 추가한 렌더 후 선행 무음 트리밍이 버퍼를 실제로 짧게 만들어, 시작이 0에서 출발하는 음원(예: 50 Hz 사인파)은 첫 1 프레임이 무음 임계값(1e-4) 미만으로 판정되어 잘려나갔습니다. 그 결과 5초 음원이 239999/240000 프레임 = 4.999979초가 되고, **Windows 탐색기가 길이를 정수 초로 내림(floor) 처리**하여 5초 파일이 "4초"로 표시되었습니다. (Audacity·실제 재생은 5초로 정상. MP3는 LAME 패딩으로 5.0초를 넘겨 5초로 표시되어 포맷 간 차이가 발생.)

- 관련 파일: `audio-engine.js`(`renderMix` 트리밍 길이 보존), `package.json`·`app.jsx`·`ui-dialogs.jsx`(버전 갱신).

- **WAV 익스포트 샘플레이트 고정 버그 (audio-engine.js / ui-dialogs.jsx)**: `renderMix()`가 `OfflineAudioContext`를 항상 시스템 장치 레이트(`ctx.sampleRate`)로 생성하여, 오디오 장치가 96 kHz로 동작하는 환경에서는 익스포트 다이얼로그에서 44100/48000을 선택해도 WAV가 96 kHz로 저장되는 문제가 있었습니다. ### v0.16.17
**익스포트 시작 딜레이 원인 확정 및 렌더 파이프라인 트리밍 적용**

- **B-35/B-38 원인 확정 — LAME 인코더 딜레이 (design-time 제약)**: WAV·MP3 A/B 시험 결과 WAV 익스포트는 딜레이 0ms, MP3 익스포트만 ~13ms 딜레이가 발생함을 확인했습니다. 원인은 FFmpeg의 libmp3lame이 MP3 인코딩 시 고유하게 추가하는 **576 샘플 인코더 딜레이(44100 Hz 기준 약 13ms)**이며, Chromium(Electron)의 `decodeAudioData`가 LAME 태그를 읽어 이를 보정하지 않기 때문입니다. foobar2000·iTunes 등 LAME 태그 지원 플레이어에서는 자동 보정됩니다. 이 딜레이는 MP3 포맷의 알고리즘적 특성으로, 코덱을 변경하지 않는 한 제거 불가하므로 현 동작을 **설계상 제약(design-time constraint)으로 수용**하기로 결정했습니다.
- **렌더 파이프라인 선행 무음 트리밍 추가 (audio-engine.js)**: 렌더 파이프라인 자체에서 발생할 수 있는 선행 무음(ConvolverNode·DelayNode 등의 render-quantum 정렬 부산물)을 제거하기 위해 `renderMix()`에서 `off.startRendering()` 반환 직후 -80 dBFS(1e-4) 미만의 연속 샘플을 트리밍합니다. 최대 50ms 범위로 제한하고, WAV 익스포트 시험에서 딜레이 0ms가 확인되어 렌더 파이프라인은 정상임이 검증됐습니다.
- 관련 파일: `audio-engine.js`(`renderMix` 렌더 후 트리밍 로직), `package.json`·`app.jsx`·`ui-dialogs.jsx`(버전 갱신).

### v0.16.16
**익스포트 시작 딜레이 완전 제거 (WaveShaperNode) + 임포트 시 duration 정확화**

- **B-35 완전 수정 — DynamicsCompressor → WaveShaperNode (audio-engine.js)**: `renderMix()` 내 Normalize 경로에서 `DynamicsCompressor`를 제거하고 `WaveShaperNode` 소프트 클리퍼(knee=0.9, tanh soft-clip)로 교체했습니다. Chromium의 `DynamicsCompressor`는 내부적으로 약 384 샘플(≈ 9ms @ 44100 Hz)의 look-ahead 지연을 OfflineAudioContext 출력에 그대로 반영해, v0.16.14에서 fadeIn 기본값을 0으로 낮추자 이 지연이 노출되었습니다. `WaveShaperNode`는 look-ahead 없이 샘플 단위로 즉각 처리되므로 0ms 시작 지점이 보장됩니다. 클리핑 특성은 |x| ≤ 0.9 구간 선형 통과 후 tanh 소프트 새추레이션으로, 원 DynamicsCompressor 대비 동적 압축 대신 피크 리미팅에 집중합니다.
- **문제2 수정 — 임포트 시 duration 과대 산정 방지 (audio-engine.js)**: `addFileBuffer()`·`addFile()` 에서 `this.duration = Math.max(this.duration, buffer.duration)` 사용 시 초기 기본값 `DURATION = 8`이 베이스라인이 되어, 5초짜리 오디오를 임포트해도 duration이 8초로 유지되고 OUTPUT FX 트랙 및 익스포트 결과물도 8초가 되는 문제를 수정했습니다. 데모 트랙이 없는 경우에는 `DURATION` 상수 대신 현재 로드된 실제 오디오 트랙들의 최대 duration을 베이스라인으로 사용합니다. 데모 트랙이 한 개라도 존재하면 기존 `Math.max(this.duration, ...)` 동작을 유지해 8초 루프 길이를 보존합니다.

- **요청 사항**: 한 트랙에서 Solo와 Mute가 동시에 켜지는 문제 해결. (1) Solo를 켜면 해당 트랙 Mute가 꺼지고, Mute를 켜면 Solo가 꺼져야 함. (2) Solo 트랙이 하나라도 있으면 나머지 트랙은 모두 mute로 표시되어야 함. (3) Solo/Mute가 모두 꺼지면 정상 재생.

- **auto-mute 표시 (ui-kit.jsx)**: `MuteBtn`에 `auto` prop 추가. 다른 트랙이 solo 중이라 암묵적으로 음소거된 상태를 하드 mute(빨강 채움)와 구분되는 dim 빨강으로 표시. 클릭 시에는 해당 트랙의 실제 mute를 토글.
- **사용처 연결 (ui-tracks.jsx, ui-mixer.jsx)**: 트랙 헤더와 믹서 채널 스트립의 `MuteBtn`에 `auto={DAW._anySolo() && !p.solo}` 전달. solo 변경 시 `Studio`의 force 리렌더 + `useTick`으로 모든 트랙 표시가 즉시 갱신.
- 관련 파일: `audio-engine.js`(`setTrackParam`), `ui-kit.jsx`(`MuteBtn`), `ui-tracks.jsx`/`ui-mixer.jsx`(`auto` 전달).

- **Empty State 보정 (app.jsx)**: `Import Files` / `Import Folder` 버튼도 Electron 환경에서는 각각 `selectFiles()` / `openFolder()`를 사용하도록 변경해 초기 프로젝트 생성 시에도 `filePath`가 보존되도록 수정.
- **Drag & Drop 보완 (app.jsx)**: Electron에서 드롭된 File 객체에 `path`가 있는 경우 Electron 재연결 경로를 사용하고, 경로가 없으면 기존 브라우저 방식으로 fallback.
- **호환성 주의**: 이미 `filePath: null`로 저장된 `.focus` 파일은 실제 오디오 경로 정보가 없으므로 자동 복원할 수 없음. 해당 프로젝트는 Electron의 Import 메뉴/`+ Track`/Empty State Import로 오디오를 다시 연결한 뒤 저장해야 다음부터 자동 복원됨.

## 8. Claude Code 인계 가이드 (Developer Handoff)

> **이 섹션의 목적**: 이 대화에 참여하지 않은 개발자(또는 Claude Code)가 **이 문서 하나만으로** 디자인을 실제 코드베이스에 이어서 구현할 수 있도록 한다.

### 8.0 디자인 파일의 성격 — 먼저 읽을 것
- 이 프로젝트의 HTML/JSX 파일들은 **디자인 레퍼런스이자 동작하는 프로토타입**이다(의도한 외형·동작·오디오 그래프를 실제로 보여줌). 다만 프로덕션 코드로 그대로 복사해 쓰기 위한 것은 아니다.
- 무빌드(Babel 인라인 JSX, `window` 전역 공유) 구조라 **빠른 프로토타이핑에는 최적**이지만, 실제 제품은 아래 8.6의 권장 스택으로 **재구성(recreate)** 하는 것을 전제로 한다.
- **단, `audio-engine.js`(순수 JS, 프레임워크 비종속)는 거의 그대로 이식 가능**하다. 오디오 그래프·합성·렌더링 로직은 검증된 자산이므로 재작성하지 말고 모듈로 가져갈 것.
- **충실도(Fidelity): High-fidelity.** 색상·타이포·간격·인터랙션이 최종안 수준이다. 아래 디자인 토큰을 그대로(픽셀 단위로) 재현할 것.

### 8.3 오디오 엔진 API (`window.DAW`)
> 단일 상태 소스. UI는 이 객체를 읽고 `requestAnimationFrame`(`useTick`)으로 리렌더. **프레임워크 비종속 — 그대로 이식.**

**상수/상태**: `duration`(초), `tracks[]`, `master{volume,eqLow,eqMid,eqHigh,fadeIn,fadeOut}`, `isPlaying`.

**트랙 객체**: `{ id, name, type, color, buffer, peaks, nodes{…}, params }`
`params = { volume(0~1), pan(-1~1), mute, solo, reverb(0~1), echo(0~1), autoOn, automation:[{t:0~1, v:0~1}…] }`

**메서드**
```
DAW.init()                         엔진/그래프 생성 + 데모 4스템 합성(최초 1회)
DAW.play() / pause() / stop()      트랜스포트(루프). play는 ctx.resume() 포함
DAW.seek(sec)                      플레이헤드 이동
DAW.getPlayhead() -> sec           현재 위치
DAW.addFile(File) -> Promise<track>  decodeAudioData → 트랙 추가(파일명=트랙명)
DAW.setTrackParam(id, key, val)    파라미터 적용(게인/팬/센드/오토메이션 즉시 반영)
DAW.setMaster(key, val)            마스터 volume/eqLow/eqMid/eqHigh/fadeIn/fadeOut
DAW.getTrackLevel(id) -> 0~1       트랙 RMS 미터
DAW.getMasterLevel() -> 0~1        마스터 RMS 미터
DAW.renderMix(onProgress) -> AudioBuffer   OfflineAudioContext 전체 믹스 렌더
DAW.automationCurve(bps, n) -> Float32Array  브레이크포인트→곡선 샘플
```

**트랙 그래프(노드 체인)**
```
source(loop) → fader → autoGain → panner
   ├─ dry ─────────────→ masterBus
   ├─ reverbSend → Convolver(절차적 IR) → masterBus
   └─ echoSend → Delay(0.25s) ⇄ fb(0.34) → masterBus
```
**마스터 체인**
```
masterBus → EQ(lowshelf160 / peaking1k Q0.9 / highshelf6.5k) → fade → volume → Compressor(limiter) → analyser → destination
```
- **솔로 로직**: 하나라도 solo면 비솔로 트랙 게인 0. mute는 항상 0.
- **오토메이션**: `autoGain`에 `setValueCurveAtTime`을 루프 단위 선스케줄. `automation`은 정규화 좌표(t,v ∈ 0~1).
- **페이드**: 마스터 `fade` 게인에 첫 루프 기준 in/out 램프(원샷 제스처).
- **클리핑 방지**: 라이브·오프라인 모두 마스터 컴프레서(threshold -2, ratio 12)로 리미팅.

### 8.4 화면·인터랙션 명세
**로더(`LoaderScreen`)** — 좌 320px 브랜드 패널 / 우 프로젝트 설정. 프로젝트명 입력 → 폴더 선택·드래그앤드롭(오디오 확장자 필터) → 감지된 스템 목록 토글 → "Create Project & Open Studio". 데모 4스템 선등록. `.focus` 프로젝트 파일 생성(개념).

**스튜디오(`Studio`)** — 컨트롤바(좌 줌+행높이 / **중앙 트랜스포트+시간** / 우 Track·Mixer·Export) + 스크롤 어레인지 영역.
- **Ruler**: 시간 스케일 m:ss, 줌 따라 눈금 0.25~60s 자동, 클릭 시크.
- **TrackRow**: 헤더(244px, 색바·번호·이름·S/M·인라인 페이더·팬·미터·VOL AUTO 토글·타입칩·삭제) + 레인(TimeGrid·Waveform·AutomationOverlay·플레이헤드). 레인 클릭=시크.
- **AutomationOverlay**: SVG, 포인트 드래그 / 라인 클릭=추가 / 우클릭=삭제(양 끝 제외). 양끝은 t 고정.
- **OutputTrack**: 마스터 EQ 노브 + 타임라인 위 fade-in(green)/fade-out(red) 삼각 영역 + 드래그 핸들.
- **단축키**: Space=재생/정지(INPUT 포커스 시 제외).

**믹서(`MixerWindow`)** — Mixer 버튼 토글. 드래그 가능 플로팅 창(자체 타이틀바·신호등). N개 `ChannelStrip`(reverb/echo/filter 노브·S/M·pan·세로 페이더·미터·dB) + `MasterStrip`(3밴드 EQ·마스터 페이더·스테레오 미터). 빨강 신호등=닫기.

**익스포트(`ExportDialog`)** — settings(파일명·포맷(MP3/WAV)·비트레이트·샘플레이트·노멀라이즈) → rendering(`stepLabel` + 진행률 바) → done(파일 다운로드 링크 + 포맷 표기). Electron = ffmpeg MP3, 브라우저 = lamejs MP3 or WAV fallback.

**설정(`SettingsDialog`)** — Settings 메뉴 → 모달. Color Theme 섹션: ThemeSwatch 카드 3개(미니 프리뷰). 클릭 즉시 테마 전환(data-theme 교체). Done 버튼으로 닫기.

### 8.5 상태 관리 매핑
- **단일 소스 = `DAW`(엔진)**. React 상태는 UI 전용(`pxPerSec`, `ampZoom`, `laneH`, `loop`, `showMixer`, `showExport`, 라우팅 `screen`/`projectName`)만 보유.
- 엔진 값 변경 → `force(n=>n+1)`로 리렌더, 미터/플레이헤드는 `useTick`(rAF)로 폴링.
- **이식 시 권장**: 엔진을 그대로 두고 얇은 store(Zustand/Redux/Context)로 감싸 구독화. 미터·플레이헤드만 rAF 구독, 나머지는 이벤트 기반.

### 8.6 권장 이식 스택 & 작업 순서
**스택**: React(또는 기존 코드베이스 프레임워크) + TypeScript + Vite, 스타일은 CSS 변수 토큰 유지(또는 Tailwind config에 토큰 매핑), 데스크톱은 Electron(또는 Tauri).
1. `audio-engine.js` → `engine.ts` 모듈화(타입 추가, `export default DAW`). **로직 변경 금지**, 인터페이스만 정리.
2. 디자인 토큰(8.2)을 `tokens.css`/`tailwind.config`로 이전, 폰트(Space Grotesk/Mono) 로드.
3. UI 키트(Knob/Fader/Meter/SleekSlider/Seg/Icon) 컴포넌트화 — 가장 재사용 많음.
4. Studio 레이아웃(컨트롤바 3분할 + 어레인지) → TrackRow → Ruler → AutomationOverlay 순.
5. MixerWindow, OutputTrack, LoaderScreen, ExportDialog.
6. Electron 셸: 폴더 스캔(`dialog`+`fs`), 별도 믹서 `BrowserWindow`+IPC/`BroadcastChannel`, ffmpeg MP3.
7. 추가 개선: Electron 실제 믹서 별도 창(IPC/BroadcastChannel), 자동 업데이트(electron-updater), 성능(Phase 6).

### 8.7 주의 사항 및 남은 과제
- **클립별 파라미터 인스펙터**: `Phase 1-D` 미완성 항목. 클립 선택 시 헤더 하단에 볼륨 오프셋·FX 편집 패널 표시 기능 미구현. `DAW.setClipParam()` 엔진 메서드는 준비됨.
- **MP3 브라우저 인코딩 속도**: lamejs는 순수 JS라 장시간(5분+) 세션에서 수십 초 소요 가능. 진행률 표시로 UX 보완 중.
- **믹서 별도 창**: 현재 인앱 플로팅 div(실제 OS 창 아님). Electron에서 진짜 `BrowserWindow` + `BroadcastChannel`/IPC 동기화 필요.
- **자동재생 정책**: 첫 소리는 사용자 제스처(Play 클릭) 후 출력.
- **데모 길이**: 8초 시간 기반 루프. 실제 파일 로드 시 `duration`이 최장 트랙으로 확장됨.
- **창 크롬(Windows/Linux)**: `frame:false` 창에서 메뉴바를 draggable 영역으로 사용하고, custom 최소화/최대화/닫기 버튼을 IPC로 연결.
- **테마와 인라인 스타일**: 일부 컴포넌트가 하드코딩 색상값을 인라인 스타일로 사용. 테마 전환 시 CSS 변수 참조로 순차 정리 권장.

### 8.8 실행 방법

**브라우저(프로토타입)**
- 정적 서버에서 `studio.html` 오픈(상대경로 스크립트 로드). 빌드 불필요. React/Babel/폰트는 CDN.
- 변형안은 `variations.html`.

**Electron 앱**
```bash
npm install          # electron + electron-builder + ffmpeg-static 설치
npm start            # Electron 앱 실행
npm run dist:win     # Windows 설치 파일(.exe) 빌드
npm run dist:mac     # macOS dmg 빌드
npm run dist:linux   # Linux AppImage 빌드
```
- `ffmpeg-static`이 설치되어야 MP3 인코딩 작동. 미설치 시 콘솔 경고 출력, WAV만 가능.
- `electron/` 폴더에 `main.js`, `preload.js` 위치. `studio.html` 기준 상대경로로 파일 로드.

## 1. 시험 항목

### B-205 (v1.17.0) — VRB/ECHO 배지 클릭 시 믹서 열고 해당 노브 강조 확인

**[시험항목]** 트랙 헤더의 VRB/ECHO 배지를 클릭하면 믹서 창이 열리고(또는 앞으로 나오고) 해당 트랙의 리버브/에코 노브가 화면에 보이도록 스크롤되며 펄스로 강조되는지.

**[시험 환경 준비]**
- 오디오 트랙이 여러 개 로드된 프로젝트. 최소 한 트랙의 리버브 또는 에코 값을 0보다 크게 설정(배지가 보이도록).

**[시험방법 및 절차]**
1. 믹서가 닫힌 상태에서, 리버브 값이 있는 트랙의 **VRB 배지 클릭** → 믹서 창이 새로 열리고, 해당 트랙 채널 스트립의 **VRB 노브**가 보이며 잠깐 밝게 펄스(glow)하는지 확인.
2. **ECHO 배지 클릭** → 같은 방식으로 해당 트랙 **ECHO 노브**가 강조되는지 확인.
3. 믹서가 **이미 열린** 상태에서 다른 트랙의 배지 클릭 → 믹서 창이 앞으로 나오고(포커스) 해당 노브로 스크롤·펄스되는지 확인.
3-1. **(v1.17.1) 같은 배지를 연속으로 빠르게 다시 클릭** → 매 클릭마다 펄스가 다시 나오는지 확인(이전엔 직전 타이머 잔존 레이스로 2번째 클릭이 안 나오던 버그).
3-2. **(v1.17.2) 믹서를 닫은 뒤 배지 클릭** → 믹서가 다시 열리면서 해당 노브 펄스가 나오는지 확인(이전엔 재오픈 시 펄스가 안 나오던 버그). 닫기→열기를 여러 번 반복해도 매번 펄스가 나오는지 확인.
4. 트랙 수가 많아 가로 스크롤이 필요한 경우, 대상 노브가 **화면 중앙으로 스크롤**되어 보이는지 확인.
5. (참고) 실제 마우스 포인터가 물리적으로 이동하지는 않음 — 노브 위치로 스크롤+강조까지가 사양(협의됨).

**결과:** ⚠️ 사용자 확인 대기.

### B-204 (v1.16.0) — 트랙 헤더 VRB/ECHO 인디케이터 표시·위치 고정 확인

**[시험항목]** 트랙별 리버브(VRB)·에코(ECHO) send 값이 0이 아닐 때만 트랙 헤더 VOL AUTO 옆에 배지가 뜨고, 켜지는 조합에 관계없이 배지 위치가 항상 고정되는지.

**[시험 환경 준비]**
- 오디오 트랙이 여러 개 로드된 프로젝트를 M(중간) 또는 L(큰) 트랙 크기로 연다.

**[시험방법 및 절차]**
1. 초기 상태(리버브/에코 0): 트랙 헤더 VOL AUTO 옆에 **VRB/ECHO 배지가 보이지 않고**, `VOL AUTO` 글자가 한 줄로 표시되며 우측 타입 칩(AUDIO)·삭제(−) 버튼이 잘리지 않는지 확인.
2. Mixer 창을 열어 한 트랙의 **VRB(리버브) 노브만** 0보다 크게 올린다 → 해당 트랙 헤더에 **VRB(보라) 배지만** 나타나는지, ECHO 자리는 비어 있고 VRB가 정해진 위치에 뜨는지 확인.
3. 이어서 **ECHO 노브만** 올린다(리버브는 0으로) → **ECHO(파랑) 배지**가 2번에서 VRB가 있던 자리 옆 **동일한 고정 위치**에 뜨는지(=VRB 없다고 ECHO가 왼쪽으로 밀리지 않음) 확인.
4. 둘 다 올림 → VRB·ECHO 두 배지가 나란히, 삭제(−) 버튼까지 한 줄에 잘림 없이 표시되는지 확인.
5. 값을 다시 0으로 내리면 배지가 사라지되 다른 컨트롤 위치는 그대로인지 확인.

**결과:** ⚠️ 사용자 확인 대기.

### B-203 (v1.16.0) — 트랙별 에코 웹/네이티브 청감 일치 확인

**[시험항목]** 같은 ECHO 노브 값에서 웹 미리듣기와 네이티브(연결) 재생/익스포트의 에코 타이밍·크기가 동일하게 들리는지.

**[시험 환경 준비]**
- 리듬감 있는(어택이 뚜렷한) 트랙 1개, 네이티브 오디오 엔진 연결 상태.

**[시험방법 및 절차]**
1. 한 트랙의 ECHO를 중간값(예: 0.5)으로 설정.
2. **네이티브 미연결(웹 엔진)** 상태로 재생하며 에코 반복 간격·크기를 듣는다.
3. **네이티브 연결** 상태로 동일 트랙 재생 → 에코 딜레이(약 0.3s 간격)·감쇠·크기가 2번과 **동일하게** 들리는지 비교.
4. (선택) Export한 파일에서도 동일한 에코가 들리는지 확인.
5. **기대 결과:** 두 엔진에서 에코 딜레이 타임(0.3s)·feedback(0.34)·wet(노브×0.45)이 같아 청감상 차이가 없어야 함. (리버브는 알고리즘 차이로 미세하게 다를 수 있으나 레벨은 유사.)

**결과:** ⚠️ 사용자 확인 대기(네이티브 연결 필요).

### B-202 (v1.15.4) — 재생 중 seek-back + zoom-in 시 플레이바가 (일반+Output FX) 트랙 헤더로 침범하지 않는지 확인

**[시험항목]** 재생 도중 현재 재생 시점보다 뒤로 seek 후 time zoom-in 했을 때, 플레이헤드 세로선이 좌측 sticky 헤더 영역(일반 트랙 + Output FX 트랙 + Ruler)을 침범하지 않고 헤더 뒤로 가려지는지.

**[시험 환경 준비]**
- 오디오 트랙이 여러 개 로드된 프로젝트를 연다. (헤더가 화면 좌측에 sticky로 고정된 arrange 화면)

**[시험방법 및 절차]**
1. 재생(Play)을 시작하고 플레이헤드가 곡 중반쯤(예: 1:20 근처) 지나가게 둔다.
2. 재생 중, 타임라인/Ruler에서 **현재 플레이헤드보다 앞쪽(이전 시점)** 을 클릭하거나 Ruler를 클릭해 **뒤로 seek** 한다.
3. `Ctrl+마우스휠`(또는 zoom 슬라이더)로 **time zoom-in** 하여 플레이헤드가 뷰포트 왼쪽으로 스크롤 아웃되게 한다.
4. **기대 결과:** 플레이헤드(흰색/amber 세로선)가 각 레인 영역에만 나타나고, **좌측 헤더 위로 세로선이 그려지지 않는다.** 아래 세 곳 모두 확인:
   - ① 일반 오디오 트랙 헤더(볼륨/솔로/뮤트 등)
   - ② **Output FX 트랙 헤더(MUTE CLR / MASTER / EFFECT 영역)** ← 이번 추가 수정 지점
   - ③ Ruler 좌상단 `TRACK +` 코너
5. (Output FX) Repeat 구간이 설정된 상태에서도 플레이바가 Output FX 헤더를 침범하지 않고, loop 하이라이트/Repeat 버튼 위로는 기존처럼 플레이헤드가 정상 표시되는지 확인.

**결과:** ⚠️ 사용자 확인 대기.

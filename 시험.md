## 2. 시험 항목

### B-178) Export(Web Audio 오프라인 렌더)에 Saturation·Widener·Exciter 반영 (v1.9.6)

**[시험 환경 준비]**
- 브라우저 단독 실행(네이티브 엔진 미연결) 또는 네이티브 엔진이 없는 환경에서 앱 실행 — 오프라인 렌더 경로를 타도록 함. (데스크톱 네이티브 엔진은 기존부터 적용되어 있어 별도 확인용)
- 데모 세션 또는 스템을 로드. 믹서 MASTER(또는 Ambience 창)의 OUTPUT EFFECTS에서 **Saturation·Widener·Exciter/Enhancer**를 뚜렷이 들리게(예: 60~100%) 설정.

**[시험 항목]**
1. **(우선) 세 효과를 모두 0(기본)으로 둔 일반 프로젝트의 Export에서 소리가 정상적으로 녹음되는가** — 무음이 아니어야 하며 종전(v1.9.5)과 동일(무음 회귀 수정 확인).
2. 세 효과를 올린 상태에서 **내보낸 MP3/WAV의 음색이 모니터링음과 일치**하는가(전보다 두툼함/넓음/반짝임이 파일에도 반영).
3. Widener를 올렸을 때 소리가 정상 녹음되고(무음 아님), 모노 합산에서 위상 붕괴 없이 좌우 폭만 넓어지는가.

**[시험 방법 및 절차]**
0. **세 효과 모두 0인 상태로 먼저 Export → 파일에 소리가 정상적으로 들어있는지(무음 아님) 확인.** (무음 회귀 1차 점검)
1. Saturation만 70%로 올리고 나머지 0 → Export(WAV) → 원본(0%) 대비 배음이 더해져 따뜻/두툼해졌는지 청취 비교.
2. Widener만 80%로 올리고 Export → 좌우가 넓어지는지, 모노로 합쳐 들었을 때 본체가 사라지지 않는지 확인.
3. Exciter만 80%로 올리고 Export → 고역의 선명함/공기감이 파일에 반영됐는지 확인.
4. 세 효과 모두 0으로 Export한 파일을 v1.9.5에서 같은 프로젝트로 내보낸 파일과 비교 → 사실상 동일해야 함.
5. (선택) 데스크톱 네이티브 엔진 연결 상태에서도 동일 설정으로 Export해 결과가 모니터링음과 일치하는지 교차 확인.

## 4. 확인 메모

- v1.2.33부터 Vari BPM 실시간 재생은 캐시형 Time Stretch 프리뷰를 우선 사용합니다. 첫 재생 또는 BPM 비율 변경 직후에는 트랙별 프리뷰 버퍼 준비 시간이 있을 수 있습니다.
- v1.2.40부터 Playback BPM 입력은 500ms 대기 후 마지막 값 기준으로 프리뷰 계산을 시작하며, 계산 중 들어간 pending 입력이 뒤늦게 중복 계산을 일으키지 않도록 정리합니다.
- Export `Keep pitch`는 Electron Export 전용 단기 안정 기능입니다. 데스크톱 경로는 `ffmpeg atempo` 기준선을 사용하며, 브라우저 fallback의 JS Time Stretch는 품질 기준 미달 실험 경로로 취급합니다.

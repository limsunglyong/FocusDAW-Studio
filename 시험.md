## 1. 시험 항목

## v1.20.9 — 파형 다운샘플링 피크 손실 수정 (녹음 중·파일 트랙)

### T-1.20.9-1. 녹음 중 라이브 파형 피크 유지
- 시험 내용: Audio In 트랙으로 수 분간 녹음하면서 초반에 박수를 몇 번 친다. 녹음이 길어지는 동안 초반 박수 피크의 높이를 관찰한다.
- 판정 기준: 녹음이 계속되어도 박수 피크가 줄어들지 않고 원래 높이를 유지한다(중간 재그리기에서 파형 모양이 확 바뀌지 않음). 녹음 종료 후의 파형과 라이브 파형의 피크 높이가 일치한다.

### T-1.20.9-2. 파일 트랙 줌 변화 시 피크 안정
- 시험 내용: 트랜지언트(박수·드럼 히트)가 있는 긴 오디오 파일(3분 이상)을 임포트하고, TIME 줌을 여러 단계로 확대/축소하며 특정 트랜지언트 피크를 관찰한다. 볼륨을 1.0 초과로 올려 클리핑 표시(빨간 오버레이)도 함께 확인한다.
- 판정 기준: 어느 줌 레벨에서도 트랜지언트 피크가 사라지거나 낮아지지 않고, 클리핑 표시도 줌과 무관하게 같은 위치에 나타난다.

## v1.20.8 — 네이티브 엔진 종료/절단 경합 하드닝

### T-1.20.8-1. 앱 종료 시 엔진 비정상 종료 코드 재발 관찰 (장기 tracing)
- 시험 내용: 평소처럼 사용하며 앱을 시작/종료할 때(믹서 창을 열어 둔 채 종료 포함) 콘솔/로그의 `[AudioEngine] Process exited with code ...` 메시지를 관찰한다. 단발성 경합이므로 일정 기간 반복 관찰이 필요하다.
- 판정 기준: 종료 코드가 항상 정상(1 = Electron kill, 또는 0)이고, `3221225477`(0xC0000005) 등 비정상 코드가 재발하지 않는다. 재발 시 `%LOCALAPPDATA%\CrashDumps`의 새 덤프 유무를 함께 확인해 보고한다.

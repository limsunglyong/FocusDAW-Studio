### B-188) 네이티브 Export에 볼륨 오토메이션 적용 확인 (v1.9.16)

**[시험 환경 준비]**
- Windows Electron 앱을 `npm start`로 실행(새 `bin/FocusDAW-AudioEngine.exe` 반영).
- 스템을 불러오고, 한 트랙에서 **VOL AUTO**를 켠 뒤 곡선을 만든다(예: 곡 후반으로 갈수록 볼륨을 크게 내리는 페이드, 또는 중간을 깊게 내리는 V자). 트랙 크기 L에서 **Curve** 토글로 직선/곡선도 각각 시험.

**[시험 항목]**
1. 볼륨 오토메이션을 적용한 트랙이 **내보낸 파일에서도** 시간에 따라 볼륨이 변하는가(귀/파형으로 확인).
2. 오토메이션을 끈(VOL AUTO off) 상태에서는 내보낸 결과가 정적 볼륨으로 일정한가.
3. **Curve(곡선 보정)** on/off에 따라 직선/부드러운 곡선이 결과에 반영되는가.
4. 저장된 `.focus` 프로젝트를 다시 열어 바로 Export해도 오토메이션이 반영되는가(재오픈 동기 경로).
5. 오토메이션 + 마스터 EQ/이펙트/Normalize를 함께 켠 export도 정상 동작하는가.

**[시험 방법 및 절차]**
1. 한 트랙에 후반 페이드아웃 오토메이션을 만들고 Export(WAV) → 파일을 파형 보기로 열어 후반부가 점점 작아지는지 확인.
2. 같은 트랙 VOL AUTO를 끄고 다시 Export → 볼륨이 일정한지 비교.
3. Curve 토글을 바꿔 두 번 Export → 곡선 모양 차이가 결과에 반영되는지 확인.
4. 프로젝트 저장 → New Project → 저장 파일 다시 열기 → 즉시 Export → 오토메이션 반영 확인.
5. (개발 검증) `FOCUS_AUTO=1`(직선)/`FOCUS_AUTO=2`(곡선)로 `node tools/export-silence-probe.js <port> "<wav>"` 실행 시, 결과의 `firstHalfRmsDb` vs `secondHalfRmsDb`로 오토메이션 적용을 수치 확인.

### B-187) Export 무음 근본 원인 2건 수정 확인 (v1.9.15)

**[시험 환경 준비]**
- Windows Electron 앱을 `npm start`로 실행(또는 패키징 앱). 새 `bin/FocusDAW-AudioEngine.exe`가 반영된 상태.
- 기존에 무음 export(`Measured LUFS: -70`)가 발생하던 동일 프로젝트(9트랙) 로드.

**[시험 항목]**
1. 마스터 출력 이펙트를 **모두 0**으로 둔 상태에서 Export한 파일에 **정상 음량의 오디오**가 들어있는가(무음 아님).
2. 터미널 로그에서 `Measured LUFS`가 더 이상 `-70`이 아니고 실제 값(예: −10~−25)으로 측정되는가.
3. 완료 화면에 `Web Audio fallback used` 경고가 **표시되지 않는가**(네이티브 export 성공).
4. 마스터 EQ 밴드를 올리거나(부스트) 프리셋을 적용한 뒤 Export하면 그 EQ가 결과 파일에 반영되는가.
5. Saturation·Widener·Exciter를 켠 export도 정상 음량으로 반영되는가.
6. Normalize ON(−9/−14 LUFS) export가 목표 음량 부근으로 정규화되는가.

**[시험 방법 및 절차]**
1. 문제 프로젝트를 열고 이펙트 0 상태로 Export → WAV → Render → Save file. 저장 파일을 외부 플레이어/파형으로 열어 소리·파형 확인.
2. 터미널에서 `Offline track rendering enabled (via transportSource)`와 `Measured LUFS` 값을 확인.
3. 같은 프로젝트로 MP3, Normalize ON(−9 LUFS) Export를 반복해 음량/정상 재생 확인.
4. (개발 검증용) 네이티브 엔진 단독 검증은 `node tools/export-silence-probe.js <port> "<wav>"`로 export WAV의 peak/RMS를 직접 측정(엔진을 `--port`로 먼저 기동). peak가 −inf가 아니어야 함.

### B-179) Export 무음 방지 — 네이티브 export 실패 시 Web Audio fallback (v1.9.7)

**[시험 환경 준비]**
- Windows Electron 앱에서 실제 오디오 파일 또는 데모 세션을 로드.
- Export 창을 열고 Format을 WAV로 먼저 선택. Normalize와 Keep pitch는 OFF로 시작.
- 가능하면 기존에 무음 export가 발생했던 동일 프로젝트/스템으로 확인.

**[시험 항목]**
1. 세 출력 이펙트(Saturation·Widener·Exciter)를 모두 0으로 둔 상태에서 Export한 WAV가 **무음이 아닌가**.
2. Electron에서 네이티브 엔진이 연결돼 있으면 기본 Export가 네이티브 경로를 먼저 시도하고, 실패/무음 감지 시에만 Web Audio fallback으로 파일을 생성하는가.
3. fallback 발생 시에도 MP3 저장이 정상 동작하고, 저장된 파일에 실제 오디오가 들어있는가.

**[시험 방법 및 절차]**
1. 기존 무음 문제가 발생했던 프로젝트를 열고, 마스터 출력 이펙트 3종을 모두 0으로 설정.
2. Export → WAV → Render → Save file 실행.
3. 저장된 WAV를 재생해 소리가 있는지 확인. 파형/미터가 있는 플레이어에서 열 수 있으면 RMS 또는 파형도 함께 확인.
4. 같은 프로젝트로 Export → MP3 → Render → Save file 실행 후 MP3도 소리가 있는지 확인.
5. 개발자 콘솔 확인이 가능하면 네이티브 실패/무음 감지 시 `"Native export failed; falling back to Web Audio export"` 경고가 표시되고, 완료 화면에 `"Web Audio fallback used."` 안내가 보이는지 확인.

### B-178) Export(Web Audio 오프라인 렌더)에 Saturation·Widener·Exciter 반영 (v1.9.6)

**[시험 환경 준비]**
- 브라우저 단독 실행(네이티브 엔진 미연결) 또는 네이티브 엔진이 없는 환경에서 앱 실행 — 오프라인 렌더 경로를 타도록 함. (데스크톱 네이티브 엔진은 기존부터 적용되어 있어 별도 확인용)
- 데모 세션 또는 스템을 로드. 믹서 MASTER(또는 Ambience 창)의 OUTPUT EFFECTS에서 **Saturation·Widener·Exciter/Enhancer**를 뚜렷이 들리게(예: 60~100%) 설정.

**[시험 항목]**
1. **(우선) 세 효과를 모두 0(기본)으로 둔 일반 프로젝트의 Export에서 소리가 정상적으로 녹음되는가** — 무음이 아니어야 하며 종전(v1.9.5)과 동일(무음 회귀 수정 확인).
2. 세 효과를 올린 상태에서 **내보낸 MP3/WAV의 음색이 모니터링음과 일치**하는가(전보다 두툼함/넓음/반짝임이 파일에도 반영).
3. Widener를 올렸을 때 소리가 정상 녹음되고(무음 아님), 모노 합산에서 위상 붕괴 없이 좌우 폭만 넓어지는가.

**[시험 방법 및 절차]**
0. **세 효과 모두 0인 상태로 먼저 Export → 파일에 소리가 정상적으로 들어있는지(무음 아님) 확인.** (무음 회귀 1차 점검)
1. Saturation만 70%로 올리고 나머지 0 → Export(WAV) → 원본(0%) 대비 배음이 더해져 따뜻/두툼해졌는지 청취 비교.
2. Widener만 80%로 올리고 Export → 좌우가 넓어지는지, 모노로 합쳐 들었을 때 본체가 사라지지 않는지 확인.
3. Exciter만 80%로 올리고 Export → 고역의 선명함/공기감이 파일에 반영됐는지 확인.
4. 세 효과 모두 0으로 Export한 파일을 v1.9.5에서 같은 프로젝트로 내보낸 파일과 비교 → 사실상 동일해야 함.
5. (선택) 데스크톱 네이티브 엔진 연결 상태에서도 동일 설정으로 Export해 결과가 모니터링음과 일치하는지 교차 확인.

## 4. 확인 메모

- v1.2.33부터 Vari BPM 실시간 재생은 캐시형 Time Stretch 프리뷰를 우선 사용합니다. 첫 재생 또는 BPM 비율 변경 직후에는 트랙별 프리뷰 버퍼 준비 시간이 있을 수 있습니다.
- v1.2.40부터 Playback BPM 입력은 500ms 대기 후 마지막 값 기준으로 프리뷰 계산을 시작하며, 계산 중 들어간 pending 입력이 뒤늦게 중복 계산을 일으키지 않도록 정리합니다.
- Export `Keep pitch`는 Electron Export 전용 단기 안정 기능입니다. 데스크톱 경로는 `ffmpeg atempo` 기준선을 사용하며, 브라우저 fallback의 JS Time Stretch는 품질 기준 미달 실험 경로로 취급합니다.

## 4. 확인 메모

- v1.2.33부터 Vari BPM 실시간 재생은 캐시형 Time Stretch 프리뷰를 우선 사용합니다. 첫 재생 또는 BPM 비율 변경 직후에는 트랙별 프리뷰 버퍼 준비 시간이 있을 수 있습니다.
- v1.2.34에서는 재생 중 Playback BPM 연속 변경 시 프리뷰 재계산을 디바운스하고, 준비 중 오버레이를 표시하도록 보정했습니다.
- v1.2.35에서는 Time Stretch 준비 표시를 전체 화면 오버레이에서 BPM 패널을 가리지 않는 작은 원형 표시로 변경하고, 연속 입력 대기 시간을 약 300ms로 조정했습니다.
- v1.2.36에서는 작은 원형 준비 표시를 툴바가 아닌 트랙 영역 overlay로 옮겨 BPM 표시기 레이아웃이 움직이지 않도록 했습니다.
- v1.2.37에서는 원형 준비 표시 위치를 트랙 영역 우상단에서 현재 보이는 트랙 묶음의 중앙으로 변경했습니다.
- v1.2.38에서는 정지 상태에서 Play를 눌렀을 때 원형 준비 표시가 먼저 paint된 뒤 Time Stretch 프리뷰 계산이 시작되도록 보정했습니다.
- v1.2.39에서는 Playback BPM 입력을 UI draft 값으로 먼저 표시하고, 300ms 동안 입력이 멈춘 뒤 마지막 값만 엔진에 커밋하도록 변경했습니다.
- v1.2.40에서는 입력 대기 시간을 500ms로 늘리고, 프리뷰 계산 완료 시 pending 입력 draft/timer를 clear해 연속 프리뷰 계산을 방지했습니다.
- Export `Keep pitch`는 계속 Electron Export 전용 단기 안정 기능입니다. 데스크톱 경로는 `ffmpeg atempo` 기준선을 사용하며, 브라우저 fallback의 JS Time Stretch는 품질 기준 미달 실험 경로로 취급합니다.

- v1.2.23부터 Electron Export의 `Keep pitch`는 JS 후처리 대신 ffmpeg `atempo` 경로를 우선 사용합니다.

- v1.2.24에서는 Electron `Keep pitch` Export에서 원속도 렌더 후 ffmpeg `atempo`를 한 번만 적용하도록 수정했습니다.

- B-112에서 BPM 조정은 정상 확인되었습니다. 음질/볼륨 문제의 주 원인은 Export 창의 Normalize(Limiter) 옵션으로 확인되었고, Normalize OFF에서는 볼륨이 정상입니다.

- v1.2.25(#57)에서 Normalize를 메이크업 게인 없는 소프트 클리퍼 → ffmpeg `loudnorm` 2-pass(−14 LUFS / −1 dBTP)로 전환했습니다. 데스크톱 경로는 클리퍼를 끄고 loudnorm이 라우드니스/트루피크를 담당하며, 브라우저 폴백은 기존 클리퍼를 유지합니다.

- B-116에서 `ffmpeg atempo` benchmark 기준선은 합성 샘플 기준으로 큰 품질 저하가 드러나지 않았습니다. 다만 샘플이 MIDI풍의 단순한 음색이라 실제 곡/보컬/복잡한 믹스에서 추가 확인이 필요합니다.

- B-117에서 실제 MP3 입력(`Another World.mp3`) 기반 변환 파일도 청감상 좋다고 확인되었습니다.

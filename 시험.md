## 2. 시험 항목

### B-169) Advanced Equalizer 창 (v1.8.0)
- **시험 환경 준비**: `npm start`로 앱 실행 → 데모/임의 프로젝트 로드(트랙 2개 이상) → Mixer 창 열기 → Mixer 또는 Advanced 메뉴에서 **Equalizer** 창 열기.
- **시험 절차 및 항목**:
  1. **창 표시(4)**: EQ 창이 빈 화면이 아니라 EQ 곡선·밴드 핸들·하단 PRESET/USER 컨트롤로 채워진다. 창 크기를 조절하면 캔버스가 따라 확장된다.
  2. **주파수별 조작(6)**: 9개 핸들을 세로로 드래그하면 해당 밴드 dB(−12~+12)가 변하고 하단 리드아웃 값이 갱신된다. 핸들 더블클릭 시 0dB로 리셋된다.
  3. **Mixer/엔진 동기화**: EQ 창에서 밴드를 바꾸면 Mixer 창의 Graphic EQ 곡선과 출력 사운드가 동일하게 변한다(반대 방향도 동일). 곡 재생 중 청취로 음색 변화 확인.
  4. **기본 프리셋(3)**: PRESET 행의 Reset/Pop/Classic/Hip Hop 클릭 시 곡선이 해당 프리셋으로 바뀌고 Mixer와 일치한다.
  5. **FFT 백그라운드(7)**: 재생 중 캔버스 뒤에 Output(전 트랙 합산) 스펙트럼 영역과 밴드별 펄스 컬럼이 표시된다. 정지 시 펄스는 멈춘다.
  6. **유저 프리셋(8)**: USER 슬롯 클릭 → 오버롤 메뉴(Save here/Recall/Rename) 표시. Save 후 현재 EQ가 저장되고 점 표시가 켜진다. 다른 EQ로 바꾼 뒤 Recall하면 저장값으로 복원된다. Rename으로 이름 변경 후 앱을 재시작해도 이름·값이 유지된다(localStorage).
  7. **Space/단축키(5)**: EQ 창 포커스 상태에서 Space로 재생/일시정지가 토글된다(footer "playing/stopped" 갱신). Esc/F3로 창이 닫히고, Ctrl+Z/Ctrl+Y로 밴드 변경이 Undo/Redo된다. 이름 편집 input에 포커스 중에는 Space가 글자 입력으로 동작한다.
- **자동 확인**: `npm run build:renderers` 가 오류 없이 `build/advanced-eq-app.js` 를 생성한다.

### B-170) 선택 EQ 프리셋 이름 표시 / 활성 버튼 fill / 평탄 라인 (v1.8.1)
- **시험 환경 준비**: `npm start` → 프로젝트 로드 → Mixer 창 + Equalizer 창 동시 표시.
- **시험 절차 및 항목**:
  1. **평탄 라인(EQ 1)**: EQ 창에서 Reset(또는 모든 밴드 0dB) 상태일 때 0dB 위치를 따라 각 밴드를 잇는 곡선 라인이 또렷이 그려진다(평탄해도 라인이 사라지지 않음).
  2. **이름 표시(EQ 3)**: EQ 창 PRESET에서 Pop/Classic/Hip Hop 선택 시 캔버스 우상단에 해당 이름이 Michroma 폰트로 표시된다. 유저 슬롯 Recall 시 그 슬롯 이름이 표시된다. 밴드를 수동으로 드래그하거나 Reset하면 이름이 사라진다.
  3. **버튼 fill(EQ 2)**: 현재 선택된 PRESET 버튼 또는 USER 슬롯 버튼이 채워진(active) 상태로 강조된다. 수동 편집/Reset 시 어떤 버튼도 채워지지 않는다.
  4. **Mixer overlay(Mixer 1)**: Mixer의 Graphic EQ 그래픽 우상단에 현재 선택된 EQ 이름이 overlay로 표시된다. 미설정/초기화/Reset 상태에서는 아무 글자도 표시되지 않는다.
  5. **양창 동기화**: Mixer에서 Pop을 고르면 EQ 창에도 "Pop"이 표시되고 버튼이 채워진다(반대도 동일). 한쪽에서 밴드를 드래그하면 양쪽 이름/표시가 함께 사라진다.
  6. **유저 프리셋 → Mixer 표시(v1.8.2)**: EQ 창에서 유저 슬롯을 Recall하면 EQ 창 우상단뿐 아니라 **Mixer Graphic EQ 우상단에도 그 유저 프리셋 이름**이 표시된다. 이후 밴드를 드래그하거나 기본 프리셋을 고르면 양쪽 모두 사라지거나 해당 이름으로 바뀐다. 활성 유저 슬롯을 Rename하면 Mixer 표시 이름도 갱신된다.
  7. **custom 표시 + 깜빡임 효과(v1.8.3/1.8.4)**: 명명된 프리셋이 없는 상태에서 밴드를 하나라도 0dB이 아니게 조정하면 EQ 창과 Mixer 창 우상단에 **'custom'**이 표시되며 글자가 점멸한다(flashing — 부드러운 페이드가 아니라 on/off 깜빡임). 모든 밴드를 0dB로 되돌리거나 Reset하면 표시가 사라진다. 기본/유저 프리셋 이름은 깜박이지 않고 정적으로 표시된다.

### B-171) 네이티브 엔진 EQ 힙 손상 크래시 수정 (v1.8.6)
- **시험 환경 준비**: 네이티브 엔진은 **재빌드 완료**(2026.06.18, `bin/FocusDAW-AudioEngine.exe` 2.3MB로 교체됨). `npm start`로 실행하면 콘솔에 `[FocusDAW] Spawning JUCE Audio Engine` 로그가 보여야 함. 트랙 2개 이상 로드. (다시 빌드하려면 `juce_skeleton/build_native.bat` — JUCE는 `third_party/JUCE`.)
- **시험 절차 및 항목**:
  1. **재생 완료 무크래시**: 곡을 끝까지(루프 경계 포함) 여러 번 재생/정지 반복. 콘솔에 `[AudioEngine] Process exited with code 3221226356`(0xC0000374)가 더 이상 나타나지 않는다.
  2. **EQ 연속 변경 스트레스**: 재생 중 EQ 밴드를 빠르게 드래그하고, 기본 프리셋(Pop/Classic/Hip Hop)과 유저 프리셋 Recall(9밴드 일괄 적용)을 반복 클릭한다. 크래시·오디오 끊김·힙 손상 로그가 없어야 한다.
  3. **기능 유지**: EQ 변경이 실제 사운드에 즉시 반영되고 Mixer/EQ 창 표시와 일치한다(계수 갱신이 오디오 스레드로 지연돼도 한 블록 내 반영되어 체감 지연 없음).
  4. **재생 중 프로젝트 교체(v1.8.7)**: 곡을 재생하는 상태에서(또는 정지 직후) 다른 프로젝트/스템을 임포트해 트랙을 교체(`clearTracks`+`loadTrack` 연속 수신)한다. 로딩 중 `Process exited with code 3221226356`가 발생하지 않아야 한다. 서로 다른 프로젝트를 빠르게 여러 번 열고 닫아도 크래시가 없어야 한다.
  5. **트랙 재로드/교체(v1.8.8 — double-free 수정)**: 이미 로드된 트랙과 **같은 trackId를 다시 loadTrack** 하는 흐름(프로젝트 열기 시 동일 스템 재로드, 같은 프로젝트를 연속으로 다시 열기)을 반복한다. 옛 `TrackAudioSource` 파괴 경로에서 더 이상 `code 3221226356`(heap-use-after-free)가 나지 않아야 한다. (이 경로가 종전 크래시의 실제 원인이었음.)
- **자동 확인**: 네이티브 빌드가 컴파일 에러 없이 성공한다(완료됨 — Release 빌드 성공, 바이너리 스모크 기동 정상).

## 4. 확인 메모

- v1.2.33부터 Vari BPM 실시간 재생은 캐시형 Time Stretch 프리뷰를 우선 사용합니다. 첫 재생 또는 BPM 비율 변경 직후에는 트랙별 프리뷰 버퍼 준비 시간이 있을 수 있습니다.
- v1.2.40부터 Playback BPM 입력은 500ms 대기 후 마지막 값 기준으로 프리뷰 계산을 시작하며, 계산 중 들어간 pending 입력이 뒤늦게 중복 계산을 일으키지 않도록 정리합니다.
- Export `Keep pitch`는 Electron Export 전용 단기 안정 기능입니다. 데스크톱 경로는 `ffmpeg atempo` 기준선을 사용하며, 브라우저 fallback의 JS Time Stretch는 품질 기준 미달 실험 경로로 취급합니다.

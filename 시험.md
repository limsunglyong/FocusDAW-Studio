## v1.43.0 — Save As 오디오 수집 + 상대경로 (Phase 7) (2026-07-21)

> 배경: 종전에는 Save As가 `.focus`(JSON)만 쓰고 오디오는 복사하지 않아, ① 미저장 프로젝트의 녹음/consolidation이 temp에 남아 OS 청소 시 소실되고, ② 프로젝트 폴더를 옮기면 깨졌다. 이제 Save As/최초 저장 시 **앱 생성 오디오(녹음·Bounce·Consolidated)를 `<Project> Audio/`로 수집**하고 소스 경로를 **상대경로**로 저장해 자기완결·이식가능하게 만든다. **import한 스템(원본)은 수집하지 않고 제자리 참조**(사용자 결정).

### T-1.43.0-1. Save As가 오디오를 수집하고 재열기·이식이 되는가

- **시험 방법 및 절차**
  1. **새 프로젝트**(미저장)에서 스템을 import하고, **Audio In 트랙에 루프/일반 녹음**을 한 뒤, 클립을 **Merge(Consolidate)** 해 병합 소스를 만든다. (녹음·consolidation이 지금은 temp에 있다)
  2. **Save As** → 이름을 정해 저장한다. 저장한 `.focus` 옆에 **`<이름> Audio/` 폴더**가 생기고 그 안에 **`Recordings/`·`Consolidated/`**(Bounce를 만들었다면 `Bounces/`)와 WAV들이 있는지 **파일 탐색기로 확인**한다.
  3. `.focus`를 텍스트 편집기로 열어 수집된 소스의 `filePath`가 **상대경로**(`<이름> Audio/...`)로, import 스템은 **절대경로 그대로**인지 본다.
  4. 앱을 **종료 → 재실행 → 그 프로젝트 열기** → 녹음·병합·스템 오디오가 **전부 정상 재생**되는지 확인한다(무음/소실 없음).
  5. **이식 시험**: 앱을 끄고, 파일 탐색기에서 **`.focus`와 `<이름> Audio/` 폴더를 함께 다른 위치로 이동**한 뒤, 앱에서 이동한 `.focus`를 연다 → 오디오가 여전히 정상 재생되는지 확인한다.
  6. **Save As 재차**: 그 프로젝트를 **또 다른 이름/위치로 Save As** → 새 폴더에도 `<새이름> Audio/`가 생기고 오디오가 복사되며, 재열기 시 정상인지 확인한다. **원본 프로젝트 폴더의 오디오는 그대로 남아 있는지**도 본다.
  7. **일반 Save 무회귀**: 이미 저장된 프로젝트에서 소소한 편집 후 **Save**(다이얼로그 없이) → 정상 저장되고, `<이름> Audio/`에 **중복 파일이 늘어나지 않는지** 확인한다(멱등).
  8. (하위호환) v1.43.0 **이전에 저장한** 프로젝트가 있으면 열어서 정상 재생되는지 확인한다(절대경로 저장분이 그대로 열려야 한다).
- **판정 기준**
  - ① Save As 후 `<이름> Audio/`에 녹음·병합(·Bounce) WAV가 **카테고리 폴더별로 수집**된다.
  - ② `.focus`의 수집 소스는 **상대경로**, import 스템은 **절대경로**로 저장된다.
  - ③ 재열기 시 모든 트랙(녹음·병합·스템)이 **정상 재생**된다(무음/파형 소실 없음).
  - ④ `.focus` + `Audio` 폴더를 **함께 옮겨도** 재열기·재생이 정상이다(이식).
  - ⑤ 다른 위치로 Save As하면 **새 폴더에 복사**되고 **원본은 보존**된다.
  - ⑥ 일반 Save는 **중복 복사 없이**(멱등) 정상 저장된다.
  - ⑦ v1.43.0 이전 저장 프로젝트(절대경로)가 **그대로 열린다**(하위호환).

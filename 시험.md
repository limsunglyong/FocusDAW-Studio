## 2. 사전 준비 및 환경 구성

### 2-A. 브라우저 환경 (Fallback 기능 검증)
1. 프로젝트 루트 경로에서 웹 서버를 구동합니다.
   ```bash
   npx http-server -p 8080
   ```
2. Chrome 또는 Edge 브라우저를 열어 `http://localhost:8080/studio.html`에 접속합니다.
3. 브라우저 설정에서 **팝업 차단이 비활성화**되어 있는지 확인합니다.

### 2-B. Electron 앱 환경
1. 프로젝트 루트 경로에서 아래 명령어로 앱을 실행합니다.
   ```bash
   npm start
   ```

## 5. 2026-06-12 추가 수정분 시험 방법

### [시험 환경 준비]
1. 프로젝트 루트에서 `npm start`로 Electron 앱을 실행합니다.
2. Demo session 또는 실제 오디오 트랙을 불러온 뒤 Mixer 버튼으로 외부 믹서 창을 엽니다.

### [시험항목]
- B-6: 테마 변경 시 열린 믹서 창 색상 즉시 변경.
- B-7: 트랙 추가 시 믹서 창 채널 추가 및 창 가로 크기 증가.
- B-10: 믹서 창에서 `Ctrl+Z` / `Ctrl+Y` 동작.
- B-15, B-16: 믹서 창 Stop / Play 버튼 및 Space 재생 토글.
- B-17: 메인 Top Bar transport 버튼 테두리 잔상 없음.
- B-18: Mixer 상세 파라미터 Undo/Redo가 미세량 단위가 아닌 조작 단위로 동작.
- B-19: 메인 창 Space 입력 시 브라우저 스크롤 기본 동작 차단.
- B-20: Mixer EQ preset 버튼 외곽선 잔상 없음.
- B-21, B-22: Help 독립 창 이동 및 문자열 검색.

### [시험방법 및 절차]
1. Mixer 창을 연 상태에서 Settings > Color Theme를 다른 테마로 바꾸고, Mixer 창 배경/버튼/노브 색상이 즉시 바뀌는지 확인합니다.
2. Mixer 창을 닫지 않은 상태에서 `+ Track`으로 트랙을 추가하고, 새 채널 스트립이 표시되며 Electron Mixer 창 폭이 필요한 경우 자동으로 넓어지는지 확인합니다.
3. Mixer 창에서 볼륨 페이더 또는 팬 노브를 변경한 뒤 `Ctrl+Z`, `Ctrl+Y`를 눌러 값이 되돌아가고 다시 적용되는지 확인합니다.
4. Mixer 창 타이틀바의 Play/Pause 버튼과 Stop 버튼을 누르고, 메인 창 재생 상태와 오디오가 동일하게 반응하는지 확인합니다.
5. Mixer 창에 포커스를 둔 뒤 Space를 눌러 재생/일시정지가 토글되는지 확인합니다.
6. 메인 Top Bar transport 버튼을 반복 클릭하고 포커스/테두리 잔상이 남지 않는지 확인합니다.
7. Mixer 창에서 Master volume, Reverb, Echo/Delay와 트랙 VRB/ECHO/PAN/volume을 각각 크게 움직인 뒤 `Ctrl+Z`, `Ctrl+Y`를 눌러 한 조작 단위로 되돌림/다시 실행되는지 확인합니다.
8. Mixer 창에서 Play 후 메인 앱 top bar를 클릭하고 Space를 여러 번 눌러 화면 스크롤 없이 재생/일시정지만 토글되는지 확인합니다.
9. Mixer 창의 Reset, POP, Classic, HIP HOP 버튼을 반복 클릭하고 외곽선 잔상이 남지 않는지 확인합니다.
10. Help > Manual을 열고 Help 창을 메인 앱 밖으로 이동할 수 있는지 확인합니다.
11. Help 상단 검색창에 `Export`, `Mixer`, `프로젝트` 등의 문자열을 입력하고 결과 카운터, 이전/다음 이동, 강조 표시가 동작하는지 확인합니다.

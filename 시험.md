## 2. 시험 항목

### B-176) 초기 화면 Import Folder 시 폴더명 → 프로젝트 이름 자동 설정 (v1.9.4)

**[시험 환경 준비]**
- Windows Electron 앱 실행(`npm start`). 앱 새로 시작(또는 Edit ▸ New Project)하여 **빈 초기 화면**(트랙 0개, 우상단 프로젝트 이름이 `untitled`) 상태로 둔다.
- 오디오 파일(mp3/wav 등) 2~3개가 들어 있는 테스트 폴더 하나를 준비(예: 폴더명 `MySong`).

**[시험 항목]**
1. 초기 화면에서 폴더 임포트 시 프로젝트 이름이 폴더명으로 바뀌는가.
2. 이미 트랙이 있거나 이름이 바뀐(저장된) 프로젝트에 폴더를 추가 임포트할 때는 이름이 유지되는가.
3. 개별 파일 임포트(Import Files)·드래그드롭은 이름이 바뀌지 않는가.

**[시험 방법 및 절차]**
1. 빈 초기 화면 → 가운데 **Import Folder** 버튼(또는 메뉴 `Import Stem Folder`) 클릭 → `MySong` 폴더 선택.
   - 기대: 트랙들이 로드되고, **우상단 프로젝트 이름과 하단 상태바·창 제목이 `MySong`으로 변경**된다.
2. 위 상태(트랙 있음)에서 다시 다른 폴더(예: `OtherStems`)를 Import Folder. 또는 프로젝트 이름을 수동으로 바꾼 뒤 폴더 임포트.
   - 기대: 트랙은 추가되지만 **프로젝트 이름은 그대로 유지**(폴더명으로 덮어쓰지 않음).
3. 빈 초기 화면에서 **Import Files**로 개별 오디오 파일 선택, 또는 파일/폴더를 타임라인에 드래그드롭.
   - 기대: 트랙은 로드되지만 이름은 `untitled` 유지(폴더명 자동 설정 대상 아님).
4. 1번 직후 앱을 껐다 켜 최근 프로젝트(자동 저장)가 `MySong` 이름으로 복원되는지 확인.

### B-177) 사용자 메뉴얼 업데이트 — Advanced Effects/신규 이펙트/폴더 자동명명/Delete all tracks (v1.9.5)

**[시험 환경 준비]**
- Windows Electron 앱 실행(`npm start`). 데모 세션 또는 스템을 로드해 둔다.
- (선택) 브라우저로 `manual/사용자 메뉴얼.html`을 직접 열어 독립 메뉴얼도 확인.

**[시험 항목]**
1. 인앱 Help ▸ Manual과 독립 HTML 메뉴얼 모두에 **신규 §7 고급 이펙트** 섹션이 보이고 목차/사이드바에 "7. 고급 이펙트"가 추가되어 있는가.
2. 추가된 12개 스크린샷이 모두 깨지지 않고 표시되는가.
3. 한글/English 토글 시 신규 내용이 양쪽 언어로 정상 표시되는가.
4. Ambience 프리셋 표에 각 프리셋(Concert Hall/Home/Far Field/Studio/Tunnel 등)의 **음악적 효과 설명**이 들어 있는가.

**[시험 방법 및 절차]**
1. 상단 메뉴 **Help ▸ Manual**(또는 About 옆 Manual) 열기 → 좌측 사이드바에서 **7. 고급 이펙트** 클릭 → Ambience/Auto Panning/Equalizer 설명과 이미지가 보이는지 확인.
2. 사이드바 번호가 **7 고급 이펙트 → 8 내보내기 → 9 설정 → 10 단축키 → 11 문제 해결** 순으로 밀려 있는지 확인.
3. §3에 "폴더 이름으로 프로젝트 이름 자동 설정", §4에 "Delete all tracks", §6 믹서에 5종 OUTPUT EFFECTS 설명이 추가됐는지 확인.
4. 상단 **English** 토글 → 같은 섹션이 영문으로 바뀌는지, 이미지/표가 동일하게 보이는지 확인.
5. 검색창에 `Ambience`, `Saturation`, `Delete all` 입력 → 신규 문단이 검색·하이라이트되는지 확인.
6. 독립 HTML 메뉴얼(`manual/사용자 메뉴얼.html`)을 브라우저로 열어 목차 링크(#advfx)와 이미지 로드, 한/영 전환을 동일하게 확인.

## 4. 확인 메모

- v1.2.33부터 Vari BPM 실시간 재생은 캐시형 Time Stretch 프리뷰를 우선 사용합니다. 첫 재생 또는 BPM 비율 변경 직후에는 트랙별 프리뷰 버퍼 준비 시간이 있을 수 있습니다.
- v1.2.40부터 Playback BPM 입력은 500ms 대기 후 마지막 값 기준으로 프리뷰 계산을 시작하며, 계산 중 들어간 pending 입력이 뒤늦게 중복 계산을 일으키지 않도록 정리합니다.
- Export `Keep pitch`는 Electron Export 전용 단기 안정 기능입니다. 데스크톱 경로는 `ffmpeg atempo` 기준선을 사용하며, 브라우저 fallback의 JS Time Stretch는 품질 기준 미달 실험 경로로 취급합니다.

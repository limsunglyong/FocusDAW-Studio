## 2. 사전 준비

### 2-A. 브라우저에서 직접 실행 (빠른 시험)

1. `studio.html` 을 Chrome/Edge 에서 직접 열거나, 로컬 정적 서버로 서빙
   ```bash
   # Python 간이 서버 예시 (프로젝트 루트에서)
   python -m http.server 8080
   ```
   → `http://localhost:8080/studio.html` 접속
2. 브라우저 DevTools → **Console** 탭 열어두기 (에러 모니터링)
3. DevTools → **Performance** 탭도 열어두기 (성능 측정용)

### 2-B. Electron 앱으로 실행

```bash
# 별도 Windows Terminal / PowerShell 에서 실행 (Claude 내부 터미널 ❌)
npm start
```

> **주의**: `npm start`는 반드시 별도 터미널에서 실행할 것.  
> Claude Code 내부 터미널에서 실행하면 `ELECTRON_RUN_AS_NODE=1` 상속으로 실패한다.

## 7. 알려진 한계 / 시험 시 유의사항

- **Loop OFF 정지 타이밍**: `_loop()` 콜백은 ~100ms 간격으로 실행되므로 duration 도달 후 최대 100ms 후에 `stop()`이 호출될 수 있음. 즉각 정지가 아닌 ~100ms 이내 정지는 정상 동작.
- **IntersectionObserver 최소 지연**: 옵저버가 비동기로 동작하므로 트랙이 뷰포트에 들어오는 순간 극히 짧은 빈 프레임이 보일 수 있음. 200ms 이내라면 정상 동작.
- **scrollLeft 클램핑 timing**: `requestAnimationFrame` 기반이므로 줌 변경 후 1프레임(~16ms) 후에 클램핑 적용됨.

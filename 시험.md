## 추후 확인 항목 (보류·장기 관찰) — 버전 무관

### T-1.20.10-2(b). 입력 미사용 프로필에서 시작 시 마이크 미개방
- ⏰ **시험 시점: Phase 6 Stage 5 착수 직전**(위 "예약된 시험 알림" 참조). 그 전까지는 회귀 위험이 없어 대기.
- ⚠️ 사전 고지: 준비 단계에서 `focusdaw-audio-input` 키를 지우므로 **그동안 맞춰둔 입력 장치 선택이 초기화된다.** 시험 후 Settings ▸ Audio Devices에서 다시 선택하면 된다.
- 시험 환경 준비: `npm start`로 실행 후 DevTools(F12 / Ctrl+Shift+I) 콘솔에서 `localStorage.removeItem('focusdaw-audio-input')` 실행 → 앱을 완전히 종료 후 콜드 재시작한다. (= 입력을 한 번도 쓴 적 없는 프로필 재현. 이 키는 `+` Audio In·arm·녹음·Settings 입력 선택 시 생성된다.)
- 시험 내용: 재시작 직후 Audio In 추가·ARM·녹음을 하지 않은 상태에서 OS 마이크 사용 표시등(Windows 11: 작업 표시줄 마이크 아이콘 / 설정 ▸ 개인정보 및 보안 ▸ 마이크 ▸ 최근 활동)을 관찰한다.
- 판정 기준: 입력 이력이 없는 프로필에서는 시작 시 마이크가 **열리지 않는다**. (대조: Audio In을 한 번 추가해 키를 재생성한 뒤 콜드 재시작하면 이번엔 시작 시 마이크가 열려야 정상 — 이력 있는 프로필의 유휴 워밍업.)
- 비고: T-1.20.10-1 및 T-1.20.10-2(a) arm/녹음은 시험 완료(시험-아카이브.md).

### T-1.20.8-1. 앱 종료 시 엔진 비정상 종료 코드 재발 관찰 (장기 tracing)
- 시험 내용: 평소처럼 사용하며 앱을 시작/종료할 때(믹서 창을 열어 둔 채 종료 포함) 콘솔/로그의 `[AudioEngine] Process exited with code ...` 메시지를 관찰한다. 단발성 경합이므로 일정 기간 반복 관찰이 필요하다.
- 판정 기준: 종료 코드가 항상 정상(1 = Electron kill, 또는 0)이고, `3221225477`(0xC0000005) 등 비정상 코드가 재발하지 않는다. 재발 시 `%LOCALAPPDATA%\CrashDumps`의 새 덤프 유무를 함께 확인해 보고한다.

### T-1.20.8-1. 앱 종료 시 엔진 비정상 종료 코드 재발 관찰 (장기 tracing)
- 시험 내용: 평소처럼 사용하며 앱을 시작/종료할 때(믹서 창을 열어 둔 채 종료 포함) 콘솔/로그의 `[AudioEngine] Process exited with code ...` 메시지를 관찰한다. 단발성 경합이므로 일정 기간 반복 관찰이 필요하다.

## v1.40.4 — Repeat/Punch 하이라이트 테마 적용 (2026-07-20)

### T-1.40.4-1. Repeat 구간 하이라이트가 컬러 테마를 따라간다
> 색상 변경이라 자동 검증이 불가능하다 — 육안 확인이 필요하다.

- **시험 방법 및 절차**
  1. 타임라인에서 **구간을 지정**하고 **Repeat ON**으로 켠다(구간이 액센트 색으로 강조된다).
  2. Settings에서 **컬러 테마를 Modern Blue로** 바꾼다.
  3. 이어서 **Forest Green**, **Classical Ivory**로도 바꿔 본다.
  4. 각 테마에서 **① 타임라인 Repeat 구간 ② 상단 미니 타임라인의 구간 ③ OUTPUT FX 레인의 `Repeat ON` 버튼 ④ `PUNCH` 글자**(Punch ON일 때)를 본다.
- **판정 기준**
  - ① 네 곳 모두 **테마 액센트 색**을 따라간다(파란 테마=파랑, 초록 테마=초록). 혼자 노란색으로 남으면 실패.
  - ② Repeat ON/OFF 전환 시 진하기 차이가 종전처럼 유지된다.
  - ③ 구간 테두리와 채움의 **색이 서로 어긋나지 않는다**(수정 전에는 테두리만 테마를 따랐다).
  - ④ 밝은 테마(Classical Ivory)에서도 구간이 배경과 구분되어 보인다.

> ⚠️ 앱의 **다른 곳 38군데**는 아직 하드코딩 상태다(클립 선택 하이라이트, `N takes` 배지, EQ 곡선 등). 이번 시험 범위는 **Repeat/Punch 관련 4곳**이며, 나머지가 노란색으로 남아 있는 것은 **정상(미수정)** 이다. 상세 [앱개발.md](앱개발.md) "알려진 버그 / 한계".

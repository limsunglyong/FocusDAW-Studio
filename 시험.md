## 2. 시험 항목

### B-195 · Dist 패키지에서 BPM·Key 감지 동작 (v1.14.2)

**배경**: 패키징 누락으로 `meyda.min.js`가 asar에 포함되지 않아, dist 버전에서만 BPM·Key 감지가 동작하지 않던 문제 수정. `package.json` `build.files`에 meyda 추가 + Meyda 미로드 시 콘솔 경고 추가.

#### [시험 환경 준비]
1. 소스에서 dist 패키지를 새로 생성: `npm run dist:win`
2. 생성물: `dist/win-unpacked/` (또는 설치본). **반드시 새로 빌드한 dist를 사용** (기존 1.14.1 dist 아님).
3. BPM/Key가 들어있는 곡 stem 오디오 파일 1세트 준비.

#### [시험 항목 1] 패키징 포함 확인 (정적)
- **방법**: `npx asar list dist/win-unpacked/resources/app.asar` 출력에 `\meyda.min.js`가 보이는지 확인.
- **합격 기준**: 목록에 `meyda.min.js`가 포함됨.

#### [시험 항목 2] Dist 앱에서 BPM 감지
- **절차**: `dist/win-unpacked/FocusDAW Studio.exe` 실행 → 곡 import → BPM 설정창에서 Detect.
- **합격 기준**: BPM 값이 정상 산출됨(`---` 또는 무반응이 아님). `npm start`와 동일한 값.

#### [시험 항목 3] Dist 앱에서 Key 감지
- **절차**: 같은 곡에서 Key 설정창 → Detect.
- **합격 기준**: 원 Key가 정상 표시됨(항상 "C"로 고정되거나 무반응이 아님). `npm start`와 동일.

#### [시험 항목 4] (회귀 방지) Meyda 미로드 경고
- **방법**: (개발 환경에서) studio.html의 meyda 스크립트를 임시로 비활성화하거나 누락 상황을 가정해 DevTools 콘솔 확인.
- **합격 기준**: `[FocusDAW] Meyda 라이브러리가 로드되지 않아 ... 감지를 사용할 수 없습니다 ...` 경고가 1회 출력. (선택 시험)

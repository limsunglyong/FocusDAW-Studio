### B-213 — 폴더 drag&drop 오디오 import 및 프로젝트 제목 조건 확인 (v1.17.13)

#### 시험 항목
- Arrange 영역에 오디오 파일이 들어 있는 폴더를 drag&drop하면 폴더 루트의 오디오 파일만 트랙으로 생성되는지 확인.
- 트랙이 하나도 없고 프로젝트 제목이 `untitled`인 상태에서 폴더를 drag&drop하면 프로젝트 제목이 폴더명으로 변경되는지 확인.
- 기존 오디오 트랙이 하나라도 있는 상태에서 폴더를 drag&drop하면 오디오 트랙만 추가되고 프로젝트 제목은 변경되지 않는지 확인.
- 프로젝트 제목이 `untitled`가 아닌 상태에서 폴더를 drag&drop하면 오디오 트랙만 추가되고 프로젝트 제목은 변경되지 않는지 확인.
- 폴더 하위 폴더 안의 오디오 파일은 자동으로 import되지 않는지 확인.

#### 기대 결과
- 루트 오디오 파일만 새 트랙으로 추가된다.
- 프로젝트 제목은 초기 `untitled` + 트랙 없음 + 미저장 프로젝트 조건에서만 폴더명으로 변경된다.
- 이미 트랙이 있거나 사용자가 프로젝트 제목을 바꾼 경우 제목이 유지된다.

### B-212 — Release Notes 수직 스크롤바 테마 색상 확인 (v1.17.12)

#### 시험 항목
- Release Notes 창의 수직 스크롤바가 앱 color theme에 맞는 색상으로 표시되는지 확인.
- Settings에서 테마를 바꿨을 때 Release Notes 스크롤 thumb/hover 색상이 어색하지 않게 따라가는지 확인.
- 기존 메인 앱 스크롤바와 비교했을 때 폭, radius, 투명 track 느낌이 일관적인지 확인.

#### 시험 방법 및 절차
1. Help > `Release Notes`를 연다.
2. 창 본문을 스크롤해 수직 스크롤바 thumb 색상을 확인한다.
3. Settings에서 다른 color theme를 적용한 뒤 Release Notes를 다시 열어 색상 변화가 자연스러운지 확인한다.
4. 스크롤 hover 상태도 확인한다.

#### 시험 환경 준비
- 앱 실행 환경.

### B-211 — Help > Release Notes 메뉴 확인 (v1.17.11)

#### 시험 항목
- Help 메뉴에 `Release Notes` 항목이 표시되는지 확인.
- Release Notes 창이 앱 스타일의 모달로 열리고, 로고/제목/버전 범위/날짜/섹션/OK 버튼이 정상 표시되는지 확인.
- 내용이 `v1.17.4 - v1.17.10` 범위의 변경 사항 중심으로 구성되어 있는지 확인.

#### 시험 방법 및 절차

2. `Release Notes`를 클릭한다.
3. 창의 로고가 `assets/logo.png` 기반으로 보이고, 폰트/색상이 앱의 About/Help 스타일과 어울리는지 확인한다.
4. `New Features`, `Improvements`, `Fixes` 섹션 내용을 확인한다.
5. OK, X, 바깥 영역 클릭으로 닫힘 동작을 확인한다.

#### 시험 환경 준비
- 앱 실행 환경.

### B-210 — `NO AUDIO` 트랙 BPM/Mute/Solo 버튼 비활성화 확인 (v1.17.10)

#### 시험 항목
- `NO AUDIO` 트랙의 BPM Source, Solo, Mute 버튼이 모두 비활성화되는지 확인.
- 비활성화된 버튼을 클릭해도 트랙 상태, 재생 상태, 다른 트랙의 Mute/Solo 표시가 변하지 않는지 확인.
- 정상 오디오 트랙의 BPM Source, Solo, Mute 버튼은 기존처럼 동작하는지 확인.

#### 시험 방법 및 절차
1. 오디오 파일 일부가 누락된 `.focus` 프로젝트를 열어 `NO AUDIO` 트랙을 만든다.
2. 해당 트랙 헤더의 `B`, `S`, `M` 버튼이 흐리게 표시되고 클릭되지 않는지 확인한다.
3. `NO AUDIO` 트랙의 버튼 영역을 여러 번 클릭해도 다른 트랙의 소리/표시가 바뀌지 않는지 확인한다.
4. 정상 오디오 트랙에서 `B`, `S`, `M` 버튼이 여전히 정상 동작하는지 확인한다.

#### 시험 환경 준비
- 하나 이상의 누락 오디오 트랙이 있는 `.focus` 프로젝트.
- 정상 오디오 트랙이 함께 있는 프로젝트.

# FocusDAW Studio 시험(Test) 계획서

> 최종 갱신: 2026-07-11
> 상태: 수동 확인 대기.

> 공통 환경 준비: 데스크톱 앱 + 네이티브 엔진(`build_native.bat`로 재빌드된 `bin/FocusDAW-AudioEngine.exe`).

---

## 1. 시험 항목

## v1.20.4 — 녹음 중 마우스 seek 차단

### T-1.20.4-1. 녹음 중 마우스 seek 무시
- 시험 내용: 녹음 중(카운트인 포함)에 상단 눈금자, 파일 트랙 레인, Audio In 트랙 레인, Output FX 트랙을 각각 클릭/드래그한다.
- 판정 기준: 플레이헤드가 이동하지 않고 녹음이 계속된다. 키보드(←/→/`,`/`.`/0/Space)도 기존처럼 무시된다.

### T-1.20.4-2. 정지/재생 상태 seek 정상 (회귀 없음)
- 시험 내용: 녹음이 아닐 때(정지·일반 재생) 같은 위치들을 클릭/드래그한다.
- 판정 기준: 기존처럼 플레이헤드가 정상 이동한다.

## v1.20.3 — 카운트다운 폰트 및 설정 창 장치 select 화살표

### T-1.20.3-1. 녹음 카운트다운 폰트/크기
- 시험 내용: 정지 상태에서 Audio In 트랙을 Arm하고 Record를 누른다.
- 판정 기준: 중앙 카운트다운 숫자가 **Orbitron, normal 굵기**로 표시되고, 크기가 기존의 약 절반(11vmin)이다.

### T-1.20.3-2. AUDIO DEVICES 드롭다운 화살표
- 시험 내용: Settings의 AUDIO DEVICES에서 긴 이름의 Input/Output Device를 선택한 상태로 드롭다운을 본다.
- 판정 기준: 장치명이 다운 화살표를 침범하지 않고 말줄임(…) 처리되며, 화살표가 온전히 보인다.

## v1.20.2 — 입력 포트 동적 채널명(인라인·믹서 일관) 및 모노 모니터링 센터링

> 환경 준비: 데스크톱 앱 + 네이티브 엔진 + 마이크(가능하면 채널명이 있는 ASIO/멀티채널 인터페이스). Audio In 트랙 1개 준비.

### T-1.20.2-1. 입력 포트 동적 채널명 — 3곳 UI 일치
- 시험 내용: ① Audio In 트랙(전체 높이)의 포트 select, ② 트랙 높이를 medium으로 줄였을 때의 인라인 포트 select, ③ 믹서 창의 Audio In 채널 포트 select를 각각 연다.
- 판정 기준: 세 곳 모두 인터페이스의 실제 입력 채널명(예: "Analogue 1", "Analogue 1 + Analogue 2")이 동일하게 표시된다. 장치가 열리기 전에는 세 곳 모두 기본 목록("Input 1/2/1-2")으로 동일하게 폴백한다.

### T-1.20.2-2. 믹서 창 채널명 갱신
- 시험 내용: 믹서 창을 닫았다가 다시 연다(장치 연결 후).
- 판정 기준: 믹서 포트 select에 채널명이 즉시 반영된다(INIT_STATE 전달).

### T-1.20.2-3. 모노 입력 모니터링 좌우 중앙 출력
- 시험 내용: Audio In 트랙을 Arm하고 MON(모니터링)을 켠 후 마이크에 소리를 낸다.
- 판정 기준: 헤드폰/양쪽 스피커 모두에서 입력 오디오가 좌우 대칭(중앙)으로 들린다.

## v1.20.1 — 자동 녹음 종료 시 Output FX/타임라인 길이 유지

> 환경 준비: 데스크톱 앱 + 네이티브 엔진 + 마이크. 파일 트랙이 있는 프로젝트에서 Audio In 트랙을 준비한다.

### T-1.20.1-1. 곡 끝 자동 녹음 종료 후 타임라인 길이 유지
- 시험 내용: 정지 상태에서 Audio In 트랙을 Arm하고 Record를 눌러 count-in 후 녹음한다. 재생이 기존 파일 트랙 끝에 도달해 자동 stop될 때까지 기다린다.
- 판정 기준: 녹음 자동 stop 후 Output FX 트랙 길이와 상단 타임라인 총 길이가 녹음 전 곡 길이와 동일하다. 끝에 60초가 추가되지 않고, 타임라인을 정상적으로 앞/뒤로 이동할 수 있다.

### T-1.20.1-2. 수동 긴 녹음의 타임라인 확장 유지
- 시험 내용: 기존 파일 트랙이 없는(곡 끝 제한이 없는) 상태에서 Audio In을 수동으로 길게 녹음한다.
- 판정 기준: 수동 녹음은 기존처럼 녹음 진행에 맞춰 타임라인이 확장되고, 정지 후 녹음 클립 전체가 보존된다.

## v1.20.0 — 녹음 트랜스포트 규칙 (카운트인·Repeat off·자동 종료·pause 무시)

> 환경 준비: 데스크톱 앱 + 네이티브 엔진 + 마이크. Audio In 트랙 1개를 Arm.

### T-1.20.0-1. 정지 상태 Record → 3초 카운트인
- 시험 내용: 재생 정지 상태에서 Record 버튼을 누른다.
- 판정 기준: 화면 중앙에 **3→2→1 카운트다운**이 표시된 뒤 현재 위치에서 **재생+녹음이 동시에** 시작된다.

### T-1.20.0-2. 재생 중 Record → 즉시 녹음
- 시험 내용: 재생 중에 Record를 누른다.
- 판정 기준: 카운트인 없이 **즉시 녹음** 시작(재생 계속).

### T-1.20.0-3. Repeat off/복원
- 시험 내용: Repeat(Loop)를 켠 상태에서 Record → 녹음 → Stop.
- 판정 기준: Record 시 **Loop 버튼이 꺼지고**, 녹음 종료 후 **다시 켜진다**(이전 상태 복원).

### T-1.20.0-4. 곡 끝 자동 종료
- 시험 내용: 파일 트랙(반주)이 있는 상태에서 녹음한다.
- 판정 기준: 재생이 **가장 긴 트랙의 끝**에 도달하면 녹음+재생이 **자동 stop**된다. (반주가 없으면 수동 Stop 전까지 계속)

### T-1.20.0-5. 녹음 중 pause/return-to-start 무시
- 시험 내용: 녹음 중 Play/Pause 버튼, Return-to-start 버튼(및 Space/0 키)을 누른다.
- 판정 기준: **아무 동작 안 함**(녹음 계속). Stop 또는 Record만 녹음을 멈춘다.

### T-1.20.0-6. 카운트인 취소
- 시험 내용: 3초 카운트인 중에 Record 또는 Stop 또는 Play를 누른다.
- 판정 기준: **카운트인과 녹음이 취소**되고(오버레이 사라짐), Repeat도 원래대로 복원된다.

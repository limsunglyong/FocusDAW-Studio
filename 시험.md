## v1.43.20 — 클립별 볼륨(clip gain) 라인 UI

> 대상: Audio In / Bounce 트랙에서 여러 클립으로 나뉜 상태. 클립을 클릭해 선택하면 클립 상단에 앰버색 수평 볼륨 라인이 나타난다. 라인을 **아래로 드래그**하면 볼륨이 줄고(컷 전용, 상단=0 dB, 하단=음소거), 드래그 중 dB 라벨이 뜨며, 놓으면 파형이 그만큼 작아진다.

### T-1.43.20-1. 클립 볼륨 라인 드래그 — 감쇠·파형·재생 반영
- 시험 내용: Audio In 트랙에서 클립 하나를 선택하고 상단 볼륨 라인을 아래로 드래그해 예: 약 −6 dB로 내린다. 놓은 뒤 파형을 확인하고, 그 구간을 재생해 소리 크기를 다른(감쇠 안 한) 클립과 비교한다.
- 판정 기준: 드래그 중 dB 라벨이 실시간 표시되고, 놓으면 해당 클립의 **파형 진폭이 눈에 띄게 축소**되며, 재생 시 그 구간만 볼륨이 낮아진다. 다른 클립·트랙 볼륨은 변하지 않는다.

### T-1.43.20-2. 저장/재열기 후 클립 볼륨 유지
- 시험 내용: 여러 클립 중 일부의 볼륨을 서로 다르게 낮춘 뒤 프로젝트를 저장하고, 앱을 재시작해 그 프로젝트를 다시 연다.
- 판정 기준: 재열기 후에도 각 클립의 볼륨(라인 위치·파형 진폭·재생 크기)이 저장 당시와 동일하다. `read-audio-file` 등 콘솔 에러가 없다.

### T-1.43.20-3. Undo / Redo
- 시험 내용: 클립 볼륨을 한 번 내린 뒤 Ctrl+Z(Undo), 이어서 Ctrl+Y(Redo)를 한다. 여러 클립을 차례로 조정한 뒤 여러 번 Undo도 시험한다.
- 판정 기준: Undo 1회가 **직전 볼륨 드래그 1건만** 되돌리고(라인·파형·소리 원복), Redo가 다시 적용한다. 드래그 1회 = Undo 1개(중간 mousemove가 여러 undo를 쌓지 않는다).

### T-1.43.20-4. Export 반영 (웹/네이티브 A/B)
- 시험 내용: 클립 볼륨을 낮춘 프로젝트를 WAV로 Export한다(네이티브 경로). 필요 시 웹 fallback 경로와 비교한다.
- 판정 기준: Export 결과물에서 감쇠한 클립 구간의 레벨이 화면 재생과 일치한다(웹·네이티브 결과가 청감/레벨상 동일).

### T-1.43.20-5. 적용 대상 제한 — 파일 트랙 불가, 이동/트림 무회귀
- 시험 내용: 파일(file) 트랙 클립에는 볼륨 라인이 없음을 확인한다. Audio In/Bounce 클립에서 볼륨 라인 도입 후에도 클립 **이동(본체 드래그)·트림(좌우 가장자리)·다중선택 이동**이 정상 동작하는지 확인한다.
- 판정 기준: 파일 트랙에는 볼륨 라인이 나타나지 않는다. 볼륨 라인(상단 8px, 좌우 8px inset)과 이동/트림 조작이 서로 간섭하지 않고 각각 의도대로 동작한다.

### T-1.43.20-6. 음소거 경계(하단) 및 부스트 없음
- 시험 내용: 볼륨 라인을 클립 최하단까지 내려 본다. 라인을 위로 올려 0 dB(상단)를 넘겨 부스트가 되는지 확인한다.
- 판정 기준: 최하단에서 해당 클립이 **무음(-∞)**이 된다. 상단(0 dB)이 최대이고 그 위로는 올라가지 않는다(부스트 없음, 컷 전용).

## v1.43.18 — Clean Up Unused Recordings 명령 (정리 도구)

> 대상: 저장된 프로젝트. Project ▸ Clean Up Unused Recordings…를 실행하면 `<Project> Audio/{Recordings,Bounces,Consolidated}/`에서 프로젝트·undo 스택이 참조하지 않는 WAV를 찾아 앱 테마 모달로 목록을 보여주고, 확인 시 **휴지통으로 이동**한다.

### T-1.43.18-1. 미사용 파일 정리 → 휴지통 이동
- 시험 내용: 녹음/Take를 여러 개 만든 뒤 일부를 Flatten 또는 삭제해 참조되지 않는 WAV가 생기게 한다(저장 상태). Project ▸ Clean Up Unused Recordings… 실행 → 목록 확인 → Move to Recycle Bin.
- 판정 기준: 참조되지 않는 WAV만 목록에 뜨고(이름·카테고리·용량·총량 표시), 확인 시 해당 파일이 **Windows 휴지통으로 이동**한다(영구 삭제 아님, 복구 가능). 현재 사용 중인 녹음/bounce는 목록에 없다.

### T-1.43.18-2. Undo 스택 참조 파일 제외
- 시험 내용: 녹음을 삭제(또는 Flatten으로 drop)한 **직후, Undo로 되돌릴 수 있는 상태에서** Clean Up을 실행한다.
- 판정 기준: 방금 삭제/드롭해 undo로 되살아날 수 있는 WAV는 **정리 목록에서 제외**된다(삭제되지 않음). 이후 Undo가 정상 동작한다.

### T-1.43.18-3. 미저장 프로젝트 안내
- 시험 내용: 저장하지 않은(새) 프로젝트에서 Clean Up Unused Recordings…를 실행한다.
- 판정 기준: "먼저 저장하라"는 앱 테마 안내가 뜨고, 아무 파일도 지우지 않는다.

## 추후 확인 항목 (보류·장기 관찰) — 버전 무관

### T-1.20.8-1. 앱 종료 시 엔진 비정상 종료 코드 재발 관찰 (장기 tracing)
- 시험 내용: 평소처럼 사용하며 앱을 시작/종료할 때(믹서 창을 열어 둔 채 종료 포함) 콘솔/로그의 `[AudioEngine] Process exited with code ...` 메시지를 관찰한다. 단발성 경합이므로 일정 기간 반복 관찰이 필요하다.
- 판정 기준: 종료 코드가 항상 정상(1 = Electron kill, 또는 0)이고, `3221225477`(0xC0000005) 등 비정상 코드가 재발하지 않는다. 재발 시 `%LOCALAPPDATA%\CrashDumps`의 새 덤프 유무를 함께 확인해 보고한다.

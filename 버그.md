

문제 현상 : 

BPM 변경 후 export 시 정상 적으로 녹음되지 않고 프리뷰(실시간)보다 빠르게 녹음이 되며 플레이 시간이 짧아져야 하지만 원곡의 시간만큼 저장되고 남은 시간에는 다시 노래가 처음부터 변경된 bpm보다 훨씬 빨라진 속도로 녹음이 되는 현상을 찾아봐. 원인만 분석해서 보고해줘.

------------------------------------------------------------------------------------------------





원인만 정리하면, 버그의 중심은 네이티브 export 경로가 BPM 변경 후의 출력 길이를 쓰지 않고 원곡 길이로 렌더링하는 것입니다.
Export UI는 BPM 변경 시 길이를 DAW.duration / tempoRate로 계산해서 보여줍니다.
[ui-dialogs.jsx (line 479)](E:/programmings/FocusDAW-Studio/ui-dialogs.jsx:479)

그런데 네이티브 export 요청은 실제로는 항상 duration: LocalDAW.duration을 보냅니다. 즉 BPM이 빨라져도 원곡 길이만큼 렌더링합니다.
[audio-bridge.js (line 311)](E:/programmings/FocusDAW-Studio/audio-bridge.js:311)

JUCE export 루프도 전달받은 durationSeconds * targetSampleRate만큼 끝까지 씁니다. 그래서 출력 파일 길이가 변경 BPM 기준으로 짧아지지 않고 원곡 길이로 고정됩니다.
[AudioEngine.cpp (line 748)](E:/programmings/FocusDAW-Studio/juce_skeleton/Source/AudioEngine.cpp:748)

동시에 BPM 비율 자체는 네이티브 엔진에 적용됩니다. playbackBpm / projectBpm이 targetTempo로 들어가고 각 트랙의 SoundTouch에 전달됩니다.
[AudioEngine.cpp (line 550)](E:/programmings/FocusDAW-Studio/juce_skeleton/Source/AudioEngine.cpp:550)

남은 시간에 곡이 처음부터 다시 나오는 이유는 네이티브 트랙 로딩 때 trackSource->setLooping(true)가 걸려 있기 때문입니다. 빨라진 BPM 때문에 실제 곡 내용은 먼저 끝나는데, export 루프는 원곡 길이까지 계속 돌고, source가 loop 상태라 처음부터 다시 읽힙니다.
[AudioEngine.cpp (line 193)](E:/programmings/FocusDAW-Studio/juce_skeleton/Source/AudioEngine.cpp:193)

부가 원인으로, 실시간 프리뷰는 기본적으로 preservePitch=true인 SoundTouch tempo 경로인데, export는 다이얼로그의 Keep pitch 값에 따라 setTempo 또는 setRate가 갈립니다. Keep pitch가 꺼져 있으면 export 쪽은 피치까지 올라가는 setRate 경로라 실시간 프리뷰보다 더 빠르거나 급하게 들리는 착각이 생길 수 있습니다.
[AudioEngine.h (line 230)](E:/programmings/FocusDAW-Studio/juce_skeleton/Source/AudioEngine.h:230)
결론: 네이티브 export에서 BPM 변경 후 목표 길이(duration / tempoRate)를 쓰지 않고 원곡 길이를 렌더링하며, 트랙 loop가 켜져 있어 남은 구간에 처음부터 다시 재생되는 구조적 버그입니다. 코드는 수정하지 않았습니다.

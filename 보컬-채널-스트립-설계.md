# 보컬 채널 스트립 — 설계 문서

> 작성: 2026-07-24 · 상태: **설계(착수 전)** · 관련: [(승룡)해야할일.txt](<(승룡)해야할일.txt>), [앱개발.md](앱개발.md) "남은 항목 및 계획"
>
> **목적** — SUNO stem(MR) 위에 사용자가 보컬을 녹음해 완성본을 만드는 워크플로에서, **스템 트랙과 보컬 트랙의 이펙트를 분리**하고 보컬 트랙에 **채널 스트립**(HPF·Noise Gate·EQ·Compressor·De-esser + 기존 reverb/echo)과 **오프라인 광대역 De-noise**를 제공한다.

## 사용자 결정 (2026-07-24)

- **이펙트 범위**: 풀 채널 스트립 — **EQ + Compressor + Noise Gate + HPF + De-esser + 광대역 De-noise(오프라인)** 전부.
- **모니터링**: **재생 경로만**(1단계). 실시간 입력 모니터에 이펙트 거는 것은 범위 밖(추후 별도).
- **착수 방식**: 설계 문서 먼저(이 문서) → 단계별 구현.

## 1. 현재 이펙트 구조 (착수 기준선)

| 위치 | 이펙트 | 대상 | 코드 |
|------|--------|------|------|
| 트랙별 | reverb send, echo(FeedbackDelay) | **모든 트랙** | 웹 [audio-engine.js:776-790](audio-engine.js#L776-L790) · 네이티브 [AudioEngine.h:771-796](juce_skeleton/Source/AudioEngine.h#L771-L796) |
| 마스터(Output FX) | 9밴드 EQ, reverb, echo, widener, saturation, exciter, room, fade | **전체 믹스** | [audio-engine.js:619-714](audio-engine.js#L619-L714) · [AudioEngine.h:1070+](juce_skeleton/Source/AudioEngine.h#L1070) |

핵심:
- **트랙별 reverb/echo는 이미 웹/네이티브 양쪽에 있다**(수치 일치). 보컬용 reverb/echo는 신규가 아니라 **UI 노출 + 보컬 프리셋** 문제.
- **Compressor·per-track EQ·Gate·De-esser·De-noise는 전무.** 이게 실제 신규 개발.
- 트랙에 `kind`(`file`/`audioIn`/`bounce`) 구분이 이미 있다([audio-engine.js:795](audio-engine.js#L795)) → **kind 게이팅으로 "스템엔 없고 보컬에만 있는 이펙트"가 구조적으로 자연스럽다.**
- **저장은 `params: { ...t.params }` 통째 직렬화**([app.jsx:1422](app.jsx#L1422)) → **새 파라미터는 저장/복원 자동.** 구프로젝트 호환은 로드 시 `?? 기본값`으로 채우면 끝.

## 2. 인서트 vs 센드 구분

- **인서트(직렬)** — 신호를 순서대로 통과: HPF → Gate → EQ → Compressor → De-esser. **신규.**
- **센드(병렬)** — 원음과 섞음: reverb, echo. **이미 존재**(트랙별). 채널 스트립의 맨 끝(fader 뒤 센드)에 그대로 둔다.

### 보컬 신호 체인(고정 순서, v1)

```
[보컬 클립/소스]
  → HPF (럼블/플로시브, 80~120Hz)
  → Noise Gate / Expander (구간 무음)
  → EQ (보컬 3~4밴드)
  → Compressor (다이내믹)
  → De-esser (치찰음)
  → [fader / pan]           ← 인서트는 pre-fader
  → reverb send / echo send (기존, 병렬)
  → 마스터 버스
```

게이트→EQ→컴프→디에서는 보컬 채널 스트립의 정석. v1은 **고정 순서**(재정렬 UI는 추후). 인서트는 **pre-fader**(표준).

## 3. 데이터 모델

`track.params`에 보컬 FX 상태를 추가한다. 통째 직렬화라 저장 자동. 예:

```js
params: {
  // ...기존...
  vocalFx: {
    enabled: false,          // 채널 스트립 전체 bypass (A/B)
    hpf:   { on: false, freq: 90 },
    gate:  { on: false, threshold: -45, ratio: 4, attack: 2, release: 120 },
    eq:    { on: false, bands: [ /* freq/gain/Q */ ] },
    comp:  { on: false, threshold: -18, ratio: 3, attack: 10, release: 120, makeup: 0 },
    deEss: { on: false, freq: 6500, threshold: -24, amount: 0.5 },
  },
  denoise: { printed: false, profile: null }, // 오프라인 De-noise 결과 메타
}
```

- **kind 게이팅**: `vocalFx`는 `kind === 'audioIn' || 'bounce'`에서만 UI 노출·엔진 적용. `file`(스템)은 무시.
- **구프로젝트 로드**: `params.vocalFx ??= 기본값`으로 채운다(hydrate 경로).
- 각 이펙트에 개별 `on` + 스트립 전체 `enabled`(A/B용 bypass).

## 4. 엔진 구현

### 4-1. 웹(fallback) — [audio-engine.js](audio-engine.js)

- 현재 그래프: `source → fader → autoGain → panner → (meter, dry, reverbSend, echoSend)`.
- 인서트를 **source와 fader 사이**에 삽입. `_addTrack`에서 보컬 트랙이면 인서트 노드 생성:
  - HPF: `BiquadFilterNode(type=highpass)`
  - EQ: `BiquadFilterNode(peaking)` 체인 — **마스터 EQ 코드 재사용**([audio-engine.js:619-627](audio-engine.js#L619-L627)).
  - Compressor: `DynamicsCompressorNode`(내장).
  - Gate: WebAudio에 게이트 노드 없음 → 간이 구현(엔벌로프 팔로워 + gain) 또는 v1은 네이티브 권위·웹은 근사(gate 생략 가능).
  - De-esser: HF 사이드체인 → 간이는 고역 `DynamicsCompressorNode`.
- **원칙 6**(웹은 fallback, 네이티브가 가청 권위) 적용 → 웹은 **근사 허용**. 단 파형/레벨이 크게 어긋나지 않을 만큼.

### 4-2. 네이티브(가청, 권위) — [AudioEngine.h](juce_skeleton/Source/AudioEngine.h) `TrackAudioSource`

- `getNextAudioBlock`에서 **transportSource 블록을 뽑은 직후, echo/reverb 센드([AudioEngine.h:771](juce_skeleton/Source/AudioEngine.h#L771)) 전에** 인서트 적용:
  1. HPF — `dsp::IIR::Filter` (highpass)
  2. Gate — 임계값 기반 직접 구현(가벼움, 블록 RMS/피크 → gain 엔벌로프)
  3. EQ — `dsp::IIR::Filter` peaking ×N (**마스터 EQ의 `dsp::IIR` 패턴 재사용**, [AudioEngine.h:1102-1106](juce_skeleton/Source/AudioEngine.h#L1102))
  4. Compressor — `dsp::Compressor<float>`
  5. De-esser — 대역 분리(HF 밴드패스) + comp
- 파라미터 전달: `TrackDef`에 필드 추가([AudioEngine.h:140-141](juce_skeleton/Source/AudioEngine.h#L140) reverbSend/echoSend 옆) + `TrackAudioSource`에 `std::atomic` 추가([AudioEngine.h:1053-1054](juce_skeleton/Source/AudioEngine.h#L1053)) + `setTrackParam` 디스패치 확장([AudioEngine.cpp:911,942](juce_skeleton/Source/AudioEngine.cpp#L911)).
- **재빌드 필요**(네이티브). 증분 빌드로 `bin/FocusDAW-AudioEngine.exe` 교체.

### 4-3. 브리지 — [audio-bridge.js](audio-bridge.js)

- `setTrackParam`([audio-bridge.js:473](audio-bridge.js#L473))에 새 키 전달(중첩이면 `vocalFx.comp.threshold` 식 평탄화 또는 전용 커맨드).
- **네이티브 (재)연결 시 파라미터 재동기화 목록**([audio-bridge.js:1551-1558](audio-bridge.js#L1551-L1558))에 보컬 FX 키 추가 — 안 하면 native 재접속 때 보컬 이펙트가 초기화됨(기존 reverb/echo와 동일 함정).

### 4-4. Export / 오프라인 렌더 — [audio-engine.js](audio-engine.js) `exportMix`

- 오프라인 그래프도 **인서트 체인을 동일 복제**해야 Export ≠ 재생이 안 생긴다. 현재 오프라인은 per-track reverb/echo를 복제한다([audio-engine.js:4181-4183](audio-engine.js#L4181-L4183)) — **같은 자리에 인서트도 추가.**
- 네이티브 Export도 재생과 같은 `TrackAudioSource` 경로를 타므로 자동 반영(설계상). 웹/네이티브 A/B 시험 항목 필수.

## 5. 광대역 De-noise — 오프라인 "프린트" (별도 스테이지)

실시간 스펙트럼 감산은 지연·아티팩트 부담이 커서 범위 밖. 대신:

- **노이즈 프로파일 학습**: 사용자가 보컬 클립의 **무음 구간(숨소리·룸톤만)을 선택** → 그 구간 스펙트럼을 노이즈 지문으로.
- **스펙트럼 감산 렌더**: 클립 전체에 STFT 기반 감산 → **새 WAV로 프린트**(비파괴).
- **인프라 재사용**: 이 앱은 이미 bake/offline-render + `<Project> Audio/Consolidated/` 기록 + 비파괴 클립 교체(Phase 5 `deleteRange`+삽입, Merge Clips)가 있다 → **De-noise 프린트를 그 위에 얹는다.** 원본 WAV 보존, Undo 복원 가능.
- 실행 위치: 네이티브(오프라인 커맨드) 권장. ffmpeg `afftdn` 활용도 후보(이미 ffmpeg 의존).

## 6. UI

- **채널 스트립 패널** — `kind === audioIn/bounce`에서만 노출. 스템(`file`)은 기존 컨트롤(볼륨/팬/reverb/echo)만.
- 배치 후보: 믹서([ui-mixer.jsx](ui-mixer.jsx))의 보컬 채널에 스트립 섹션, 또는 트랙 헤더 확장 패널([ui-tracks.jsx](ui-tracks.jsx)). Advanced 창(EQ/Pan/Ambience 별창) 패턴도 참고([advanced-eq-app.jsx](advanced-eq-app.jsx)).
- 각 이펙트: on 토글 + 파라미터 + 프리셋. 스트립 전체 bypass(A/B).
- 하드코딩 액센트 색 이슈(앱개발.md) 재발 방지 — 처음부터 `var(--amber)` 등 테마 변수 사용.

## 7. 단계별 구현 계획

| 스테이지 | 내용 | 엔진 | 위험 |
|---|---|---|---|
| **A. 골격** | `vocalFx` 데이터 모델 + kind 게이팅 + 채널 스트립 UI 셸 + bypass, 저장/복원 | JS/UI | 낮음 |
| **B. EQ + Compressor** | 웹·네이티브·Export·저장. 보컬 프리셋 | 웹+네이티브 | 낮음~중간 |
| **C. HPF + Noise Gate** | 럼블 컷 + 구간 무음 | 웹+네이티브 | 중간 |
| **D. De-esser** | 치찰음 대역 comp | 웹+네이티브 | 중간 |
| **E. 광대역 De-noise** | 오프라인 프린트(노이즈 프로파일 → STFT 감산 → 새 WAV, 비파괴) | 네이티브/ffmpeg | 높음 |

각 스테이지 독립 시험 가능. B가 가장 체감 큼(EQ+Comp) → **B부터 착수 권장**(A는 B에 흡수 가능).

## 8. 시험 관점(착수 시 [시험.md](시험.md)에 항목화)

- 스템(`file`) 트랙엔 채널 스트립이 **안 보인다**(회귀 없음).
- 보컬 트랙 EQ/Comp/Gate/HPF/De-esser가 재생·Export(웹/네이티브 A/B)에 동일 반영.
- 저장/재열기 후 보컬 FX 파라미터 유지. 구프로젝트(필드 없음) 정상 로드.
- 네이티브 재접속 후 보컬 FX 유지(재동기화 확인).
- bypass(A/B)로 완전 우회. Undo/Redo 정합.
- De-noise: 원본 WAV 불변, 프린트 결과 Undo 복원.

# Advanced Ambience — 음장효과(Sound Environment / Room Type) 구현 제안서

> 대상: **advanced-ambience.html (Advanced Effect Factory > Ambience 모듈)**
> 작성일: 2026-06-17 (앱 v1.5.24 기준)
> 목적: Ambience 창에 **음장(Room) 프리셋 선택 UI**를 추가하고, 선택한 공간감을 스튜디오 마스터 출력에 실시간/Export 모두 반영한다.

---

## 1. 목표 및 범위

- Ambience 창에 **설정용 메뉴(음장 설정 패널)** 를 추가한다.
- **5가지 음장 프리셋**을 **그림(일러스트)으로 보고 선택**할 수 있게 한다.
  - `concert hall`, `home`, `far field`, `studio`, `tunnel`
- 선택한 프리셋이 **마스터 출력의 리버브 IR + 웨트/프리딜레이/톤**에 매핑되어, studio.html·mixer.html 재생음과 Export 결과에 동일하게 반영된다.
- 기존 Pan 모듈과 동일한 **BroadcastChannel 동기화 + 테마 연동 + Undo** 패턴을 따른다.

### 범위 밖(이번 단계 제외)
- 트랙별 개별 리버브 센드(현재는 마스터 단일 음장만).
- 실측 IR(.wav) 파일 임포트 — 1차는 **합성 IR(procedural)** 로 구현(패키지 경량·JUCE 이식성 우선). 실측 IR 임포트는 후속 과제.

## 2. 현재 구조 요약 (구현 근거)

오디오 엔진에는 이미 마스터 리버브 경로가 존재한다.

| 위치 | 내용 |
|---|---|
| `audio-engine.js: makeIR(ctx, seconds, decay)` (≈165행) | 지수 감쇠 노이즈로 합성 IR 생성 |
| `audio-engine.js` 실시간 그래프 (≈280·365행) | `mConv = createConvolver(); mConv.buffer = makeIR(ctx, 2.8, 3.0)` → `masterVol → mRevSend(gain=master.reverb) → mConv → masterMix` |
| `audio-engine.js: setMaster("reverb", v)` (≈921·925행) | `mRevSend.gain` 램프 |
| `audio-engine.js` 오프라인 Export (≈1476·1485행) | `mConvO.buffer = makeIR(off, 2.8, 3.0)` 동일 경로 재구성 |
| 스냅샷/복원 (≈995·1017행) | `master.reverb`가 프로젝트 저장/Undo에 포함 |
| `app.jsx` BroadcastChannel 핸들러 (≈1573~1620행) | `ADVANCED_READY → sendInit(INIT_STATE)`, `SET_TRACK_PARAM` 등 라우팅 |

**핵심 아이디어**: "음장 프리셋"은 결국 **(IR 모양 + 웨트량 + 프리딜레이 + 톤)의 묶음**이다. 위 합성 IR 생성기와 컨볼버 버퍼를 프리셋에 따라 교체하면 된다.

### 설계 결정 — 전용 Ambience 버스 (권장)
기존 믹서의 마스터 `reverb` 노브와 의미 충돌을 피하기 위해, **마스터 리버브와 별개의 전용 Ambience 컨볼버 버스**를 신설하는 것을 권장한다.

- **Option A (권장)**: `masterVol → ambSend(gain) → ambConv(room IR) → ambPreDelay → masterMix` 별도 경로 추가. 믹서 reverb 노브는 그대로 두고, 음장은 독립적으로 on/off.
- **Option B**: 기존 `mConv.buffer`를 프리셋 IR로 교체하고 `master.reverb`를 웨트로 공유. 추가 노드 없이 가장 단순하지만 믹서 reverb 노브와 결합됨.

아래 구현 방안은 **Option A** 기준으로 기술하되, 노드 1세트(send/convolver/predelay)만 추가하므로 비용이 작다.

## 3. 프리셋 파라미터 사양

각 프리셋은 합성 IR 스펙과 믹스 파라미터의 묶음으로 정의한다. (값은 1차 튜닝 기준선이며 청취 후 조정)

| 프리셋 | 키 | decay(s) | shape(감쇠지수) | pre-delay(ms) | wet | damping LP(Hz) | width | 비고 |
|---|---|---|---|---|---|---|---|---|
| Studio (Dry/Booth) | `studio` | 0.35 | 1.6 | 5 | 0.12 | 9000 | 0.8 | 타이트·근접, 거의 드라이 |
| Home (Living Room) | `home` | 0.7 | 2.2 | 8 | 0.20 | 5500 | 0.9 | 가구·카펫 흡음, 따뜻함 |
| Concert Hall | `concert` | 3.0 | 2.8 | 40 | 0.38 | 7000 | 1.4 | 길고 풍성한 잔향, 넓은 스테레오 |
| Far Field | `far` | 2.2 | 3.0 | 60 | 0.50 | 3500 | 1.2 | 원거리 청취, 드라이 비중↓·HF 흡음↑ |
| Tunnel | `tunnel` | 2.6 | 1.4 | 30 | 0.45 | 4000 | 0.7 | 금속·콤필터 공명 + 슬랩 에코 |

- `wet`는 Ambience send gain(0~1)에 해당.
- `damping LP`: IR 생성 시 1-pole 로우패스를 적용해 고역 흡음(거리·재질) 모사.
- `width`: IR L/R 디코릴레이션 정도(스테레오 확산).
- `tunnel`은 추가로 기존 마스터 echo(슬랩)와 약한 피드백을 병용해 금속성 반사 표현.

### 데이터 규격 (JSON 직렬화 — JUCE 이식 대비)
```js
// audio-engine.js 상단 상수로 정의
const ROOM_PRESETS = {
  none:    { decay: 0,    shape: 1,   preDelay: 0,   wet: 0,    damp: 20000, width: 1.0 },
  studio:  { decay: 0.35, shape: 1.6, preDelay: 5,   wet: 0.12, damp: 9000,  width: 0.8 },
  home:    { decay: 0.7,  shape: 2.2, preDelay: 8,   wet: 0.20, damp: 5500,  width: 0.9 },
  concert: { decay: 3.0,  shape: 2.8, preDelay: 40,  wet: 0.38, damp: 7000,  width: 1.4 },
  far:     { decay: 2.2,  shape: 3.0, preDelay: 60,  wet: 0.50, damp: 3500,  width: 1.2 },
  tunnel:  { decay: 2.6,  shape: 1.4, preDelay: 30,  wet: 0.45, damp: 4000,  width: 0.7, echo: 0.25 },
};
```
- 마스터 상태에 `room` 필드 추가, 기본값 `'none'`(= 현재 동작 유지, 기존 프로젝트 무영향).

## 4. 오디오 엔진 구현 방안 (`audio-engine.js`)

### 4.1 합성 IR 생성기 확장 — `makeRoomIR(ctx, spec)`
기존 `makeIR`를 확장해 **프리딜레이 + 초기반사(early reflections) + 감쇠 테일 + 고역 흡음 + 스테레오 디코릴레이션**을 포함한 IR을 생성.

```js
function makeRoomIR(ctx, spec) {
  const sr = ctx.sampleRate;
  const pre = Math.floor((spec.preDelay / 1000) * sr);
  const tail = Math.max(1, Math.floor(spec.decay * sr));
  const len = pre + tail;
  const ir = ctx.createBuffer(2, len, sr);
  const lpCoef = Math.exp(-2 * Math.PI * (spec.damp / sr)); // 1-pole LP
  for (let c = 0; c < 2; c++) {
    const d = ir.getChannelData(c);
    let lp = 0;
    // 초기 반사 몇 개(채널별 위상차로 width 표현)
    // ... sparse taps ...
    for (let i = 0; i < tail; i++) {
      const env = Math.pow(1 - i / tail, spec.shape);
      let s = (Math.random() * 2 - 1) * env;
      lp = lp + (1 - lpCoef) * (s - lp); // 고역 흡음
      d[pre + i] = lp * (c === 0 ? 1 : spec.width); // 간이 디코릴레이션
    }
  }
  return ir;
}
```

### 4.2 그래프에 Ambience 버스 추가 (Option A)
초기화 시 `ambSend / ambConv` 노드를 1세트 추가하고 `masterVol → ambSend → ambConv → masterMix`로 연결. `ambConv.buffer = makeRoomIR(ctx, ROOM_PRESETS[this.master.room])`, `ambSend.gain = spec.wet`.

### 4.3 `setRoom(presetKey)` 메서드
```js
setRoom(key) {
  const spec = ROOM_PRESETS[key] || ROOM_PRESETS.none;
  this.master.room = key;
  ambConv.buffer = makeRoomIR(ctx, spec);
  ramp(ambSend.gain, spec.wet);
  if (spec.echo !== undefined) ramp(mEchoSend.gain, spec.echo); // tunnel 슬랩
}
```

### 4.4 Export(오프라인) 경로 동일 반영
`renderMix()`의 오프라인 그래프(≈1476행)에도 동일하게 `ambConvO.buffer = makeRoomIR(off, ROOM_PRESETS[this.master.room])` 와 send를 추가 → **재생음과 Export 결과 일치**.

### 4.5 상태/스냅샷/복원
- `this.master` 기본 객체에 `room: 'none'` 추가(≈240·577행).
- 스냅샷 직렬화에 `room` 포함(≈995행), 복원 시 `this.setRoom(snap.master.room ?? 'none')`(≈1017행).
- → 프로젝트 저장·로드, **Undo/Redo** 자동 지원.

## 5. 동기화 / IPC (`app.jsx` ↔ ambience 창)

기존 Pan 모듈과 동일한 `focusdaw-advanced-effects-sync` BroadcastChannel 재사용.

```mermaid
sequenceDiagram
    participant Amb as advanced-ambience-app
    participant Std as app.jsx (studio)
    participant DAW as audio-engine
    Amb->>Std: ADVANCED_READY
    Std-->>Amb: INIT_STATE { ..., room, theme }
    Note over Amb: 프리셋 카드에 현재 room 하이라이트
    Amb->>Std: SET_ROOM_PRESET { room }
    Std->>DAW: pushUndo(); DAW.setRoom(room)
    Std-->>Amb: SYNC_STATE { room }  (다른 창 동기화)
```

- **`app.jsx` 핸들러 추가**(≈1612행 switch에 case 추가):
  ```js
  case "SET_ROOM_PRESET":
    pushUndo();
    DAW.setRoom(msg.room);
    saveRecentProject(projectName, projectPath);
    force((n) => n + 1);
    break;
  ```
- **`sendInit`** 의 INIT_STATE에 `room: DAW.master.room` 추가(≈1587행).
- 프리셋 변경 시 다른 Advanced 창(pan/eq)도 일관성 유지하도록 `SYNC_STATE { room }` 브로드캐스트(선택).

## 6. UI/UX — Ambience 설정 패널 (`advanced-ambience-app.jsx`)

현재 빈 `aef-room` 영역을 **음장 설정 패널**로 교체.

### 6.1 레이아웃
```
┌───────────────────────────────────────────────┐
│  SOUND ENVIRONMENT                              │  ← 섹션 라벨
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ │
│  │ [그림]│ │ [그림]│ │ [그림]│ │ [그림]│ │ [그림]│ │  ← 프리셋 카드(라디오)
│  │Concert│ │ Home │ │ Far  │ │Studio│ │Tunnel│ │
│  │ Hall  │ │      │ │ Field│ │      │ │      │ │
│  └──────┘ └──────┘ └──────┘ └──────┘ └──────┘ │

```

- **카드 = 라디오 선택**: 클릭 시 해당 프리셋이 즉시 적용(`SET_ROOM_PRESET` 전송), 선택 카드에 테마 강조색(amber) 테두리·glow.
- 그림(일러스트)으로 공간을 직관 인지: 콘서트홀(아치/객석), 가정(소파/방), 원거리(작은 음원+거리감), 스튜디오(부스/모니터), 터널(원통 터널).
- 상단 라벨 `SOUND ENVIRONMENT`, 각 카드에 이름 + 1줄 설명.
- 테마 연동: 카드/일러스트는 `currentColor` 기반(기존 악기 아이콘과 동일 방식)으로 테마 색을 따른다.

### 6.3 일러스트 자산 방식 (확정: 제공된 PNG 사용)
프리셋 그림은 **이미 제작되어 아래 폴더에 제공됨**:

```
.\Advanced Effects Factory\(template) Room Type\
  ├─ concert hall.png
  ├─ far field.png
  ├─ home.png
  ├─ studio.png
  └─ tunnel.png
```

- 위 5개 PNG를 배포 자산 경로 **`assets/ambience/`** 로 복사하고(공백 제거한 정규화 파일명 권장: `concert-hall.png`, `far-field.png`, `home.png`, `studio.png`, `tunnel.png`), 각 프리셋 카드의 이미지로 사용한다(`<img>` 또는 카드 배경).
- `assets/**/*` 는 이미 `electron-builder` files 목록에 포함되어 있어 **추가 패키징 설정 불필요**.
- PNG는 래스터라 테마색과 자동 연동되지 않으므로, 카드 프레임/라벨/선택 하이라이트(테두리·glow)만 테마 토큰(`--surface/--amber/--line-strong`)으로 처리해 그림은 그대로 두고 테두리로 테마감을 준다.
- 파일 크기(약 0.3~0.5MB/장)가 부담되면 빌드 단계에서 가로 ~480px 내외로 리사이즈/압축한 사본을 쓰는 것을 후속 최적화로 고려.

> **매핑**: `concert hall.png→concert`, `home.png→home`, `far field.png→far`, `studio.png→studio`, `tunnel.png→tunnel` (3절 `ROOM_PRESETS` 키와 일치).

### 6.4 CSS
`advanced-ambience.html` `<style>`에 카드 그리드 클래스 추가(`.amb-grid`, `.amb-card`, `.amb-card.active`) — 기존 `--surface/--amber/--line-strong` 토큰 사용으로 전 테마 호환.

## 7. 구현 단계 (체크리스트)

1. **자산**: `(template) Room Type`의 5개 PNG를 `assets/ambience/`로 정규화 파일명 복사.
2. **엔진**: `ROOM_PRESETS` 상수 + `makeRoomIR()` + Ambience 버스 노드 + `setRoom()` 추가 (`audio-engine.js`).
3. **상태**: `master.room` 기본값/스냅샷/복원 연동 (저장·Undo).
4. **재생 반영**: 실시간 그래프(`masterVol → ambSend → ambConv → masterMix`)에 room IR 적용 — **재생(play) 시 즉시 반영**.
5. **Export 반영**: `renderMix()` 오프라인 그래프(≈1476행)에 **동일한 room IR/send 추가** — 내보낸 파일에 음장 포함.
6. **IPC**: `app.jsx` INIT_STATE에 `room` 추가 + `SET_ROOM_PRESET` 케이스 추가(+pushUndo).
7. **UI**: `advanced-ambience-app.jsx`에 프리셋 카드 그리드 + PNG 일러스트 + 선택/동기화 로직.
8. **CSS**: `advanced-ambience.html`에 카드 그리드 스타일.
9. **빌드/검증**: `npm run build:renderers` + 8절 재생·Export 일치 검증.

> ⚠️ **필수 요구사항 — Play & Export 일치**: 음장 효과는 **재생(play)과 내보내기(export) 양쪽에서 동일하게 반영**되어야 한다. 실시간 그래프(4)와 오프라인 그래프(5)가 **같은 `ROOM_PRESETS` 스펙과 `makeRoomIR()` 로직을 공유**하도록 구현하고(코드 중복 시 동일 함수 재사용), 아래 8절에서 두 경로가 일치함을 반드시 검증한다.

## 8. 시험 계획 (초안 — `시험.md` 등록용)

| 구분 | 항목 | 기대 결과 |
|---|---|---|
| 자동 | `npm run build:renderers` 통과 | 오류 없이 번들 생성 |
| 자동 | `node --check audio-engine.js / app.jsx 번들` | 문법 오류 없음 |
| 자동 | 자산 존재 확인 | `assets/ambience/*.png` 5개 파일 존재, 카드 src 경로와 일치 |
| 수동 | 5개 프리셋 청취(**Play**) | 각 음장의 잔향 길이·톤·공간감 차이가 재생음에서 명확히 들림 |
| 수동 | **Export 반영** | 각 프리셋으로 내보낸 WAV/MP3에 음장이 실제로 포함됨(드라이 아님) |
| 수동 | **Play ↔ Export 일치** | 동일 프리셋에서 재생 모니터링 음과 내보내기 파일의 공간감이 일치 |
| 정량(권장) | Export 결과 검증 | `none`(드라이) 대비 프리셋 적용 시 **잔향 꼬리(RT) 증가·종료 시점 연장**이 파형/스펙트로그램에서 측정됨 |
| 수동 | 동기화 | 다른 창/믹서와 상태 일치, 카드 그림 표시·테마 색(테두리) 연동 |

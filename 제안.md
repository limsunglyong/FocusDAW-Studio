# Advanced Effect Factory — Auto Panning 제안서

본 제안서는 FocusDAW Studio에 별도 **Advanced Effects** 작업 창을 추가하고, 첫 번째 고급 이펙트로 **Auto Panning / Spatial Panner** 기능을 도입하기 위한 개발 방향과 단계별 구현 방안을 정리합니다. 목표는 기존 `studio.html` 및 `mixer.html`의 트랙 PAN/Volume 값을 더 직관적인 청취 공간 UI에서 편집하고, 이후 Saturation, Widener, Exciter, Modulation 계열 이펙트를 같은 Factory 구조로 확장할 수 있는 기반을 만드는 것입니다.

---

### 핵심 목표 사항
1. **Advanced Effects 진입점 추가**: 메인 앱 상단 메뉴와 Mixer OUTPUT EFFECTS 영역에서 고급 이펙트 창을 열 수 있게 합니다.
2. **Auto Panning 전용 창 구현**: 트랙별 PAN과 볼륨 gain을 가상 청취 공간의 좌우 위치와 거리로 시각화합니다.
3. **양방향 실시간 동기화**: Advanced PAN 화면에서 악기 아이콘 또는 PAN 노브를 조작하면 `studio.html`과 `mixer.html`의 PAN/Volume 값이 즉시 반영됩니다.
4. **확장 가능한 Effect Factory 구조**: Auto Panning을 첫 모듈로 만들되, 이후 고급 이펙트가 같은 창/메뉴/상태 동기화 구조를 재사용하도록 설계합니다.

> **진행 상황 (2026-06-16 / 앱 버전 v1.5.0 기준)**
> 1차 구현을 시작해 Advanced PAN 독립 창, 메인 메뉴 진입점, Mixer Advanced 버튼, 테마 연동, PAN/Volume 실시간 동기화 경로를 추가했습니다. Auto Pan LFO/DSP 파라미터는 다음 단계에서 확장합니다.

---

## 1. 구현 대상과 사용자 경험

### 1.1 진입 경로
- 메인 앱 상단 메뉴의 `Edit`와 `Settings` 사이에 **Advanced Effects** 메뉴를 추가합니다.
- `Advanced Effects` 메뉴의 첫 번째 항목으로 **Auto Panning**을 배치합니다.
- Mixer 창 우측 하단 `OUTPUT EFFECTS` 헤더 오른쪽에 **Advanced** 버튼을 추가합니다.
- 두 진입 경로 모두 동일한 Advanced PAN 창을 엽니다.

### 1.2 Advanced PAN 창의 역할
- 하단에는 트랙 수만큼 작은 원형 PAN 노브를 배치합니다.
- 중앙에는 가상의 청취 공간을 표시하고, 트랙별 악기 아이콘을 현재 PAN/Volume 값에 맞춰 배치합니다.
- 좌우 위치는 트랙 `params.pan` 값과 일치시킵니다.
- 청취자와의 거리는 트랙 `params.volume` gain 값과 일치시킵니다.
- 사용자가 아이콘을 드래그하면 PAN과 Volume이 함께 갱신됩니다.
- 사용자가 하단 노브를 조작하면 PAN 값이 갱신되고, 중앙 아이콘 위치도 즉시 움직입니다.

## 2. UI 설계 방향

### 2.1 기존 템플릿 활용
참고 템플릿은 `Advanced Effects Factory/(template) Auto Pan/Advanced Effect Factory v2.dc.html`입니다. 템플릿은 다음 장점을 이미 갖고 있습니다.

- FocusDAW와 어울리는 어두운 아날로그 톤의 윈도우 디자인
- 상단 타이틀바, 툴바, 가상 공간, 하단 노브 랙의 4영역 구성
- Piano, Percussion, Guitar, Bass, Synth 등 악기 심볼 SVG 정의
- 악기 노드 드래그와 PAN 노브 조작 프로토타입
- 트랙 이름 기반 악기 매칭 로직 초안

실제 앱 구현에서는 이 템플릿을 그대로 런타임에 사용하는 대신, 필요한 SVG 심볼과 레이아웃 개념을 `Advanced Effects Factory`의 앱 코드로 복사해 FocusDAW의 React/Electron 구조에 맞게 재구성하는 것이 좋습니다.

### 2.2 화면 구성
권장 레이아웃은 다음과 같습니다.

1. **최상단 윈도우 바**
   - `ADVANCED EFFECT FACTORY`
   - 현재 모듈명 `Auto Panning`
   - 닫기/최소화 버튼

2. **제목 및 컨트롤 바**
   - `SPATIAL PANNER`
   - 선택 트랙 요약
   - `Reset Pan`, 추후 `Reset Distance`, `Apply Preset` 확장 영역

3. **청취 공간 + 선택 트랙 정보**
   - 왼쪽 또는 중앙: 방사형 청취 공간
   - 오른쪽: 선택 트랙 정보 패널
   - 정보 패널은 공간 그림 위에 겹쳐 UI를 가리지 않도록, 데스크톱에서는 좌우 분할을 기본으로 권장합니다.
   - 창 폭이 좁을 때만 반투명 오버레이 카드로 전환합니다.

4. **하단 PAN 노브 랙**
   - 템플릿 노브의 약 60% 크기
   - 트랙 수가 많을 때 가로 스크롤
   - 각 노브에는 트랙 번호, 짧은 이름, PAN 라벨 표시

### 2.3 선택 트랙 정보 패널
선택된 트랙이 있을 때 오른쪽 패널에 다음 정보를 표시합니다.

- 트랙 번호와 트랙 이름
- 악기 분류 아이콘과 분류명
- 현재 PAN 값: `L/R/C` 라벨과 내부 `-1.0 ~ +1.0` 값
- 현재 Volume gain: dB 표시
- 추후 이펙트 적용 상태 영역
  - Auto Pan 활성 상태
  - Motion Rate
  - Depth
  - Shape
  - Sync 상태

## 3. 데이터 모델 및 동기화

### 3.1 현재 값 매핑
FocusDAW의 기존 트랙 파라미터를 우선 재사용합니다.

- 좌우 위치: `track.params.pan`
  - 현재 앱 기준 `-1.0 ~ +1.0`
  - UI 표시 기준 `L100 ~ C ~ R100`

- 청취자 거리: `track.params.volume`
  - 현재 앱 기준 gain 값
  - UI 표시 기준 dB
  - 가까움 = 높은 gain, 멀어짐 = 낮은 gain

### 3.2 좌표 변환
권장 변환은 다음과 같습니다.

- `pan = -1`이면 왼쪽 끝, `0`이면 중앙, `+1`이면 오른쪽 끝
- `volume = 1.0`은 0 dB 기준 원에 배치
- `volume < 1.0`은 청취자에서 더 멀리 배치
- `volume > 1.0`은 청취자에 더 가까운 영역으로 배치하되, UI상 과도하게 겹치지 않도록 최소/최대 반경을 제한

방사형 눈금에서는 0 dB 원을 더 굵게 표시합니다. `-inf`, `-12 dB`, `-6 dB`, `0 dB`, `+6 dB`처럼 실제 gain/dB와 의미가 맞는 라벨을 사용하는 것이 좋습니다.

### 3.3 창 간 동기화
Advanced PAN 창도 Mixer 창처럼 `BroadcastChannel` 기반 동기화를 사용하는 것을 권장합니다.

- 채널명 예: `focusdaw-advanced-effects-sync`
- 초기 메시지: `ADVANCED_READY`
- 메인 → Advanced:
  - `INIT_STATE`
  - `SYNC_STATE`
  - track list, selected track, theme, play state
- Advanced → 메인:
  - `SET_TRACK_PARAM`
  - `BEFORE_CHANGE`
  - `REQUEST_CLOSE`

이 구조를 쓰면 Advanced PAN 창에서 변경한 `pan`과 `volume`이 메인 앱 및 Mixer 창에도 동일한 엔진 상태를 통해 즉시 전파됩니다.

## 4. 악기 심볼 매칭

트랙 이름을 소문자로 정규화한 뒤 키워드 매칭으로 악기 심볼을 선택합니다.

| 분류 | 대표 키워드 |
|---|---|
| Piano | piano, keys, keyboard, rhodes, wurli |
| Percussion | drum, drums, kick, snare, perc, percussion |
| Organ | organ, hammond |
| Guitar | guitar, gtr, acoustic, electric |
| Bass | bass, sub |
| Strings | string, strings, violin, viola, cello |
| Brass | brass, trumpet, trombone, horn |
| Reed | sax, reed, clarinet, oboe, bassoon |
| Pipe | flute, pipe, recorder |
| Synth Lead | synth, lead, pad, arp, saw |
| Vocal | vocal, vox, voice, singer |
| Unknown | 위 조건에 해당하지 않는 경우 음표 심볼 |

템플릿에는 일부 심볼이 이미 있으므로, 빠진 `vocal`, `unknown` 등은 FocusDAW 스타일에 맞는 간단한 SVG 심볼로 추가합니다.

## 5. 단계별 개발 로드맵

### Phase 1: 설계 및 정적 프로토타입 정리
- `Advanced Effects Factory` 폴더 아래에 실제 앱용 파일 구조를 정리합니다.
- 템플릿에서 필요한 SVG 심볼, 색상, 공간 레이아웃 개념을 복사합니다.
- `(template) Auto Pan` 폴더는 참조 전용으로 유지하고, 실제 구현은 별도 파일에 둡니다.
- 이번 문서 제안 단계가 여기에 해당합니다.

### Phase 2: Electron Advanced PAN 창 추가
- `electron/main.js`에 Advanced Effects 창 생성/재사용 로직을 추가합니다.
- `advanced-effects.html` 또는 `advanced-pan.html`을 추가합니다.
- `electron/preload.js`에 창 제어 API를 노출합니다.
- 메인 앱 상단 메뉴와 Mixer의 `Advanced` 버튼에서 창을 열도록 연결합니다.

### Phase 3: React UI 구현
- `advanced-pan-app.jsx`를 작성합니다.
- 트랙 목록, 악기 심볼, 청취 공간, 하단 PAN 노브 랙을 구현합니다.
- 선택 트랙 정보 패널을 추가합니다.
- 현재 테마와 색상 변수를 최대한 재사용합니다.

### Phase 4: 실시간 파라미터 동기화
- Advanced 창에서 `pan`과 `volume` 변경 시 `DAW.setTrackParam()`로 전달합니다.
- 메인 앱, Mixer 창, Advanced 창이 같은 값을 표시하는지 검증합니다.
- Undo 단위는 드래그 시작 시 `BEFORE_CHANGE`를 한 번만 보내도록 합니다.

### Phase 5: Auto Panning 이펙트 확장
- 정적 PAN 편집이 안정화된 뒤, 자동 움직임 파라미터를 추가합니다.
- 예시:
  - Enable
  - Rate
  - Depth
  - Shape: sine, triangle, random, pendulum
  - Sync: free Hz 또는 BPM sync
- Web Audio fallback과 JUCE 네이티브 엔진 양쪽에 동일한 파라미터를 전달할 수 있도록 데이터 모델을 별도 설계합니다.

## 6. 구현 시 주의점

- 첫 단계에서는 오디오 DSP를 새로 만들지 말고, 기존 `pan`과 `volume` 편집 UI로 시작하는 것이 안전합니다.
- Advanced PAN 창은 Mixer 창과 마찬가지로 열고 닫을 때 파괴하지 않고 재사용하는 편이 실행 속도와 상태 유지에 유리합니다.
- 트랙 수가 많을 때 하단 노브와 중앙 아이콘이 겹칠 수 있으므로, 중앙 공간은 아이콘 크기와 라벨 길이에 반응하는 충돌 완화가 필요합니다.
- 드래그 중에는 너무 많은 undo snapshot을 만들지 않도록 조작 시작 시점에만 undo를 기록해야 합니다.
- 템플릿의 폰트 외부 로드는 앱 패키징 및 오프라인 실행을 고려해 제거하거나 로컬 스타일로 대체하는 것이 좋습니다.

---

## 7. 권장 결론

Advanced Effect Factory는 곧바로 복잡한 자동 이펙트 엔진부터 만들기보다, **정적 Spatial Panner UI + 기존 PAN/Volume 실시간 편집**을 1차 목표로 삼는 것이 가장 안전합니다. 이 단계만으로도 사용자는 숫자와 노브 대신 실제 청취 공간에서 트랙 위치를 조정하는 경험을 얻을 수 있고, 이후 Auto Pan의 Rate/Depth/Shape 같은 시간 기반 이펙트는 같은 창 안에서 자연스럽게 확장할 수 있습니다.

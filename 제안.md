# 파일 트랙 정리와 Audio In 녹음 트랙 구현 제안

> 목표: 파일 기반 트랙은 프로젝트 시작점에 고정해 관리하고, 마이크 등 실시간 입력을 녹음하는 `Audio In` 트랙은 여러 테이크와 이동 가능한 클립을 중심으로 별도 동작하게 한다.
>
> 현재 코드에는 `clips`, `splitClip`, `joinClips`, 클립별 볼륨의 기초가 있다. 그러나 트랙 하나가 `AudioBuffer` 하나만 소유하는 구조이므로, 다중 녹음 전에 오디오 source와 타임라인 clip을 분리해야 한다.

---

## 1. 권장 UI

### 파일 기반 트랙

- 파일 트랙은 `File Tracks` 그룹에 모은다.
- 그룹 헤더의 화살표로 전체를 접거나 펼친다.
- 접힌 상태에는 트랙 수, Mute/Solo 상태, 그룹 레벨을 표시한다.
- 파일 클립은 현재처럼 `00:00`에 고정하며 좌우 이동을 허용하지 않는다.
- 2개 이상 선택하면 `Merge Tracks…`를 활성화한다.

### Audio In 트랙

- `+ Audio In`으로 생성하고 입력 장치, 채널, 입력 Gain, Monitor, Limiter를 설정한다.
- Arm된 트랙만 녹음한다.
- 루프 녹음 시 매 회차를 `Take A`, `Take B`, `Take C`로 만든다.
- 기본 화면에는 활성 Take 또는 Comp 결과를 표시하고, `Take Lanes`를 펼치면 모든 Take를 보여준다.
- Audio In 클립은 시간축에서 이동, 분할, trim, 삭제, 볼륨 변경할 수 있다.

```text
▼ File Tracks (6)                         [Merge…]
  Drums.wav        ┃ waveform (위치 고정)
  Bass.wav         ┃ waveform (위치 고정)

▼ Audio In 1 — Vocal       [Input 1] [M] [S] [Arm] [Monitor]
  Comp             ┃        [선택된 구간]
  Take A           ┃        [=========]
  Take B           ┃        [=========]
  Take C           ┃        [=========]
```

## 2. 데이터 모델 변경

현재는 트랙이 버퍼 하나를 직접 소유하고 클립이 그 버퍼의 `offset`만 참조한다. 여러 녹음 파일을 한 트랙에 담도록 source를 분리한다.

```js
track = {
  id,
  kind: "file" | "audioIn" | "bounce",
  name,
  color,
  lockedToZero: true | false,
  params: { volume, pan, mute, solo, ... },
  input: {
    deviceId,
    channel: 0,
    armed: false,
    monitor: false,
    inputGainDb: 0,
    limiterEnabled: true,
    limiterCeilingDb: -1.0
  },
  sources: [{
    id: "src-...",
    filePath,
    duration,
    sampleRate,
    channels,
    peaks,
    needsAudio: false
  }],
  clips: [{
    id: "clip-...",
    sourceId: "src-...",
    start: 12.5,       // 프로젝트 시간 위치
    sourceOffset: 0.0, // 녹음 파일 내부 시작점
    duration: 8.0,
    gain: 1.0,
    muted: false,
    takeId: "take-B"
  }],
  takes: [
    { id: "take-A", label: "A", active: false },
    { id: "take-B", label: "B", active: true }
  ]
}
```

규칙:

- `file`: `lockedToZero = true`, 원본 파일 하나와 시작점 0을 유지한다.
- `audioIn`: 여러 source/clip을 가지며 clip의 `start`를 변경할 수 있다.
- `bounce`: 여러 트랙 또는 클립을 렌더해 만든 파일 트랙이다.
- 프로젝트 길이는 모든 클립의 `start + duration` 최댓값으로 계산한다.
- 프로젝트에 `schemaVersion`을 추가하고 기존 `buffer + clips`를 `sources[0] + sourceId`로 자동 변환한다.

## 3. 파일 트랙 접기와 트랙 합치기

### 접기/펼치기

`app.jsx`의 현재 `DAW.tracks.map(...)` 렌더링을 `FileTrackGroupHeader`, 파일 트랙 목록, Audio In 트랙 목록으로 나눈다.

접힘은 오디오 상태가 아닌 프로젝트 UI 상태다.

```js
project.ui = {
  fileTracksCollapsed: false,
  expandedTakeLanes: ["track-id"]
}
```

- 접어도 재생, Export, Mute/Solo에는 영향이 없다.
- 접힌 파일 트랙은 렌더링하지 않아 waveform canvas 비용도 줄인다.
- 그룹 헤더는 현재 sticky 트랙 헤더 폭과 타임라인 정렬을 유지한다.

### 2개 이상 트랙 합치기

`Merge Tracks`는 선택 트랙을 새 WAV 하나로 Offline 렌더하는 Bounce로 정의한다.

1. `Ctrl/Shift + Click`으로 트랙 다중 선택
2. `Merge Tracks…` 실행
3. 전체 프로젝트/루프 범위, Mono/Stereo, Gain·Pan·자동화·FX 포함 여부 선택
4. 원본 처리: 유지/Mute/삭제 선택
5. 결과를 프로젝트 `Audio/Bounces`에 저장하고 `kind: "bounce"`로 추가

기본값은 `원본 유지 + Mute`가 안전하다. 현재 `joinClips()`는 한 트랙의 인접 클립을 잇는 기능이므로 새 API `mergeTracks(trackIds, options)`와 분리한다.

## 4. Audio In 및 실시간 Limiter

첫 버전은 JUCE 네이티브 엔진을 녹음 기준으로 삼는 것을 권장한다. 현재 브라우저 엔진과 네이티브 엔진을 동시에 사용하면 장치 시계와 지연 보정이 달라질 수 있다.

```text
Audio Input
  → Input Gain
  → High-pass/DC Filter(선택)
  → Look-ahead Limiter
  ├→ Recorder Writer
  ├→ Input Meter
  └→ Monitor Gain → Master Bus → Output
```

Limiter 권장값:

- 기본 On, Ceiling `-1.0 dBFS`
- Look-ahead 3~5 ms
- 빠른 attack, release 50~100 ms
- Limiter 전/후 레벨과 Gain Reduction 표시
- ADC 단계에서 이미 잘린 신호는 복구할 수 없으므로 별도의 `Input gain을 낮추세요` 경고 표시
- 첫 버전은 녹음과 Monitor 양쪽에 같은 Limiter 결과 적용

JUCE 변경:

- `AudioDeviceManager` 입력 장치/채널/sample rate/buffer size 설정 확장
- 오디오 콜백에서 입력 버퍼 수신과 Limiter 처리
- lock-free FIFO로 writer thread에 전달
- 녹음 시작/중지/취소 및 장치 오류 이벤트

오디오 콜백에서는 파일 쓰기, JSON 처리, 동적 메모리 할당, mutex 대기를 하지 않는다.

## 5. 다중 루프 녹음

### 상태와 명령

```text
idle → armed → countIn → recording → stopping → finalizing → armed
                         ↘ deviceError / diskError
```

브리지 명령:

- `createAudioInputTrack`
- `armTrack`
- `setInputDevice`, `setInputChannel`
- `startRecording({ trackId, start, end, loopEnabled, countIn })`
- `stopRecording`, `cancelRecording`

엔진 이벤트:

- `recordingStarted`, `recordingProgress`
- `takeCompleted`, `recordingStopped`
- `inputLevel`, `limiterReduction`
- `recordingError`

### 루프 경계

- 루프 시작/끝을 정확한 sample index로 변환한다.
- 오디오 블록이 루프 끝을 걸치면 경계에서 둘로 나눈다.
- 앞부분은 현재 Take, 뒷부분은 다음 Take 파일에 기록한다.
- UI 타이머가 아니라 엔진의 sample counter로 Take를 나눈다.
- Stop 시 불완전한 마지막 회차도 `Partial` Take로 보존한다.
- 입력 지연 보정값을 측정 또는 설정하여 clip의 `start`에 반영한다.

```text
Project/Audio/Recordings/Vocal_20260706_153012_Take-A.wav
Project/Audio/Recordings/Vocal_20260706_153012_Take-B.wav
```

녹음 중에는 `.wav.part`에 쓰고 정상 종료 후 `.wav`로 바꾼다. 비정상 종료 후 복구 후보로 사용할 수 있다.

## 6. Audio In 편집

### 삭제

- `Delete Clip`: 타임라인 참조만 제거
- `Delete Take`: 해당 Take의 모든 clip 제거
- WAV는 즉시 삭제하지 않고 `Unused`로 표시
- `Clean Up Unused Recordings…`에서만 영구 삭제

이 정책이어야 Undo/Redo가 녹음 원본을 복구할 수 있다.

### 볼륨

- 트랙 Volume과 clip Gain을 분리한다.
- clip Gain은 상단 핸들 또는 Inspector에서 `-∞ ~ +12 dB`로 조절한다.
- 기존 `setClipParam(..., "volume", value)`를 dB 표시와 source별 재생에 맞게 확장한다.
- 클립 경계에는 기본 3~5 ms micro fade를 적용해 click/pop을 막는다.

### 시간축 이동

- `audioIn`과 `bounce` clip만 드래그할 수 있다.
- 루프 경계, playhead, 다른 clip 경계, 박자 grid에 snap한다.
- `moveClip(trackId, clipId, newStart)` 한 명령으로 적용하고 drag 종료 시 Undo를 한 번만 만든다.
- 첫 버전에서는 같은 트랙 내 clip 겹침을 막고, 겹친 녹음은 Take Lane으로 표현한다.

### 구간 자르기/삭제

기존 `splitClip()`에 다음을 추가한다.

- `trimClipStart`, `trimClipEnd`
- `deleteClip`
- `deleteRange(trackId, from, to)`: 양 경계에서 split 후 가운데 조각 제거

기본 `Delete Range`는 뒤쪽 클립을 당기지 않고 빈 공간을 남긴다. 전체 시간축을 당기는 `Ripple Delete`는 별도 기능으로 분리해야 파일 트랙과 다른 녹음 트랙의 동기화가 깨지지 않는다.

### 합치기 명령 구분

1. `Join Edits`: 같은 source의 연속된 두 조각에서 편집 경계만 제거
2. `Consolidate Clips`: 한 Audio In 트랙의 선택 clip을 새 WAV로 렌더
3. `Merge Tracks`: 여러 트랙을 새 트랙 하나로 Bounce

`Join Edits`는 같은 `sourceId`, 연속된 `sourceOffset`, 맞닿은 시간 위치, 호환되는 Gain/automation일 때만 허용한다. 그 외에는 `Consolidate Clips`를 사용한다.

### Take 선택과 Comp

- 1단계: Take A/B/C 중 활성 Take 하나 선택
- 2단계: Take Lane의 구간을 선택해 Comp Lane에 반영
- Comp 구간 경계에는 짧은 equal-power crossfade 자동 생성
- Comp는 원본 WAV를 변경하지 않는 편집 결정 목록으로 저장

## 7. 재생 엔진 변경

현재 Web Audio 경로는 트랙당 `AudioBufferSourceNode` 하나를 시작한다. 다중 source와 이동 clip은 clip별 source 예약이 필요하다.

```js
for (const clip of audibleClips) {
  const src = ctx.createBufferSource();
  src.buffer = sourceMap.get(clip.sourceId).buffer;
  src.connect(clipGain).connect(trackBus);
  src.start(
    transportStartAudioTime + (clip.start - playhead),
    clip.sourceOffset,
    clip.duration
  );
}
```

- seek, pause, tempo/key 변경, loop wrap 시 예약 source를 취소하고 다시 예약한다.
- 긴 프로젝트는 수 초 앞만 예약하는 look-ahead scheduler를 사용한다.
- JUCE도 `loadTrack(trackId, filePath)` 중심에서 `loadSource`와 `setClipLayout` 구조로 확장한다.
- `audio-bridge.js`에 `loadSource`, `removeSource`, `setTrackLayout`, `setClipParam`, `moveClip` 동기화를 추가한다.

## 8. 프로젝트와 오디오 파일 관리

```text
My Project.focus
My Project Audio/
  Recordings/
  Bounces/
  Unused/
```

- `.focus`에는 프로젝트 기준 상대 경로를 우선 저장한다.
- Save As 시 녹음/Bounce 파일도 새 오디오 폴더로 복사한다.
- 저장 전 프로젝트는 앱 임시 녹음 폴더를 사용하고 첫 저장 시 이동한다.
- source에 파일 크기, 수정 시간, 선택적으로 hash를 기록한다.
- Autosave는 편집 메타데이터만 저장하고 WAV는 writer가 즉시 디스크에 확정한다.
- 녹음, Merge, Consolidate 결과 파일은 세션 동안 보존하여 Undo/Redo 가능하게 한다.

## 9. 권장 구현 순서

### Phase 1 — 모델과 파일 트랙 UI

- `track.kind`, `sources`, 새 clip 모델 도입
- 기존 프로젝트 migration과 `schemaVersion`
- 파일 트랙 0초 고정 및 접기/펼치기
- 다중 선택과 Offline `Merge Tracks`

완료 기준: 기존 프로젝트가 손실 없이 열리고 Bounce가 Export 결과와 일치한다.

### Phase 2 — 단일 Audio In 녹음

- JUCE 입력 장치/채널
- Arm, Monitor, 입력 미터
- Limiter와 Gain Reduction 미터
- 비루프 WAV 스트리밍 녹음
- 프로젝트 저장/재연결

완료 기준: 30분 녹음에서 드롭아웃과 지속적인 메모리 증가가 없고 중지 후 바로 재생된다.

### Phase 3 — 클립 편집

- 이동, trim, split, delete, delete range
- clip Gain과 micro fade
- Consolidate와 Undo/Redo

완료 기준: 편집 후 seek/loop/export 결과가 일치하고 경계 click이 없다.

### Phase 4 — 루프 Take

- sample-accurate 루프 분할
- Take A/B/C 및 Take Lanes
- 활성 Take, Partial Take, 오류 복구

완료 기준: 각 Take 길이가 루프 sample 수와 일치하고 반복해도 누적 밀림이 없다.

### Phase 5 — Comp와 안정화

- 구간별 Comp/crossfade
- latency compensation
- 미사용 녹음 정리
- 장치 분리, sample rate 변경, 디스크 부족 복구
- 필요하면 `getUserMedia + AudioWorklet` 브라우저 fallback 추가

## 10. 필수 테스트

- 입력 장치 없음/녹음 중 분리
- 입출력 sample rate 불일치
- buffer size별 지연과 드롭아웃
- Limiter On/Off, 과입력, Mono/Stereo
- 루프 경계가 오디오 블록 중간에 있는 경우
- 녹음 중 Stop, 앱 종료, 디스크 부족
- Take 100개 이상과 장시간 녹음
- clip 이동 후 seek, loop, BPM/key 변경
- 경계 click/pop과 Comp crossfade
- 파일 트랙+Audio In 혼합 Merge/Export
- 기존 `.focus` migration, Save As, 파일 재연결
- Undo/Redo 후 파일 참조 정합성
- 네이티브 전환 시 이중 재생 방지

## 11. 결정 요약

| 항목 | 권장 결정 |
|---|---|
| 파일 트랙 위치 | 0초 고정, 이동 불가 |
| 파일 트랙 접기 | `File Tracks` 그룹을 UI에서 접기 |
| 여러 트랙 합치기 | 비파괴 Offline Bounce, 원본 유지+Mute 기본 |
| Audio In 저장 | 메모리가 아닌 스트리밍 WAV |
| 녹음 엔진 | 우선 JUCE 네이티브 |
| Limiter | 기본 On, -1 dBFS, 녹음/Monitor 동일 적용 |
| 다중 녹음 | 루프마다 독립 source와 Take 생성 |
| Audio In 이동 | clip `start` 변경과 snap |
| 구간 삭제 | 기본 gap 유지, Ripple은 별도 |
| 편집 방식 | 원본 WAV를 바꾸지 않는 비파괴 편집 |
| 합치기 | Join/Consolidate/Merge를 구분 |
| 파일 삭제 | 즉시 삭제하지 않고 Unused 관리 |

가장 중요한 선행 작업은 **트랙당 단일 버퍼 모델을 source/clip 모델로 확장하는 것**이다. 이 기반 없이 녹음부터 추가하면 Take마다 임시 트랙이나 버퍼 복제가 필요해 이동, 저장, Undo, 네이티브 동기화가 함께 복잡해진다. 모델을 먼저 정리하면 기존 split/join/clip volume 구현을 살리면서 Audio In을 단계적으로 추가할 수 있다.

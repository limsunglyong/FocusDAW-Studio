## 0. 현황 분석

- [advanced-eq.html](advanced-eq.html) / [advanced-eq-app.jsx](advanced-eq-app.jsx) 은 이미 **창 프레임만 존재하는 빈 스텁**입니다(타이틀바 + View 전환 메뉴 + 푸터, 내부는 "coming soon").
- 창 자체는 이미 배선 완료: [electron/main.js](electron/main.js#L718) `advancedFiles.eq`, [scripts/build-renderers.js](scripts/build-renderers.js#L17) 빌드 목록, [ui-kit.jsx](ui-kit.jsx#L281) `AdvancedViewMenu` 의 Pan ↔ Ambience ↔ EQ 전환. → **새 창/IPC 신설 불필요, 단일 Advanced 창을 재사용**.
- 오디오 엔진에 **9밴드 그래픽 EQ가 이미 구현**되어 있음:
  - [audio-engine.js](audio-engine.js#L284) `EQ_FREQS = [60,150,320,640,1200,2400,4800,9000,15000]`, `EQ_PRESETS = {Flat, Pop, Classic, HipHop}`.
  - `DAW.master.bands`(−12~+12 dB ×9), `DAW.setMasterBand(i,v)`, `DAW.applyEQPreset(name)`, 실제 사운드는 `eqNodes` 체인으로 적용됨.
  - 출력단 FFT: `masterAnalyser`(fftSize 1024) → `DAW.computeSpectrum()`(전체 스펙트럼), `DAW.getMasterBandLevels()`(9밴드 레벨). 둘 다 이미 Mixer로 브로드캐스트 중([app.jsx](app.jsx#L997)).
- 동기화 버스: `BroadcastChannel("focusdaw-advanced-effects-sync")`. Ambience 창이 이 채널로 `REQUEST_PLAY_PAUSE`/`BEFORE_CHANGE`/`SET_*` 패턴을 사용 → **동일 패턴 그대로 차용**.

→ 결론: **새 DSP는 필요 없고**, 기존 master EQ 인프라를 Advanced 창에서 "크고 미려하게" 조작·시각화하는 작업이 핵심입니다.

## 1. 요청 항목별 구현 방안

### (1) advanced-eq.html 창 구현 / (2) 진보된 EQ 적용 방식
- [advanced-eq-app.jsx](advanced-eq-app.jsx) 의 빈 `aef-room` 내부를 **풀사이즈 Graphic EQ 에디터**로 채움.
- Ambience 창과 동일하게 **로컬 React state + BroadcastChannel** 방식(별도 DAW shim 없음). 스튜디오가 single source of truth, 창은 메시지로 상태를 수신·요청.

### (3) Mixer 기본 preset 가져오기
- 엔진의 `EQ_PRESETS`(Flat·Pop·Classic·HipHop)를 **`INIT_STATE`에 동봉**하여 그대로 노출.
- 프리셋 버튼 클릭 → `APPLY_EQ_PRESET` 메시지(현재 Mixer 채널에만 있는 핸들러를 **Advanced 채널에도 추가**).

### (4) 화려하고 미려한 그래픽
- 대형 EQ 캔버스(가변, 창 크기에 맞춰 확장):
  - 로그 주파수축 그리드 + dB 그리드(0dB 강조).
  - EQ 곡선: 부드러운 path + 그라데이션 채움 + 글로우(테마 `--amber` 계열).
  - 밴드별 드래그 핸들: 저역(red)/중역(amber)/고역(blue) 존 색상, LED링 스타일.
  - 전 구간 테마 변수(`var(--…)`) 사용 → 10종 컬러 테마 자동 추종.

### (5) Space 재생/일시정지
- Ambience와 동일한 keydown 핸들러([advanced-ambience-app.jsx](advanced-ambience-app.jsx#L336) 패턴):
  - `Space` → `REQUEST_PLAY_PAUSE`, `Esc`/`F3` → 창 닫기, `Ctrl/⌘+Z` → `REQUEST_UNDO`, `Ctrl+Y`/`Shift+Z` → `REQUEST_REDO`.

### (6) 주파수별 GUI 세팅
- 9개 밴드 핸들을 **세로 드래그**로 −12~+12 dB 설정, **더블클릭 시 0 dB 리셋**(Mixer `MasterEQOverlay` 인터랙션 동일).
- 드래그 중 `SET_MASTER_BAND`, 드래그 시작 시 `BEFORE_CHANGE`(undo 1회 캡처).
- 각 밴드 주파수 라벨(60·150·…·15k Hz)과 현재 dB 값 표시.
- ※ Q/주파수 가변(파라메트릭)은 현재 엔진 구조 밖이므로 **본 단계 범위 외**(향후 확장 후보로 본 문서 하단 기록).

### (7) Output Track 기준 FFT 배경
- 스튜디오가 EQ 창 열림 동안 매 프레임 Advanced 채널로 `DAW.computeSpectrum()` + `getMasterBandLevels()`를 추가 송신(현재 Advanced의 `LEVEL_METERS`는 trackLevels만 보냄 → **fft 필드 추가**).
- EQ 캔버스 **뒤 레이어**에 실시간 스펙트럼(글로우 영역/바)을 렌더 → 곡선 편집과 실제 출력 스펙트럼을 겹쳐 봄.

### (8) 사용자 Preset 저장 (5슬롯)
- Advanced 창 로컬 `localStorage["focusdaw-eq-user-presets"]`에 슬롯 5개 저장: `{name, bands[9]}`.
- 각 버튼 클릭 시 **오버롤(over-roll) 팝오버 메뉴** 펼침 → **Save / Recall** 선택.
  - Save: 현재 `bands`를 슬롯에 저장.
  - Recall: 슬롯 `bands`를 적용(9회 `SET_MASTER_BAND`, 단일 `BEFORE_CHANGE`로 묶음).
- **프리셋 이름 변경**: 슬롯 라벨 더블클릭(또는 메뉴 내 Rename) → 인라인 편집, localStorage 반영.
- 저장 데이터는 창 로컬이라 프로젝트/엔진 비침투 → 기존 프로젝트 무영향.

### (9) 본 제안서 작성 → 보고 → 승인 대기
- 본 문서가 그 산출물. 승인 후 (2)의 순서로 구현.

## 2. 코드 변경 요약 (승인 후)

| 파일 | 변경 |
|---|---|
| [advanced-eq-app.jsx](advanced-eq-app.jsx) | 빈 `aef-room` → Graphic EQ 에디터(곡선·핸들·FFT 배경·프리셋 바·유저 슬롯) 전체 구현. Space/Undo/Redo keydown. |

| [version.js](version.js) · [package.json](package.json) · package-lock.json | 버전 `1.8.0`. |
| [앱개발.md](앱개발.md) · [시험.md](시험.md) | Changelog #143 및 시험 항목(B-169~) 기록. |

## 3. 향후 확장 후보(이번 범위 외)
- 파라메트릭 EQ(밴드별 주파수/Q 가변), 스펙트럼 hold/peak, 좌우 채널 분리 EQ. → 엔진 `eqNodes` 구조 확장 필요.

## 3. JUCE 네이티브 DSP 통합 및 검증 흐름

실시간 피치/템포 변환은 Rubber Band 및 SoundTouch C++ 원본 라이브러리를 직접 탑재하여 수행하며, 재생 음질 및 내보내기 결과의 일치성을 최우선으로 검증합니다.

```mermaid
flowchart TD
    INPUT[오디오 트랙 데이터]
    JUCE_DSP[JUCE Audio Engine<br/>Rubber Band / SoundTouch C++]
    CHECK{실시간 재생 시<br/>품질/속도/Key 정상인가?}

    OK[실시간 플레이백 통과]
    RETRY[DSP 파라미터 튜닝<br/>버퍼 크기 조정]
    EXPORT[JUCE 오프라인 렌더러 작동]
    MATCH{Export 결과가<br/>재생 음질과 일치하는가?}
    
    FINAL_OK[JUCE 엔진 정식 채택]
    DEBUG[렌더 스케줄링<br/>및 스레드 동기화 디버그]

    INPUT --> JUCE_DSP --> CHECK
    CHECK -->|아니오| RETRY --> JUCE_DSP
    CHECK -->|예| OK --> EXPORT --> MATCH
    MATCH -->|예| FINAL_OK
    MATCH -->|아니오| DEBUG --> EXPORT

    style JUCE_DSP fill:#2d5a3d,color:#fff
    style CHECK fill:#2d5a3d,color:#fff
    style MATCH fill:#2d5a3d,color:#fff
    style FINAL_OK fill:#2d5a3d,color:#fff
```

# 지침

*중요*
1. 하단의 수정요청사항을 읽고 문제점을 파악한뒤 수정계획을 `앱개발.md`에 기록한다.
2. 사용자에게 내용을 알리고 수정에 대한 승인을 받은 뒤 코드/문서 수정을 진행한다.
3. 수정이 완료되면 구현된 사항을 `앱개발.md` 에 기록하고 이에 대한 시험 방안을 쉽고 간단하게 [시험항목], [시험방법 및 절차], [시험 환경 준비] 등의 내용을 시험.md 에 내용을 추가한다.
4. 아래 수정요청사항의 각 항목에 대한 수정이 완료되면 항목을 삭제한다. 만약 어떤 이유로 수정을 하지 못했다면 그 이유와 함께 적어두고 혹시 추가 수정사항이 필요한 경우 해당 내용을 추가해둔다.

# 수정요청사항

1. Export 단계에서 아래와 같이 wav/mp3 의 정보를 넣을 수 있도록 해주세요.
- 앨범 사진 : 사진은 wav 선택일 때는 비활성화. mp3에서만 적용함. default 값은 없음. 사진을 넣는 방식을 드롭 다운 방식과 파일 선택 창을 띄워서 선택할 수 있도록.
- 작곡가 : default 는 unknown.
- 앨범년도 : default 는 현재의 년도 사용.
- 제목 : default 는 프로젝트 제목과 동일한 값으로.
- 앨범 제목 : default 는 프로젝트 제목과 동일한 값으로.
- 날짜 : default 는 현재 날짜 사용.

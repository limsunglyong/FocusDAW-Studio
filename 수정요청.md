# 지침

*중요*
1. 하단의 수정요청사항을 읽고 문제점을 파악한뒤 수정계획을 `앱개발.md`에 기록한다.
2. 앱 버전 {Major}.{Minor}.{Patch} 을 변경한다.
   - Major 버전 : 사용자의 요구에 따라 변경
   - Minor 버전 : 기능 추가 또는 큰 구조 변경이 있을 시 +1
   - Patch 버전 : 버그 수정, 기능 수정, 그 밖에 사소한 변경이 있으면 +1
3. 사용자에게 내용을 알리고 수정에 대한 승인을 받은 뒤 코드/문서 수정을 진행한다.
4. 수정이 완료되면 구현된 사항을 `앱개발.md` 에 기록하고 이에 대한 시험 방안을 쉽고 간단하게 [시험항목], [시험방법 및 절차], [시험 환경 준비] 등의 내용을 시험.md 에 내용을 추가한다.
5. 아래 수정요청사항의 각 항목에 대한 수정이 완료되면 항목을 삭제한다. 만약 어떤 이유로 수정을 하지 못했다면 그 이유와 함께 적어두고 혹시 추가 수정사항이 필요한 경우 해당 내용을 추가해둔다.

# 수정요청사항


## 2. 앱 초기 실행 시 BPM 미설정 값이 `---`이 아니라 `20`으로 표시되는 문제

- **현상**: 앱을 처음 실행하거나 BPM이 아직 설정되지 않은 프로젝트를 불러올 때, BPM 표시가 `---`이 아니라 `20`으로 표시됨.
- **원인**: [audio-engine.js](audio-engine.js)의 `_normalizeBpm()`에서 `Number(null)`이 `0`으로 변환되고, 이후 최소값 보정(`Math.max(20, ...)`) 때문에 `null` BPM이 `20`으로 저장됨.
- **영향 경로**:
  - 앱 시작 시 [app.jsx](app.jsx)의 `loadRecentProject()`가 최근 프로젝트를 자동 로드.
  - [audio-engine.js](audio-engine.js)의 `importProject()` / `applySnapshot()`에서 `tempo.projectBpm` 또는 `tempo.playbackBpm`이 `null`인 경우에도 `_normalizeBpm(null)`을 호출.
  - 결과적으로 미설정 BPM이 `20`으로 정규화되어 UI가 `---` 대신 `20`을 표시.
- **수정 방향**:
  - `_normalizeBpm()`에서 `null`, `undefined`, 빈 문자열(`""`)을 먼저 검사해 `null`을 반환하도록 변경.
  - BPM이 실제 숫자로 입력된 경우에만 20~300 범위로 클램프.
  - 앱 초기 실행, 빈 프로젝트, BPM 미설정 최근 프로젝트 로드 시 BPM 표시가 `---`인지 확인.

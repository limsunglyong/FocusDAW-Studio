# FocusDAW Studio 성능 최적화 및 트러블슈팅 가이드

본 문서는 FocusDAW Studio 실행 시 고사양 데스크톱(예: Intel i9 하이브리드 CPU)이 저사양 노트북보다 앱 로딩, 파일 디코딩, 타임라인 줌 조작 및 MP3 내보내기(Export) 등에서 성능이 느리게 나타나는 현상의 원인을 규명하고 이를 해결하기 위한 환경 설정 방법을 설명합니다.

---

## 1. Intel 하이브리드 CPU 스케줄링 (P-core / E-core) 이슈

Intel 12세대 이상(i9-13900K 등) CPU는 고성능 **P-코어(Performance Core)**와 저전력 **E-코어(Efficiency Core)**로 나뉘어 있습니다. Windows 스케줄러(Thread Director)는 백그라운드 프로세스나 서명되지 않은 커스텀 Electron 앱, 그리고 자식 프로세스(`ffmpeg`)를 E-코어에 강제 배치하여 성능 저하를 유발하곤 합니다.

### 증상
* 앱 로딩 시 JSX 실시간 컴파일(Babel 트랜스파일링) 속도가 느려짐.
* MP3/WAV 파일 로드 시 디코딩 및 파형 피크 캐시 추출 연산 지연.
* MP3 Export 시 인코딩 속도가 비정상적으로 느려짐.

### 해결 방법
1. **Windows 전원 모드 변경**:
   * `설정` -> `시스템` -> `전원(또는 전원 및 배터리)`으로 이동합니다.
   * 전원 모드를 **"최고 성능"**으로 설정하여 OS가 고성능 코어를 적극적으로 할당하게 만듭니다.
2. **프로세스 우선순위 설정**:
   * 앱 실행 후 `작업 관리자` -> `세부 정보` 탭을 엽니다.
   * `electron.exe` 및 인코딩 시 활성화되는 `ffmpeg.exe` 프로세스를 우클릭한 뒤, **우선순위 설정**을 **"높음"**으로 변경합니다.
   * **선호도 설정(Affinity)**에서 E-코어(인덱스가 높은 코어들) 선택을 해제하고 P-코어만 선택하여 성능을 강제 조율할 수 있습니다.

## 2. 크로미움 하드웨어 가속 (GPU) 점검

타임라인의 줌인/아웃 및 줌 트래킹 동작은 다수의 캔버스(`canvas`) 엘리먼트에 파형 데이터를 그리는 렌더링 집약적 작업입니다. 외장 그래픽 카드를 사용하는 데스크톱에서 하드웨어 가속이 꺼져 있으면 CPU가 이 작업을 처리하므로 심각한 병목이 발생합니다.

### 증상
* 타임라인 줌인/아웃 슬라이더 작동 시 화면 프레임이 심하게 떨어지고 반응이 둔해짐.
* 플레이헤드 이동 시 화면 재생 지연 발생.

### 해결 방법
1. **GPU 가속 활성화 여부 확인**:
   * 앱 내 개발자 도구(DevTools) 콘솔을 열거나 일반 Chrome 브라우저에서 `chrome://gpu`를 입력하여 **Canvas: Hardware accelerated** 상태인지 확인합니다.
   * 하드웨어 가속이 `Software only, hardware acceleration unavailable`로 되어 있다면 그래픽 드라이버를 최신 버전으로 업데이트해야 합니다.
2. **Electron 가속 옵션 확인**:
   * `electron/main.js` 내부에서 `app.disableHardwareAcceleration()`이 실수로 호출되었는지 점검하고, 필요한 경우 가속이 켜진 상태로 시작되도록 유지합니다.

## 3. NVIDIA G-Sync 및 백그라운드 프레임 제한

데스크톱 그래픽 카드 제어판(예: NVIDIA 제어판)의 전역 설정이 창 모드(Windowed) 응용 프로그램에 엄격한 프레임 제한을 걸거나, G-Sync 동작 버그로 인해 프레임이 낮게 고정되는 경우가 있습니다.

### 증상
* 창 모드로 실행된 FocusDAW Studio의 화면 주사율이 20~30 FPS로 강제 클램핑되어 마우스 포인터 반응과 화면 렌더링 전체가 버벅임.

### 해결 방법
1. **NVIDIA 제어판 설정**:
   * 바탕화면 우클릭 후 `NVIDIA 제어판`을 엽니다.
   * `3D 설정 관리` -> `프로그램 설정` 탭으로 이동합니다.
   * 사용자 정의 프로그램에 `electron.exe` 또는 개발용 `node.exe`를 추가합니다.
   * 다음 설정을 조정합니다:
     * **모니터 기술**: `기본 재생 빈도` (G-Sync 비활성화 또는 호환성 모드로 전환)
     * **최대 프레임 속도**: `끄기`
     * **백그라운드 응용 프로그램 최대 프레임 속도**: `끄기`
2. **모니터 주사율 동기화**:
   * 고주사율(144Hz 이상) 모니터를 사용하는 경우, Windows 디스플레이 설정에서 올바른 주사율로 설정되어 있는지 재확인합니다.

## 4. 백신 실시간 감시 및 Ransomware 방어 예외 설정

데스크톱에 설치된 타사 백신(AhnLab V3, 알약 등) 또는 Windows Defender의 실시간 보호 기능은 서명되지 않은 실행 파일의 파일 I/O 및 자식 프로세스 생성을 검사하느라 레이턴시를 추가합니다.

### 증상
* 오디오 스템 파일이 담긴 폴더를 등록하거나 개별 MP3를 로드할 때 초기 I/O 지연 발생.
* MP3 파일 익스포트 시작 시 수 초간 멈춤(Freezing) 현상 발생.

### 해결 방법
* 사용 중인 백신 프로그램이나 Windows 보안의 `바이러스 및 위협 방지` 설정으로 이동합니다.
* `제외` 목록에 **FocusDAW Studio 프로젝트 루트 폴더** 및 오디오 스템 폴더를 등록하여 검사 오버헤드를 우회합니다.

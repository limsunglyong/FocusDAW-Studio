

답변: 현재는 불가능합니다 (동시에 서로 다른 device로 녹음 X)
현재 구조는 단일 입력 장치 + 한 번에 한 트랙 녹음 모델입니다. 두 가지 구조적 이유가 있습니다:

1. 입력 장치가 트랙별이 아니라 앱 전역 설정 1개입니다

Settings의 입력 장치는 getSavedAudioInput() 하나의 전역 값(localStorage)입니다 (audio-bridge.js). 트랙에는 device 필드가 없고 arm / monitor / inputGain / limiter만 있습니다. 즉 모든 Audio In 트랙이 같은 장치를 공유합니다.
2. Arm(녹음 대기)이 배타적입니다 — 한 번에 한 트랙만 arm 가능

한 Audio In 트랙을 arm하면 나머지 armed 트랙은 자동으로 해제됩니다 (app.jsx:2687):

if (track.id !== id && track.kind === "audioIn" && track.params.arm)
  DAW.setTrackParam(track.id, "arm", false);
따라서 물리적으로 동시에 녹음되는 트랙은 항상 1개입니다.
3. 네이티브 엔진도 단일 장치 기반

JUCE AudioDeviceManager는 한 번에 활성 입력 장치 1개만 엽니다. arm 시 그 전역 장치를 열어 녹음합니다.
지금 가능한 "다중 입력"의 유일한 형태
같은 장치의 다른 채널 선택은 됩니다 — Input Channel 선택(Mono Input 1 / Mono Input 2 / Stereo 1–2, ui-dialogs.jsx:985). 하지만 이것도 arm이 배타적이라 동시 녹음은 안 됩니다.

서로 다른 device 동시 녹음을 지원하려면 (참고)
큰 아키텍처 변경이 필요합니다:

트랙별 입력 장치 지정 — 전역 1개 → 트랙마다 device/channel 필드
네이티브 다중 장치 입력 — JUCE 단일 device 모델을 벗어나야 함. 실무에선 보통 OS의 애그리게이트(집합) 장치를 만들어 하나의 device처럼 묶거나(가장 현실적), 여러 device 인스턴스를 각각 열어 클럭 드리프트 동기화 처리
배타적 arm 제약 제거 + 여러 입력 스트림 동시 캡처/믹싱/싱크
가장 현실적인 우회책은 오디오 인터페이스(다입력 장치)나 OS 애그리게이트 장치로 여러 마이크를 하나의 device·여러 채널로 묶는 것입니다. 다만 그 경우에도 지금은 채널당 개별 트랙 동시 녹음이 아니라 arm된 1트랙만 녹음됩니다.
